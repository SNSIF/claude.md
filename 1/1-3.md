# 1-3. 기본 명령어

## CLI 플래그 상세 설명

### `claude` — 대화형 세션

인자 없이 실행하면 인터랙티브 REPL 환경이 시작됩니다.

```bash
claude
```

---

### `claude "task"` — 초기 프롬프트로 시작

세션을 시작하면서 첫 번째 메시지를 바로 전달합니다. 세션은 계속 유지됩니다.

```bash
claude "현재 프로젝트 구조를 분석해줘"
claude "src/auth.py에서 JWT 토큰 만료 처리를 추가해줘"
```

---

### `claude -p "query"` — 한 번 실행 후 종료 (non-interactive)

프롬프트를 실행하고 결과를 출력한 뒤 즉시 종료합니다. 스크립트, CI/CD 파이프라인에 적합합니다.

```bash
claude -p "이 프로젝트의 README를 한 줄로 요약해줘"
```

출력:
```
FastAPI + React 기반의 사용자 인증 및 게시판 웹 애플리케이션입니다.
```

셸 스크립트 내 사용 예:

```bash
#!/bin/bash
SUMMARY=$(claude -p "package.json의 프로젝트 설명을 한 문장으로 요약해줘")
echo "프로젝트: $SUMMARY"
```

---

### `cat file | claude -p "query"` — 파이프 입력

표준 입력(stdin)으로 파일 내용을 전달합니다.

```bash
cat error.log | claude -p "이 에러 로그에서 가장 중요한 오류를 찾아줘"
cat src/main.py | claude -p "이 코드를 리뷰해줘"
git diff HEAD~1 | claude -p "이 변경사항을 커밋 메시지로 요약해줘"
```

여러 파일 합쳐서 전달:

```bash
cat src/*.py | claude -p "중복 코드가 있으면 찾아줘"
```

---

### `claude -c` — 최근 대화 계속

마지막으로 사용한 세션을 이어서 시작합니다.

```bash
claude -c
claude -c "어제 작업하던 로그인 기능 이어서 완성해줘"
```

---

### `claude -r "session-id"` — 특정 세션 재개

세션 ID 또는 이름으로 특정 세션을 재개합니다.

```bash
claude -r "feature-auth-login"
claude -r "abc123def456"
```

세션 목록 확인:

```bash
claude --list-sessions
```

---

### `claude --model` — 모델 선택

사용할 Claude 모델을 지정합니다.

```bash
# 단축 이름 사용
claude --model sonnet
claude --model opus
claude --model haiku

# 전체 모델 ID 사용
claude --model claude-sonnet-4-5
claude --model claude-opus-4-5
claude --model claude-haiku-3-5
```

| 모델 | 특징 | 적합한 용도 |
|------|------|------------|
| haiku | 빠르고 저렴 | 간단한 질의, 자동화 |
| sonnet | 균형 잡힌 성능 | 일반 개발 작업 |
| opus | 최고 성능 | 복잡한 아키텍처, 심층 분석 |

---

### `claude --permission-mode` — 권한 모드 설정

Claude가 파일 수정, 명령 실행 시 어떻게 권한을 처리할지 설정합니다.

```bash
# 계획만 수립, 실행하지 않음
claude --permission-mode plan

# 모든 작업 자동 승인 (주의: 위험)
claude --permission-mode auto

# 기본값: 작업마다 승인 요청
claude --permission-mode default
```

`plan` 모드는 Claude가 무엇을 할 것인지 설명만 하고 실제로 파일을 건드리지 않습니다. 작업 계획을 검토한 후 승인하고 싶을 때 유용합니다.

```bash
claude --permission-mode plan "로그인 API를 리팩토링해줘"
# Claude가 수행할 단계를 나열하고 멈춤
# 확인 후 일반 모드로 재실행
```

---

### `claude update` — 업데이트

Claude Code를 최신 버전으로 업데이트합니다.

```bash
claude update
```

또는 npm으로 직접:

```bash
npm update -g @anthropic-ai/claude-code
```

---

### `claude --version` — 버전 확인

```bash
claude --version
# 출력: claude-code/1.x.x
```

---

### `claude --help` — 도움말

전체 CLI 옵션 목록을 출력합니다.

```bash
claude --help
```

## 출력 포맷 옵션

`-p` (non-interactive) 모드에서 출력 형식을 지정할 수 있습니다.

```bash
# 기본 텍스트 출력
claude -p "파일 요약해줘" --output-format text

# JSON 출력 (파싱 용이)
claude -p "파일 요약해줘" --output-format json

# 스트리밍 JSON (실시간 처리)
claude -p "파일 요약해줘" --output-format stream-json
```

### JSON 출력 예시

```bash
claude -p "현재 디렉토리 설명해줘" --output-format json
```

```json
{
  "response": "현재 디렉토리에는 FastAPI 백엔드 프로젝트가 있습니다...",
  "model": "claude-sonnet-4-5",
  "usage": {
    "input_tokens": 245,
    "output_tokens": 128
  }
}
```

### 파이프라인에서 JSON 파싱

```bash
RESULT=$(claude -p "에러 요약해줘" --output-format json | jq -r '.response')
echo "$RESULT"
```

### stream-json

실시간으로 토큰이 생성될 때마다 JSON 청크를 출력합니다. 긴 응답을 스트리밍 처리할 때 적합합니다.

```bash
claude -p "긴 코드 리뷰해줘" --output-format stream-json | while read line; do
  echo "$line" | jq -r '.delta // empty'
done
```

## 최대 턴 제한: `--max-turns`

에이전트가 자동으로 실행하는 최대 대화 턴 수를 제한합니다.

```bash
# 최대 5번 도구 호출 후 멈춤
claude -p "프로젝트 전체를 분석해줘" --max-turns 5
```

무한 루프 방지 및 비용 통제에 유용합니다.

---

## 예산 제한: `--max-budget-usd`

API 크레딧 사용 시 최대 비용을 달러로 제한합니다.

```bash
# 최대 $0.50 사용
claude -p "대규모 리팩토링 작업해줘" --max-budget-usd 0.50

# 최대 $2.00 사용
claude --max-budget-usd 2.00 "전체 테스트 코드 작성해줘"
```

예산 초과 시 Claude가 작업을 중단하고 현재까지의 진행 상황을 보고합니다.

## 플래그 조합 예제

### 예제 1: CI에서 코드 리뷰 자동화

```bash
git diff main...HEAD | claude -p "PR 코드 리뷰해줘. 버그와 개선사항을 나열해줘." \
  --output-format json \
  --max-turns 3 \
  --max-budget-usd 0.10
```

### 예제 2: 계획 확인 후 실행

```bash
# 먼저 계획만 확인
claude --permission-mode plan --model opus "인증 모듈 전체 리팩토링해줘"

# 계획이 마음에 들면 실제 실행
claude --model opus "인증 모듈 전체 리팩토링해줘"
```

### 예제 3: 빠른 질의 (Haiku 모델 사용)

```bash
claude -p "Python에서 리스트 중복 제거하는 가장 빠른 방법은?" --model haiku
```

### 예제 4: 로그 분석 파이프라인

```bash
tail -n 100 /var/log/app.log \
  | claude -p "ERROR 레벨 로그만 분류하고 빈도순으로 정리해줘" \
  --output-format text \
  --max-turns 2
```

## 환경 변수로 기본값 설정

매번 플래그를 입력하지 않으려면 환경 변수로 기본값을 설정할 수 있습니다.

```bash
# ~/.bashrc 또는 ~/.zshrc에 추가
export CLAUDE_MODEL="claude-sonnet-4-5"
export ANTHROPIC_API_KEY="sk-ant-..."
```

## 실습

### 실습 1: non-interactive 모드 테스트

```bash
claude -p "피보나치 수열을 출력하는 Python 함수를 작성해줘"
```

### 실습 2: 파이프 입력 테스트

```bash
echo "def add(a, b): return a - b  # 버그 있음" | claude -p "이 코드의 버그를 찾아줘"
```

### 실습 3: JSON 출력 파싱

```bash
claude -p "오늘 날짜를 알려줘" --output-format json | python3 -c "
import json, sys
data = json.load(sys.stdin)
print('응답:', data['response'])
"
```

### 실습 4: 예산 제한 실험

```bash
claude -p "매우 큰 프로젝트를 분석해줘" --max-budget-usd 0.001
# 예산 초과 메시지 확인
```
