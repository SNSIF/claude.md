# 5-2. 자동 메모리

자동 메모리는 Claude가 대화 세션 중 학습한 내용을 파일로 자동 저장하는 기능입니다. 다음 세션에서 이 파일을 읽어 사용자의 선호도, 프로젝트 결정사항, 반복 패턴 등을 기억합니다.

---

## 자동 메모리 개념

일반적인 AI 대화는 세션이 끝나면 모든 내용이 사라집니다. Claude Code의 자동 메모리는 이 한계를 극복합니다.

```
세션 1: "항상 함수형 컴포넌트만 써줘"
  → Claude가 memory/project.md에 저장

세션 2 (새 터미널):
  → 저장된 메모리 자동 로드
  → 코드 작성 시 함수형 컴포넌트 사용 (별도 지시 없이)
```

---

## 저장 위치

자동 메모리 파일은 프로젝트별로 분리되어 저장됩니다.

```
~/.claude/projects/
└── <프로젝트-경로-해시>/
    └── memory/
        ├── MEMORY.md         ← 인덱스 파일 (자동 관리)
        ├── user.md           ← 사용자 정보, 선호도
        ├── project.md        ← 프로젝트 결정사항
        ├── feedback.md       ← 수정·확인 사항
        └── reference.md      ← 외부 리소스
```

프로젝트 경로는 현재 작업 디렉토리의 절대 경로를 기반으로 해시되므로, 서로 다른 프로젝트의 메모리가 섞이지 않습니다.

---

## MEMORY.md 구조와 타입

메모리 항목은 4가지 타입으로 분류됩니다.

### `user` 타입 — 사용자 역할, 선호도

```markdown
---
type: user
created: 2025-03-15
updated: 2025-03-20
---

## 사용자 정보

- 역할: 풀스택 개발자, 팀 리드
- 경력: TypeScript 5년, Python 3년
- 선호: 간결한 코드보다 가독성 우선
- 응답 스타일: 단계별 설명보다 핵심 요약 선호
- 언어: 한국어로 대화, 코드 식별자는 영어
```

### `feedback` 타입 — 수정·확인 사항

```markdown
---
type: feedback
created: 2025-03-18
---

## 코드 스타일 피드백

- 화살표 함수보다 일반 함수 선언 선호 (가독성)
- 에러 메시지는 구체적으로 작성 요청 (코드 번호 포함)
- 주석은 "왜"를 설명하는 형식 선호
- 긴 조건문은 변수로 추출해서 명명하도록 수정 요청함
```

### `project` 타입 — 진행 중인 작업, 결정사항

```markdown
---
type: project
created: 2025-03-10
updated: 2025-03-22
---

## 현재 진행 중

- Phase 3: 결제 모듈 구현 중
- 사용 중인 결제 라이브러리: Stripe SDK v3
- 웹훅 처리 방식 결정: 큐 기반 비동기 처리

## 아키텍처 결정사항

- ORM: SQLAlchemy Core 사용 (ORM 레이어 미사용)
- 캐싱: Redis, TTL 5분
- 인증: JWT + Refresh Token 방식

## 완료된 작업

- 사용자 인증 모듈 완료 (2025-03-15)
- 상품 CRUD API 완료 (2025-03-18)
```

### `reference` 타입 — 외부 리소스 위치

```markdown
---
type: reference
created: 2025-03-12
---

## 외부 리소스

- Stripe 웹훅 문서: https://stripe.com/docs/webhooks
- 사내 API 규약: docs/api-conventions.md
- DB 스키마: docs/schema.dbml
- 배포 가이드: docs/deployment.md
```

## 세션 시작 시 로드 방식

Claude Code는 세션 시작 시 메모리 파일의 **처음 200줄**을 자동으로 컨텍스트에 포함합니다.

```
세션 시작
  ↓
CLAUDE.md 로드
  ↓
MEMORY.md 인덱스 확인
  ↓
각 메모리 파일 처음 200줄 로드
  ↓
대화 시작
```

200줄을 초과하는 메모리는 로드되지 않으므로, 자주 참조하는 내용을 파일 상단에 배치하는 것이 중요합니다.

---

## 언제 자동 저장되나

Claude가 대화 중 다음 정보를 발견하면 자동으로 메모리에 저장합니다.

| 상황 | 저장 타입 | 예시 |
|------|-----------|------|
| 사용자 역할/배경 언급 | `user` | "저는 스타트업 CTO입니다" |
| 코드 수정 요청 반복 | `feedback` | 3번 이상 같은 패턴 수정 |
| 기술 스택 결정 | `project` | "Redis 쓰기로 결정했어" |
| 외부 문서 참조 | `reference` | "이 문서 참고해서 구현해줘" |
| 명시적 저장 요청 | 해당 타입 | "이거 기억해줘" |

## 메모리 파일 frontmatter 형식

메모리 파일은 아래 frontmatter를 사용합니다.

```yaml
---
type: project          # user / feedback / project / reference
created: 2025-03-10    # 최초 생성일
updated: 2025-03-22    # 마지막 수정일
version: 3             # 자동 증가하는 버전 번호
---
```

## 비활성화 방법

자동 메모리 기능이 불필요하거나 민감한 정보가 저장되는 것을 원하지 않을 때 비활성화합니다.

### 환경 변수로 비활성화

```bash
# 터미널에서 설정
export CLAUDE_CODE_DISABLE_AUTO_MEMORY=true

# 또는 .bashrc / .zshrc에 영구 설정
echo 'export CLAUDE_CODE_DISABLE_AUTO_MEMORY=true' >> ~/.zshrc
```

### settings.json으로 비활성화

```json
{
  "env": {
    "CLAUDE_CODE_DISABLE_AUTO_MEMORY": "true"
  }
}
```

## 프로젝트별 메모리 확인

현재 프로젝트의 메모리가 어디에 저장되어 있는지 확인하는 방법입니다.

```bash
# 프로젝트 메모리 디렉토리 목록 보기
ls ~/.claude/projects/

# 현재 작업 디렉토리에 대응하는 메모리 찾기
# (경로 해시는 프로젝트마다 다름)
ls ~/.claude/projects/*/memory/
```

또는 `/memory` 커맨드를 사용하면 현재 프로젝트의 메모리를 바로 확인할 수 있습니다.

```
/memory
```

## 자동 메모리와 CLAUDE.md 비교

| 항목 | CLAUDE.md | 자동 메모리 |
|------|-----------|-------------|
| 작성자 | 개발자 (수동) | Claude (자동) |
| 내용 | 영구 프로젝트 규칙 | 세션 중 학습한 동적 정보 |
| 버전 관리 | Git에 포함 | `~/.claude/`에 로컬 저장 |
| 팀 공유 | O (Git push) | X (개인 로컬) |
| 우선순위 | 낮음 | 높음 (더 최신 정보) |

실무에서는 두 가지를 함께 사용합니다. CLAUDE.md에 변경되지 않는 규칙을 쓰고, 자동 메모리가 작업 진행 상황과 개인 선호도를 보완합니다.
