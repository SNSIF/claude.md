# 5-3. 메모리 관리

자동으로 저장된 메모리는 시간이 지나면 오래된 정보가 쌓입니다. 정기적으로 검토하고 불필요한 항목을 정리하면 Claude의 응답 품질이 높아집니다.

---

## `/memory` 커맨드

현재 프로젝트의 메모리 파일을 확인하고 편집합니다.

```
/memory
```

실행하면 메모리 파일 목록이 표시됩니다.

```
현재 프로젝트 메모리

1. user.md       사용자 정보, 선호도          (42줄)
2. project.md    프로젝트 결정사항            (87줄)
3. feedback.md   코드 스타일 피드백           (35줄)
4. reference.md  외부 리소스                  (18줄)

편집할 파일 번호를 선택하세요 (q: 종료):
```

번호를 입력하면 기본 에디터로 해당 파일이 열립니다.

---

## 메모리 파일 직접 편집

에디터로 직접 열어 편집할 수도 있습니다.

```bash
# 메모리 파일 위치 찾기
ls ~/.claude/projects/

# 특정 프로젝트 메모리 열기 (VS Code)
code ~/.claude/projects/<해시>/memory/project.md

# 또는 터미널 에디터
nano ~/.claude/projects/<해시>/memory/project.md
vim ~/.claude/projects/<해시>/memory/feedback.md
```

현재 작업 디렉토리에 해당하는 해시 폴더를 찾는 방법:

```bash
# Claude Code 내에서
/memory
# 첫 줄에 파일 경로가 표시됨
```

## 특정 메모리 삭제

### 파일 전체 삭제

더 이상 필요 없는 메모리 타입 전체를 삭제합니다.

```bash
rm ~/.claude/projects/<해시>/memory/reference.md
```

### 특정 항목만 삭제

파일을 열어 해당 섹션만 지웁니다.

```bash
# 편집기로 열기
code ~/.claude/projects/<해시>/memory/project.md

# 오래된 섹션 삭제 후 저장
```

삭제 후 MEMORY.md 인덱스가 자동으로 업데이트됩니다. 수동 수정이 필요한 경우:

```bash
# MEMORY.md에서 삭제한 파일 참조 제거
code ~/.claude/projects/<해시>/memory/MEMORY.md
```

## "기억해줘" — 명시적 저장 요청

대화 중 특정 정보를 메모리에 저장하도록 명시적으로 요청할 수 있습니다.

```
앞으로 이 프로젝트에서 에러 처리는 항상 Result 타입 패턴을 쓸 거야. 기억해줘.
```

Claude가 해당 내용을 적절한 타입(이 경우 `project` 또는 `feedback`)의 메모리 파일에 저장합니다.

```
이 API 문서 URL 기억해줘: https://api.myservice.com/docs
```

`reference` 타입으로 저장됩니다.

```
나는 주석 없이 코드만 보여주는 걸 선호해. 앞으로 기억해줘.
```

`user` 타입으로 저장됩니다.

## "잊어줘" — 특정 메모리 삭제 요청

저장된 메모리를 삭제하도록 요청합니다.

```
아까 기억해달라고 했던 Result 타입 패턴 기억 지워줘. 방향이 바뀌었어.
```

Claude가 해당 내용을 메모리 파일에서 찾아 삭제합니다.

```
내 역할에 대한 정보 다 지워줘. 처음부터 다시 알려줄게.
```

`user.md`의 역할 관련 내용이 삭제됩니다.

## MEMORY.md 인덱스 파일

`MEMORY.md`는 모든 메모리 파일의 목차 역할을 합니다. Claude Code가 자동으로 관리하지만 구조를 이해하면 디버깅에 도움이 됩니다.

```markdown
# 프로젝트 메모리 인덱스

프로젝트: /Users/john/projects/my-saas
마지막 업데이트: 2025-03-22

## 파일 목록

- [user.md](./user.md) — 사용자 정보, 개발 선호도
- [project.md](./project.md) — 아키텍처 결정, 진행 상황
- [feedback.md](./feedback.md) — 코드 리뷰 피드백 패턴
- [reference.md](./reference.md) — 참고 문서, 외부 리소스

## 세션 요약

최근 세션 (2025-03-22):
- 결제 모듈 웹훅 처리 구현 완료
- Stripe 이벤트 큐 구조 확정
- 다음 작업: 구독 상태 동기화 로직
```

## 메모리 정기 정리 방법

메모리가 오래될수록 부정확한 정보가 쌓입니다. 다음 주기로 정리를 권장합니다.

### 주간 정리

```
지금까지 저장된 메모리 내용을 보여주고, 오래됐거나 바뀐 내용이 있으면 알려줘.
```

Claude가 현재 메모리를 읽고 시제가 맞지 않는 항목(예: "구현 중"인데 이미 완료된 것)을 찾아줍니다.

### Phase 전환 시 정리

```
Phase 3가 완료됐어. project.md에서 Phase 3 관련 진행 중 항목들을 완료로 업데이트하고, Phase 4 시작을 기록해줘.
```

### 전체 초기화

메모리를 완전히 리셋하고 싶을 때:

```bash
# 현재 프로젝트 메모리 전체 삭제
rm -rf ~/.claude/projects/<해시>/memory/

# 또는 특정 파일만
rm ~/.claude/projects/<해시>/memory/feedback.md
```

## 메모리 vs CLAUDE.md 차이와 활용 전략

두 가지의 차이를 명확히 구분하면 더 효과적으로 사용할 수 있습니다.

| 항목 | CLAUDE.md | 자동 메모리 |
|------|-----------|-------------|
| 작성 주체 | 개발자 | Claude (자동) |
| 내용 유형 | 변하지 않는 규칙 | 변하는 상태, 선호도 |
| 갱신 방법 | 수동 편집 | Claude가 자동 갱신 |
| Git 포함 | 포함 (팀 공유) | 미포함 (개인 로컬) |
| 용도 | "항상 이렇게 해" | "지금까지 이렇게 했어" |

### 실용적인 활용 전략

**CLAUDE.md에 넣어야 할 것:**

```markdown
# 변하지 않는 규칙 → CLAUDE.md
- TypeScript strict 모드 필수
- 함수형 컴포넌트만 사용
- 에러 처리는 Result 타입 패턴
```

**자동 메모리에 맡길 것:**

```
# 변하는 상태 → 자동 메모리
- 현재 어떤 Phase를 진행 중인가
- 이번 주에 내린 기술 결정
- 사용자가 최근 요청한 수정 패턴
```

## 실습

### 실습 1: 현재 메모리 확인

```
/memory
```

각 파일을 열어 어떤 내용이 자동 저장되어 있는지 확인합니다.

### 실습 2: 명시적 메모리 저장

```
우리 프로젝트는 API 응답에 항상 { data, error, meta } 구조를 사용해. 기억해줘.
```

저장 후 확인:

```
/memory
# project.md 선택 → 방금 저장된 내용 확인
```

### 실습 3: 메모리 수정

```
아까 저장한 API 응답 구조에서 meta 필드는 선택적으로 변경됐어. 수정해줘.
```

### 실습 4: 오래된 메모리 정리

```
지금까지 기억하고 있는 내용 중 3개월 이상 된 것들 보여줘. 삭제할지 확인할게.
```

### 실습 5: 비활성화 테스트

```bash
# 메모리 비활성화 후 세션
export CLAUDE_CODE_DISABLE_AUTO_MEMORY=true
claude

# 정보 공유 후 세션 종료 → 재시작 → 기억하지 못하는지 확인
unset CLAUDE_CODE_DISABLE_AUTO_MEMORY
```
