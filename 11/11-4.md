# 11-4. 원격 제어 (Remote Control)

## 원격 제어란?

원격 제어(Remote Control)는 로컬 머신에서 실행 중인 Claude Code 세션을 웹 브라우저나 모바일 앱에서 제어할 수 있게 해주는 기능입니다.

즉, 집 PC에서 Claude Code를 실행해두고 외출 중에 스마트폰으로 진행 상황을 확인하거나 명령을 내릴 수 있습니다.

```
로컬 PC (Claude Code 실행)
       ↕  (같은 Anthropic 계정)
Claude.ai 웹 또는 모바일 앱
```

---

## 원격 제어 활성화

로컬 머신에서 아래 명령으로 원격 제어 세션을 시작합니다.

```bash
claude --remote-control "My Project"
```

실행하면 터미널에 다음과 같은 메시지가 출력됩니다.

```
Remote control enabled for session "My Project"
Connect at: https://claude.ai/remote-sessions
Session ID: rcss_xxxxxxxxxxxxxxxx
Waiting for remote commands...
```

### 옵션

```bash
# 세션 이름 지정 (웹/앱에서 식별용)
claude --remote-control "백엔드 리팩토링"

# 특정 디렉토리에서 원격 제어 시작
claude --remote-control "프론트엔드 작업" --dir /home/user/projects/frontend

# 읽기 전용 모드 (명령 실행 불가, 모니터링만)
claude --remote-control "빌드 모니터링" --read-only
```

---

## Claude.ai 웹에서 로컬 세션 제어

1. **https://claude.ai** 에 접속합니다.
2. 로컬 머신과 **동일한 Anthropic 계정**으로 로그인합니다.
3. 좌측 사이드바에서 **"Remote Sessions"** 메뉴를 클릭합니다.
4. 활성화된 세션 목록에서 원하는 세션(예: "My Project")을 선택합니다.
5. 채팅창에 명령을 입력하면 로컬 머신에서 실행됩니다.

```
[웹 채팅창]
> src/auth.py 파일의 버그를 찾아서 수정해줘

[로컬 터미널 — 자동 실행]
Claude Code: src/auth.py를 분석 중...
  - 17번째 줄: JWT 토큰 만료 검증 누락 발견
  - 수정 적용 중...
  ✓ 완료
```

---

## Claude 모바일 앱에서 제어

1. **Claude 모바일 앱**(iOS / Android)을 설치합니다.
2. 동일한 Anthropic 계정으로 로그인합니다.
3. 앱 하단 탭에서 **"Sessions"** 탭을 선택합니다.
4. 로컬에서 실행 중인 세션이 표시됩니다.
5. 세션을 탭하고 메시지를 입력합니다.

**모바일 전용 기능:**
- 푸시 알림으로 작업 완료 통보
- 음성 입력으로 명령 전달
- 터미널 출력 실시간 스트리밍 확인

---

## 요구사항

원격 제어를 사용하려면 아래 조건이 모두 충족되어야 합니다.

| 요구사항 | 설명 |
|----------|------|
| 동일 계정 | 로컬 머신과 웹/앱이 같은 Anthropic 계정으로 로그인되어 있어야 함 |
| 네트워크 연결 | 로컬 머신이 인터넷에 연결되어 있어야 함 |
| Claude Code 버전 | 최신 버전 (`claude --version`으로 확인, `claude update`로 업데이트) |
| 활성 세션 | `--remote-control` 플래그로 세션이 시작된 상태여야 함 |

## Web Sessions (클라우드 실행)

Web Sessions는 원격 제어와 다른 개념입니다. 로컬 설치 없이 클라우드 서버에서 Claude Code를 직접 실행하는 방식입니다.

```bash
# 클라우드에서 원격으로 작업 실행
claude --remote "Fix the login bug"
```

### Web Sessions 주요 특징

**1. 로컬 설치 불필요**
```bash
# 로컬에 Claude Code가 없어도 클라우드에서 실행 가능
# (단, 코드 저장소는 클라우드 환경에 있어야 함)
claude --remote "리포지토리를 클론하고 테스트 실행해줘" \
  --repo https://github.com/myorg/myproject
```

**2. 오래 걸리는 작업을 백그라운드로 처리**
```bash
# 명령만 내리고 터미널 종료해도 계속 실행
claude --remote "전체 코드베이스 리팩토링 후 PR 생성" --background
# → 세션 ID 반환: wsess_xxxxxxxxxx

# 나중에 결과 확인
claude --remote-status wsess_xxxxxxxxxx
```

**3. 모바일에서 진행 상황 확인**

백그라운드로 실행 중인 Web Session은 모바일 앱의 "Remote Sessions" 탭에서 실시간으로 로그와 진행 상황을 확인할 수 있습니다.

```
[모바일 앱 화면]
Web Session: "전체 리팩토링"
상태: 실행 중 (47분 경과)
─────────────────────────────
✓ src/auth/ 모듈 완료 (12개 파일)
✓ src/api/ 모듈 완료 (8개 파일)
⏳ src/database/ 처리 중 (6/15 파일)
```

## 원격 제어 vs Web Sessions 비교

| 항목 | 원격 제어 (Remote Control) | Web Sessions |
|------|--------------------------|--------------|
| 실행 위치 | 로컬 머신 | 클라우드 서버 |
| 로컬 설치 필요 | 필요 | 불필요 |
| 로컬 파일 접근 | 직접 접근 가능 | 클론/마운트 필요 |
| 터미널 종료 시 | 세션 종료 | 계속 실행 |
| 네트워크 끊길 시 | 세션 일시 중단 | 영향 없음 |
| 사용 사례 | 자리 비울 때 진행 모니터링 | 장시간 배치 작업 |
| 비용 | 로컬 리소스 사용 | 클라우드 컴퓨팅 비용 |

## 보안 고려사항

### 세션 접근 제어

원격 제어 세션은 반드시 같은 Anthropic 계정에서만 접근 가능합니다. 하지만 추가 보안을 위해 다음을 권장합니다.

```bash
# 세션에 PIN 설정
claude --remote-control "민감한 작업" --pin 1234

# 특정 시간 후 자동 만료
claude --remote-control "임시 작업" --expires 2h

# IP 화이트리스트 (특정 네트워크에서만 연결 허용)
claude --remote-control "작업" --allow-ip 192.168.1.0/24
```

### 민감 파일 노출 주의

원격 제어를 활성화하면 Claude Code가 접근할 수 있는 파일들이 원격에서도 제어 가능합니다.

**주의사항:**
- `.env` 파일, API 키, 비밀번호 파일이 있는 디렉토리에서는 신중하게 사용
- 공용 네트워크(카페 Wi-Fi 등)에서는 원격 세션을 활성화하지 않는 것을 권장
- 작업 완료 후 반드시 세션을 종료

```bash
# 세션 목록 확인
claude --remote-list

# 특정 세션 종료
claude --remote-kill rcss_xxxxxxxxxxxxxxxx

# 모든 원격 세션 종료
claude --remote-kill-all
```

### CLAUDE.md로 접근 제한

원격 세션에서 실행 가능한 작업을 프로젝트 레벨에서 제한할 수 있습니다.

```markdown
<!-- CLAUDE.md -->
## 원격 세션 제한 사항
- /secrets/ 디렉토리는 절대 읽지 말 것
- git push는 반드시 사용자 확인 후 실행
- 프로덕션 환경 변수 파일(.env.production) 접근 금지
```

## 실습

### 실습 1: 원격 제어 세션 시작하기

```bash
# 1. 현재 디렉토리에서 원격 제어 세션 시작
claude --remote-control "내 첫 원격 세션"

# 2. 출력된 Session ID를 메모해둠
# Session ID: rcss_xxxxxxxxxxxxxxxx

# 3. 다른 기기의 브라우저에서 claude.ai 접속 후 확인
```

### 실습 2: 백그라운드 Web Session 활용

```bash
# 1. 오래 걸리는 작업을 백그라운드로 실행
claude --remote "이 프로젝트의 모든 Python 파일에 type hint 추가해줘" \
  --background

# 2. 터미널을 닫고 나중에 확인
claude --remote-status wsess_xxxxxxxxxx

# 3. 모바일 앱으로도 확인 가능
```

### 실습 3: 읽기 전용 모니터링

```bash
# CI/CD 파이프라인 실행 중 모니터링만 허용
claude --remote-control "빌드 모니터링" --read-only

# 이 세션에서는 명령 실행 불가, 로그 확인만 가능
```
