# 11-1. Plan Mode

## Plan Mode 개념

Plan Mode는 Claude가 실제 파일을 수정하거나 명령을 실행하기 전에 **계획을 먼저 수립하고 사용자에게 제시**하는 작업 모드입니다. 대규모 변경이나 복잡한 리팩토링을 시작하기 전에 Claude의 의도를 명확히 파악하고 잘못된 방향으로 진행되는 것을 사전에 방지할 수 있습니다.

```
일반 모드:  요청 → [코드 분석] → [파일 수정] → [명령 실행] → 결과 보고
Plan 모드:  요청 → [코드 분석] → [계획 제시] → 사용자 승인 → [실행]
```

---

## Plan Mode의 특징

### 허용/제한 작업

| 작업 | Plan Mode | Auto Mode |
|------|-----------|-----------|
| 파일 읽기 | ✅ 허용 | ✅ 허용 |
| 디렉토리 탐색 | ✅ 허용 | ✅ 허용 |
| 파일 쓰기 | ❌ 불가 | ✅ 허용 |
| 파일 편집 | ❌ 불가 | ✅ 허용 |
| 명령 실행 | ❌ 불가 | ✅ 허용 |
| 파일 삭제 | ❌ 불가 | ✅ 허용 |

### Claude의 계획 제시 방식

Plan Mode에서 Claude는 작업 목록 형태로 계획을 제시합니다:

```
[Claude의 계획 예시]

다음 작업을 수행할 예정입니다:

1. 파일 분석
   - src/auth/login.ts 읽기
   - src/auth/token.ts 읽기
   - src/middleware/auth.ts 읽기

2. 변경 사항
   - src/auth/login.ts: JWT 만료 처리 로직 추가 (라인 45-67)
   - src/auth/token.ts: refreshToken 함수 신규 추가
   - src/middleware/auth.ts: 만료된 토큰 자동 갱신 미들웨어 추가

3. 테스트
   - tests/auth.test.ts: 새 함수에 대한 유닛 테스트 추가

이 계획을 진행해도 될까요?
```

## 활성화 방법

### 방법 1: CLI 플래그

```bash
# Plan Mode로 Claude Code 시작
claude --permission-mode plan

# 단축 형식
claude -p plan
```

### 방법 2: 실행 중 전환 (Shift+Tab)

```
현재 모드가 Auto일 때:
Shift+Tab 1번 → Plan Mode 전환
Shift+Tab 2번 → Manual Mode 전환
Shift+Tab 3번 → Auto Mode 복귀
```

상태바에서 현재 모드를 확인할 수 있습니다:
```
🟡 plan   ← Plan Mode 활성 중
```

### 방법 3: 기본값으로 설정

```json
// ~/.claude/settings.json
{
  "defaultMode": "plan"
}
```

프로젝트별 설정:
```json
// 프로젝트루트/.claude/settings.json
{
  "defaultMode": "plan",
  "planConfirmationRequired": true
}
```

### 방법 4: `/plan` 커맨드

대화 중 언제든지 Plan Mode로 전환:

```
> /plan
Plan Mode가 활성화되었습니다. 이제 파일을 읽고 계획을 세울 수 있지만
실제 변경을 수행하려면 모드를 전환해야 합니다.

> 사용자 인증 시스템을 OAuth2로 마이그레이션해줘
```

## 워크플로우

### Plan → Execute 전체 흐름

```
Step 1: Plan Mode 활성화
────────────────────────
$ claude --permission-mode plan

Step 2: 작업 지시
────────────────────────
> 현재 Express 앱을 FastAPI로 마이그레이션해줘

Step 3: Claude의 탐색 (읽기만)
────────────────────────
Claude: server.js 분석 중...
Claude: routes/ 디렉토리 탐색 중...
Claude: package.json 읽기 중...

Step 4: 계획 제시
────────────────────────
Claude:
  Phase 1: 프로젝트 구조 생성
  - requirements.txt 생성
  - app/ 디렉토리 구조 생성

  Phase 2: 라우터 마이그레이션 (5개 파일)
  - routes/auth.js → app/routers/auth.py
  - routes/users.js → app/routers/users.py
  ...

  Phase 3: 미들웨어 변환
  ...

  총 예상 변경: 23개 파일, 신규 생성 8개, 수정 15개

Step 5: 사용자 검토 및 피드백
────────────────────────
> Phase 2까지만 먼저 진행해줘. Phase 3는 나중에 따로 할게.

Step 6: Auto Mode로 전환하여 실행
────────────────────────
Shift+Tab (Plan → Auto)
> 위 계획의 Phase 1, 2를 실행해줘
```

## 대규모 리팩토링에서의 활용

### 사용 시나리오

**시나리오 1: 데이터베이스 마이그레이션**
```bash
claude --permission-mode plan
# > MySQL 쿼리를 모두 PostgreSQL 구문으로 변환해줘
# Claude가 변경이 필요한 파일 목록과 변경 내용을 제시
# 확인 후 실행 모드로 전환
```

**시나리오 2: 의존성 업그레이드**
```bash
claude --permission-mode plan
# > React 17에서 React 18로 업그레이드하고
#   변경이 필요한 모든 파일을 수정해줘
# Claude가 Breaking Changes 목록과 수정 계획 제시
```

**시나리오 3: 아키텍처 변경**
```bash
claude --permission-mode plan
# > 현재 MVC 구조를 Clean Architecture로 변환해줘
# Claude가 레이어별 이동/분리 계획 제시
```

## 실습

### 실습 1: 기능 추가를 Plan Mode로 접근

예제 프로젝트 준비:

```bash
mkdir plan-mode-demo && cd plan-mode-demo
git init

# 간단한 Express 앱 생성
cat > app.js << 'EOF'
const express = require('express')
const app = express()
app.use(express.json())

const users = []

app.get('/users', (req, res) => {
  res.json(users)
})

app.post('/users', (req, res) => {
  const user = { id: Date.now(), ...req.body }
  users.push(user)
  res.status(201).json(user)
})

app.listen(3000, () => console.log('Server running on port 3000'))
EOF

echo '{"name":"demo","version":"1.0.0"}' > package.json
```

**Plan Mode 실습**:

```bash
# Plan Mode로 시작
claude --permission-mode plan
```

프롬프트:
```
이 Express 앱에 다음 기능을 추가해줘:
1. 사용자 인증 (JWT)
2. 사용자 정보 수정 (PUT /users/:id)
3. 사용자 삭제 (DELETE /users/:id)
4. 입력값 유효성 검사

먼저 전체 계획을 세워줘.
```

예상 계획:
```
Claude의 예상 계획:

1. 패키지 추가
   - package.json: jsonwebtoken, express-validator 추가

2. 미들웨어 파일 생성
   - middleware/auth.js: JWT 검증 미들웨어

3. 라우터 분리
   - routes/users.js: 사용자 CRUD 라우터 이동

4. app.js 수정
   - JWT 미들웨어 등록
   - 라우터 연결

5. 유효성 검사 추가
   - POST /users: name, email 필수 검사
   - PUT /users/:id: 부분 업데이트 허용
```

계획 검토 후 승인:
```
Shift+Tab   # Auto Mode로 전환
> 위 계획을 실행해줘
```

### 실습 2: 계획 수정 후 실행

```bash
claude --permission-mode plan
```

프롬프트:
```
이 프로젝트의 테스트 커버리지를 80% 이상으로 높여줘
```

Claude의 계획을 받은 후 수정 요청:
```
> 계획에서 통합 테스트는 제외하고 유닛 테스트만 추가해줘.
  그리고 테스트 파일은 __tests__ 폴더 대신 src 옆의 tests 폴더에 만들어줘.
```

수정된 계획 확인 후 실행 모드로 전환.
