# 11-3. Worktree & 병렬 작업

## Git Worktree 심화

Git Worktree는 하나의 git 저장소에서 여러 브랜치를 동시에 서로 다른 디렉토리에 체크아웃하는 기능입니다. Claude Code와 결합하면 여러 작업을 완전히 독립된 환경에서 병렬로 진행할 수 있습니다.

```
기존 방식 (순차 작업):
main ──●──────────────────●── feature-auth 완료
       │    feature-auth      │
       └──────────────────────┘

Worktree + 병렬 Claude (병렬 작업):
main ──●──────────────────────────
       ├── worktree1 (feature-auth)  ← Claude 세션 1
       └── worktree2 (bugfix-payment) ← Claude 세션 2
```

---

## 사용 시나리오

### 시나리오 1: 기능 개발 + 버그 수정 동시 진행

```
상황: PR 리뷰 중 긴급 버그 발견
→ 기능 개발을 멈추지 않고 버그 수정을 병행
```

### 시나리오 2: 실험적 리팩토링

```
상황: 현재 개발을 유지하면서 리팩토링 아이디어 검증
→ 실험용 worktree에서 마음껏 시도, 실패해도 main에 영향 없음
```

### 시나리오 3: 여러 PR 동시 작업

```
상황: GitHub/GitLab에서 여러 PR을 병렬로 진행
→ 각 PR마다 worktree 생성, 독립 Claude 세션으로 처리
```

## 실제 사용법

### 기본 Worktree 생성

```bash
# 저장소 루트로 이동
cd /path/to/my-project

# 새 브랜치로 worktree 생성
git worktree add ../my-project-feature-auth feature-auth

# 기존 브랜치로 worktree 생성
git worktree add ../my-project-bugfix bugfix/payment-issue

# 새 브랜치 생성과 동시에 worktree 생성 (가장 많이 사용)
git worktree add -b feature-auth ../my-project-auth
git worktree add -b bugfix-payment ../my-project-bugfix
```

### 병렬 Claude 세션 실행

```bash
# 터미널 1: 인증 기능 개발
cd ../my-project-auth
claude
# 또는 프롬프트 직접 전달
claude -p "JWT 기반 사용자 인증 시스템을 구현해줘"
```

```bash
# 터미널 2: 결제 버그 수정 (완전히 독립적)
cd ../my-project-bugfix
claude
# 또는
claude -p "결제 처리 시 간헐적으로 발생하는 NullPointerException을 수정해줘"
```

### `--worktree` 플래그 사용

```bash
# 현재 디렉토리에서 바로 worktree 생성 후 진입
claude --worktree feature-auth
# → ../[프로젝트명]-feature-auth 디렉토리를 자동 생성하고 해당 경로에서 Claude 시작

claude --worktree bugfix-payment
# → ../[프로젝트명]-bugfix-payment 디렉토리에서 Claude 시작
```

## 각 Worktree의 특성

### 독립적인 파일 상태

```bash
# worktree1 (feature-auth)
ls src/auth/
# login.ts, register.ts, jwt.ts (새로 추가된 파일들)

# worktree2 (main)
ls src/auth/
# (feature-auth 파일이 없음 - 완전히 독립)
```

### 독립적인 Claude 컨텍스트

각 worktree에서 실행되는 Claude는 서로 다른 컨텍스트를 가집니다:

```
Claude (worktree1):
- 인식하는 파일: feature-auth 브랜치의 파일들
- 대화 이력: feature-auth 작업에 관한 내용
- 컨텍스트: 인증 시스템 구현 중

Claude (worktree2):
- 인식하는 파일: bugfix-payment 브랜치의 파일들
- 대화 이력: 결제 버그 수정에 관한 내용
- 컨텍스트: 결제 모듈 디버깅 중
```

### 공유된 git 이력

```bash
# 어느 worktree에서든 전체 git 이력 조회 가능
git log --oneline --all

# main의 최신 변경사항을 feature-auth에 반영
# (feature-auth worktree에서)
git merge main
# 또는
git rebase main
```

## Worktree 관리 명령어

### 목록 확인

```bash
git worktree list
```

출력 예시:
```
/home/user/my-project           abc1234 [main]
/home/user/my-project-auth      def5678 [feature-auth]
/home/user/my-project-bugfix    ghi9012 [bugfix-payment]
```

### 특정 Worktree 상태 확인

```bash
# 각 worktree에서 개별적으로 git status 실행
git -C ../my-project-auth status
git -C ../my-project-bugfix status
```

### Worktree 삭제

```bash
# 작업 완료 후 worktree 제거
git worktree remove ../my-project-auth

# 강제 삭제 (미저장 변경 있어도)
git worktree remove --force ../my-project-auth

# 브랜치도 함께 삭제
git worktree remove ../my-project-auth
git branch -d feature-auth  # 브랜치 별도 삭제
```

### 정리 (Prune)

```bash
# 이미 삭제된 디렉토리에 대한 참조 정리
git worktree prune
```

## 충돌 피하기

### 같은 파일 동시 편집 주의

```bash
# 위험한 상황: 두 worktree에서 같은 파일 수정 중
# worktree1: src/config/database.ts 수정
# worktree2: src/config/database.ts 수정
# → 나중에 merge 시 충돌 발생
```

**권장 전략**:
```
1. 작업 전 영역 분리
   - worktree1: src/auth/ 담당
   - worktree2: src/payment/ 담당

2. 공유 파일 변경 시 커뮤니케이션
   - 공유 설정 파일, 타입 정의 파일 등은 순차 처리

3. 주기적으로 main에서 rebase
   git -C ../my-project-auth rebase main
```

### 락(Lock) 파일 충돌 방지

```bash
# node_modules는 워크트리마다 별도 설치 필요
cd ../my-project-auth
npm install

cd ../my-project-bugfix
npm install
```

## 실전 예제: 두 Claude가 동시 작업

### 시나리오 설정

```bash
# 메인 프로젝트
cd ~/projects/ecommerce-app

# Worktree 1: 상품 검색 기능 개선
git worktree add -b feature-search ../ecommerce-search
# → Claude가 검색 알고리즘 개선 작업

# Worktree 2: 장바구니 버그 수정
git worktree add -b fix-cart ../ecommerce-cart
# → Claude가 장바구니 계산 오류 수정
```

### 터미널 1 (검색 기능)

```bash
cd ~/projects/ecommerce-search
claude -p "
현재 상품 검색이 정확도가 낮아.
검색 알고리즘을 개선해서 다음을 구현해줘:
1. 오타 허용 (퍼지 검색)
2. 동의어 처리
3. 검색 결과 정렬 개선 (관련성 기반)
"
```

### 터미널 2 (장바구니 버그)

```bash
cd ~/projects/ecommerce-cart
claude -p "
장바구니에서 할인 쿠폰 적용 시 최종 금액이
가끔 음수가 되는 버그가 있어.
src/cart/calculator.ts를 분석하고 수정해줘.
"
```

### 두 작업 완료 후 통합

```bash
# 검색 기능 PR
cd ~/projects/ecommerce-search
git push origin feature-search

# 장바구니 수정 PR
cd ~/projects/ecommerce-cart
git push origin fix-cart
```

## 실습

### 실습 1: 기본 워크트리 병렬 작업

```bash
# 테스트 저장소 준비
mkdir parallel-work-demo && cd parallel-work-demo
git init
echo "# My App" > README.md
mkdir -p src
echo 'function hello() { console.log("Hello!") }' > src/app.js
git add -A && git commit -m "초기 커밋"

# 두 워크트리 생성
git worktree add -b feature-greeting ../demo-greeting
git worktree add -b feature-farewell ../demo-farewell

# 워크트리 목록 확인
git worktree list
```

각 워크트리에서 다른 터미널로 Claude 실행:

```bash
# 터미널 1
cd ../demo-greeting
claude -p "src/app.js에 다양한 언어로 인사하는 함수들을 추가해줘 (영어, 한국어, 일본어, 스페인어)"

# 터미널 2
cd ../demo-farewell
claude -p "src/app.js에 다양한 언어로 작별 인사하는 함수들을 추가해줘 (영어, 한국어, 일본어, 스페인어)"
```

### 실습 2: 워크트리 정리

```bash
# 작업 완료 후
git worktree list

# 워크트리 제거
git worktree remove ../demo-greeting
git worktree remove ../demo-farewell

# 브랜치 정리 (병합 후)
git branch -d feature-greeting
git branch -d feature-farewell

# 확인
git worktree list
```
