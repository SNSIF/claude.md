# 9-4. 에이전트 호출 (Calling Agents)

커스텀 에이전트를 만들었다면 4가지 방법으로 호출할 수 있습니다. 각 방법은 상황에 따라 장단점이 있습니다.

---

## 4가지 호출 방식

### 1. 자동 위임 (Auto-delegation)

Claude가 사용자의 요청을 분석하여 **적합한 에이전트를 자동으로 선택**하고 작업을 위임합니다.

**작동 방식**:
```
사용자 요청 → Claude 메인 에이전트 분석
  → 각 에이전트의 description 확인
  → 요청 내용과 가장 일치하는 에이전트 선택
  → 자동으로 에이전트 실행
  → 결과를 메인 에이전트가 취합하여 응답
```

**예시**:
```
사용자: "auth 모듈 코드 리뷰해줘"
         ↓
Claude 내부: description에 "코드 리뷰"가 포함된 에이전트 탐색
         ↓
code-reviewer 에이전트 자동 선택 및 실행
         ↓
리뷰 결과를 사용자에게 전달
```

**자동 위임을 잘 받으려면**:

description을 구체적으로 작성하는 것이 핵심입니다.

```yaml
# 자동 위임이 잘 안 되는 경우 (모호한 description)
description: 개발 도움 에이전트

# 자동 위임이 잘 되는 경우 (구체적인 description)
description: >
  Python/TypeScript 코드의 보안 취약점, 성능 문제, 코드 품질을
  검토하고 심각도별로 분류된 리뷰 보고서를 작성하는 에이전트.
  "리뷰", "코드 검토", "버그 찾아줘" 등의 요청에 활용.
```

**자동 위임이 트리거되는 요청 패턴**:
```
"코드 리뷰해줘"           → code-reviewer
"테스트 작성해줘"          → test-writer
"문서화해줘"               → doc-generator
"보안 점검해줘"            → security-auditor
"성능 분석해줘"            → performance-analyzer
```

---

### 2. @mention 호출

대화 중 `@에이전트명`으로 특정 에이전트를 직접 지정하여 호출합니다.

**기본 문법**:
```
@에이전트명 [작업 내용]
```

**실제 사용 예시**:
```
@code-reviewer auth/login.ts 파일 검토해줘

@test-writer UserService 클래스에 대한 테스트를 작성해줘

@doc-generator api/routes.ts 파일의 JSDoc을 완성해줘

@security-auditor 전체 프로젝트 보안 감사를 진행해줘
```

**자동 완성 활용**:
- `@` 입력 후 `Tab` 또는 `Shift+Space`로 에이전트 목록 확인
- 에이전트 이름 일부 입력 후 자동 완성 가능

**@mention 호출의 장점**:
- 원하는 에이전트를 명확하게 지정 가능
- 자동 위임 오판 없음
- 동일 요청을 여러 에이전트에게 순차 실행 가능

**여러 에이전트에게 순차 실행**:
```
@code-reviewer src/payment.ts 리뷰해줘
→ (리뷰 완료 후)
@test-writer 방금 리뷰한 파일에 대한 테스트 작성해줘
```

### 3. `/agents` 커맨드

Claude Code의 슬래시 커맨드로 에이전트를 관리하고 모니터링합니다.

**주요 기능**:

```bash
# 현재 사용 가능한 에이전트 목록 확인
/agents

# 특정 에이전트 상세 정보 확인
/agents info code-reviewer

# 실행 중인 에이전트 상태 모니터링
/agents status

# 실행 중인 에이전트 중지
/agents stop code-reviewer
```

**`/agents` 출력 예시**:
```
사용 가능한 에이전트 (프로젝트):
  ● code-reviewer    — 코드 품질, 보안, 성능 검토 (sonnet, plan 모드)
  ● test-writer      — 단위/통합 테스트 자동 작성 (sonnet, acceptEdits 모드)
  ● doc-generator    — 코드 문서화 자동 생성 (sonnet, acceptEdits 모드)

사용 가능한 에이전트 (전역):
  ● my-helper        — 개인 맞춤 도우미 (haiku, default 모드)

현재 실행 중: 없음
```

### 4. CLI 플래그로 직접 실행

터미널에서 claude 명령어에 플래그를 추가해 에이전트를 직접 지정합니다.

```bash
# 기본 문법
claude --agent 에이전트명 "작업 내용"

# 예시
claude --agent code-reviewer "PR #42의 변경 파일들을 리뷰해줘"

claude --agent test-writer "src/utils/ 폴더의 모든 함수에 대한 테스트 작성해줘"

claude --agent security-auditor "전체 프로젝트 보안 감사 수행해줘"
```

**파이프라인 활용**:
```bash
# git diff 결과를 에이전트에게 전달
git diff HEAD~1 | claude --agent code-reviewer "이 변경사항을 리뷰해줘"

# 특정 파일 목록을 에이전트에게 전달
find . -name "*.ts" -newer last-review | \
  xargs claude --agent code-reviewer "이 파일들을 리뷰해줘"
```

**CI/CD에서 활용**:
```yaml
# GitHub Actions 예시
- name: 코드 리뷰
  run: |
    claude --agent code-reviewer \
           --permission-mode bypassPermissions \
           "PR의 변경 파일을 리뷰하고 결과를 review-report.md에 저장해줘"
```

## 에이전트 실행 결과 확인 방법

### 대화형 모드에서 실시간 확인

```bash
claude

> @code-reviewer auth/ 폴더 리뷰해줘

# 에이전트 실행 중 진행 상황 표시:
# [code-reviewer] auth/login.ts 분석 중...
# [code-reviewer] auth/middleware.ts 분석 중...
# [code-reviewer] auth/utils.ts 분석 중...
# [code-reviewer] 리뷰 완료

# 결과: (리뷰 보고서 출력)
```

### `--verbose` 플래그로 상세 로그 확인

```bash
claude --verbose

> @code-reviewer auth/ 폴더 리뷰해줘

# 상세 로그 출력:
# [Sub-agent: code-reviewer] Started
# [Sub-agent: code-reviewer] Tool call: Glob("auth/**/*.ts")
# [Sub-agent: code-reviewer] Tool result: ["auth/login.ts", "auth/middleware.ts"]
# [Sub-agent: code-reviewer] Tool call: Read("auth/login.ts")
# ...
```

## 에이전트 체이닝

A 에이전트의 결과를 B 에이전트의 입력으로 연결합니다.

### 수동 체이닝

```bash
claude

# 1단계: 코드 리뷰
> @code-reviewer src/payment.ts 리뷰해줘

# 리뷰 결과 확인 후...

# 2단계: 리뷰 결과 기반 테스트 작성
> @test-writer 위에서 code-reviewer가 발견한 문제들을 검증하는 테스트를 작성해줘

# 3단계: 문서 업데이트
> @doc-generator 수정된 payment.ts에 대한 문서를 업데이트해줘
```

### 자동 체이닝 (메인 에이전트에게 위임)

```bash
claude

> "src/payment.ts 파일을 먼저 리뷰하고, 발견된 문제들에 대한 테스트를 작성한 다음, 최종적으로 문서를 업데이트해줘"

# 메인 에이전트가 자동으로:
# 1. @code-reviewer 실행
# 2. 리뷰 결과를 @test-writer에 전달
# 3. @doc-generator 실행
```

## 여러 에이전트 병렬 실행

작업이 독립적인 경우 여러 에이전트를 동시에 실행하여 시간을 단축합니다.

```bash
claude

# 메인 에이전트에게 병렬 실행 요청
> "frontend/, backend/, tests/ 세 폴더를 동시에 분석해줘. 각 폴더를 병렬로 탐색해"

# 메인 에이전트가 자동으로:
# [병렬] Explore 에이전트 A: frontend/ 탐색
# [병렬] Explore 에이전트 B: backend/ 탐색
# [병렬] Explore 에이전트 C: tests/ 탐색
# → 세 결과를 통합하여 응답
```

**CLI에서 병렬 실행**:
```bash
# 백그라운드로 여러 에이전트 동시 실행
claude --agent code-reviewer "frontend/ 리뷰해줘" &
claude --agent code-reviewer "backend/ 리뷰해줘" &
wait

echo "두 폴더 리뷰 완료"
```

## 에이전트 디버깅 팁

### 팁 1: description 테스트

자동 위임이 의도대로 작동하는지 확인합니다.

```bash
claude

# description 테스트 — 어떤 에이전트가 선택되는지 확인
> "지금 내 요청에 가장 적합한 에이전트가 있으면 어떤 것인지 말해줘 (실행은 하지 말고)"
```

### 팁 2: maxTurns 조정

에이전트가 중간에 종료되면 maxTurns를 늘립니다.

```bash
# 에이전트가 작업 중 종료되면:
> "아까 code-reviewer가 3개 파일만 보고 끝났어. 나머지도 계속 리뷰해줘"
# 또는 에이전트 파일의 maxTurns 값을 늘리기
```

### 팁 3: 권한 모드 충돌 확인

에이전트의 `permissionMode`와 `tools` 설정이 충돌하지 않는지 확인합니다.

```yaml
# 문제: plan 모드인데 Write 도구 허용
permissionMode: plan
tools: Read, Write, Edit  # Write와 Edit은 plan 모드에서 동작 안 함

# 수정: plan 모드면 읽기 도구만
permissionMode: plan
tools: Read, Grep, Glob
```

### 팁 4: 에이전트 로그 확인

```bash
# ~/.claude/logs/ 에서 에이전트 실행 로그 확인
ls ~/.claude/logs/

# 특정 에이전트 로그 보기
cat ~/.claude/logs/code-reviewer-latest.log
```

## 실습: code-reviewer 에이전트를 4가지 방법으로 호출

### 준비: code-reviewer 에이전트 생성

```bash
mkdir -p .claude/agents
cat > .claude/agents/code-reviewer.md << 'EOF'
---
name: code-reviewer
description: >
  코드의 품질, 보안 취약점, 성능 문제를 검토하고 심각도별 보고서를 작성하는 에이전트.
  "리뷰", "검토", "코드 분석", "버그 찾기" 요청에 자동으로 선택됨.
model: sonnet
tools: Read, Grep, Glob
permissionMode: plan
maxTurns: 20
---

당신은 시니어 개발자입니다.
코드를 검토하고 발견된 문제를 심각도(Critical/Major/Minor)와 함께 보고하세요.
EOF
```

### 방법 1: 자동 위임으로 호출

```bash
claude
> "src/ 폴더의 코드를 리뷰해줘"
# → code-reviewer가 자동 선택되어 실행되는지 확인
```

### 방법 2: @mention으로 호출

```bash
claude
> "@code-reviewer src/auth.ts 파일을 검토해줘"
# → 직접 지정하여 실행
```

### 방법 3: /agents 커맨드로 확인 후 호출

```bash
claude
> /agents
# 에이전트 목록에서 code-reviewer 확인

> /agents info code-reviewer
# 에이전트 상세 정보 확인

> "@code-reviewer src/ 전체 리뷰해줘"
# 목록 확인 후 호출
```

### 방법 4: CLI 플래그로 직접 실행

```bash
# 터미널에서 직접 실행
claude --agent code-reviewer "src/auth.ts 파일을 리뷰하고 결과를 출력해줘"

# CI 환경에서 실행
claude --agent code-reviewer \
       --permission-mode bypassPermissions \
       "변경된 파일들을 리뷰하고 review.md 파일에 결과를 저장해줘"
```

### 결과 비교

4가지 방법 모두 동일한 code-reviewer 에이전트가 실행되지만:
- **자동 위임**: 편리하지만 description이 불명확하면 다른 에이전트 선택될 수 있음
- **@mention**: 가장 명확하고 확실한 방법
- **/agents**: 에이전트 목록을 먼저 확인하고 싶을 때 유용
- **CLI 플래그**: 스크립트 자동화에 최적
