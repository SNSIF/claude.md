# 8-2. MCP 서버 설정

## .mcp.json 파일

MCP 서버는 `.mcp.json` 파일에 정의한다. 이 파일은 어떤 MCP 서버를 어떻게 실행할지 기술한다.

### 기본 위치

```
프로젝트 루트/
├── .mcp.json          ← 여기에 작성
├── .claude/
│   └── settings.json
└── src/
```

### 기본 구조

```json
{
  "mcpServers": {
    "서버명": {
      "type": "stdio",
      "command": "npx",
      "args": ["@서버패키지/server"],
      "env": {
        "API_KEY": "$MY_API_KEY"
      }
    }
  }
}
```

각 필드 설명:

| 필드 | 설명 |
|------|------|
| `서버명` | Claude에서 이 서버를 부르는 이름 (자유롭게 지정) |
| `type` | 연결 방식 (`stdio`, `http`, `sse`) |
| `command` | 서버 실행 명령 |
| `args` | 명령 인수 배열 |
| `env` | 서버에 전달할 환경 변수 |

---

## 서버 유형별 설정

### stdio — 로컬 프로세스 (가장 일반적)

로컬에서 프로세스를 직접 실행해 표준 입출력으로 통신한다.

```json
{
  "mcpServers": {
    "filesystem": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/home/user/docs"]
    }
  }
}
```

**`-y` 플래그**: 패키지를 자동으로 설치하고 실행한다. 처음 실행 시 다운로드가 발생한다.

여러 경로를 허용하려면:

```json
{
  "mcpServers": {
    "filesystem": {
      "type": "stdio",
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/home/user/docs",
        "/home/user/projects",
        "/tmp"
      ]
    }
  }
}
```

---

### http — HTTP 엔드포인트

이미 실행 중인 HTTP 서버에 연결한다.

```json
{
  "mcpServers": {
    "my-api": {
      "type": "http",
      "url": "http://localhost:3000/mcp",
      "headers": {
        "Authorization": "Bearer $MY_API_TOKEN"
      }
    }
  }
}
```

**활용 상황:**
- 팀 내부 MCP 서버를 별도로 운영할 때
- 이미 실행 중인 서비스에 MCP 엔드포인트를 추가했을 때
- 원격 서버에 연결할 때

---

### sse — Server-Sent Events

SSE 방식으로 연결하는 서버. 지속적인 스트림 연결이 필요할 때 사용한다.

```json
{
  "mcpServers": {
    "stream-server": {
      "type": "sse",
      "url": "http://localhost:3000/sse"
    }
  }
}
```

---

## 환경 변수 처리

API 키와 같은 민감한 정보는 직접 `.mcp.json`에 넣지 말고 환경 변수 참조를 사용한다.

```json
{
  "mcpServers": {
    "github": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "$GITHUB_TOKEN"
      }
    }
  }
}
```

`$GITHUB_TOKEN`은 셸 환경 변수를 참조한다. 실제 값은 `.bashrc` 또는 `.zshrc`에 설정한다:

```bash
# ~/.bashrc 또는 ~/.zshrc
export GITHUB_TOKEN="ghp_xxxxxxxxxxxxxxxxxxxx"
export DATABASE_URL="postgresql://user:pass@localhost/mydb"
```

## 여러 서버 동시 설정

```json
{
  "mcpServers": {
    "filesystem": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/home/user/projects"]
    },
    "github": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "$GITHUB_TOKEN"
      }
    },
    "postgres": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres"],
      "env": {
        "DATABASE_URL": "$DATABASE_URL"
      }
    }
  }
}
```

## 서버 추가 후 확인

### Claude Code에서 확인

```
/mcp
```

`/mcp` 명령어를 입력하면 현재 연결된 MCP 서버 목록과 상태가 표시된다.

```
MCP 서버 상태:
✅ filesystem — 연결됨 (도구 5개)
✅ github — 연결됨 (도구 23개)
❌ postgres — 연결 실패
```

### 도구 목록 확인

연결된 서버의 도구 목록은 다음 명령으로 확인한다:

```
/mcp filesystem
```

## 연결 문제 트러블슈팅

### 문제 1: 서버가 연결되지 않는다

```bash
# npx가 정상 작동하는지 확인
npx -y @modelcontextprotocol/server-filesystem /tmp

# 오류 발생 시 Node.js 버전 확인 (18 이상 필요)
node --version
```

### 문제 2: 환경 변수가 전달되지 않는다

```bash
# 환경 변수가 설정됐는지 확인
echo $GITHUB_TOKEN
echo $DATABASE_URL

# 설정 안 됐으면 직접 추가
export GITHUB_TOKEN="your-token-here"
```

### 문제 3: 권한 오류

```bash
# 파일시스템 서버에서 접근 거부 시
# args에 허용할 경로가 올바른지 확인
ls -la /path/to/dir
```

### 문제 4: 포트 충돌 (http/sse)

```bash
# 포트 사용 여부 확인
lsof -i :3000
# 또는
netstat -an | grep 3000
```

### 로그 확인

Claude Code 실행 시 `--debug` 플래그를 사용하면 MCP 통신 로그를 볼 수 있다:

```bash
claude --debug
```

## 실습: Filesystem MCP 서버 연결

### 1단계 — 테스트 디렉토리 준비

```bash
mkdir -p ~/mcp-test
cat > ~/mcp-test/project-info.md << 'EOF'
# 테스트 프로젝트

## 개요
MCP 테스트용 프로젝트입니다.

## 기술 스택
- Backend: FastAPI
- Database: PostgreSQL
- Frontend: React
EOF
```

### 2단계 — .mcp.json 작성

프로젝트 루트에 `.mcp.json` 파일 생성:

```json
{
  "mcpServers": {
    "docs": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/Users/사용자명/mcp-test"]
    }
  }
}
```

경로는 본인 환경에 맞게 수정한다.

### 3단계 — 연결 확인

Claude Code를 재시작하거나 `/mcp reload` 실행 후:

```
/mcp
```

### 4단계 — 동작 테스트

```
Claude에게 요청: "~/mcp-test 디렉토리의 project-info.md 파일 내용 보여줘"
```

Claude가 MCP를 통해 파일을 읽어 응답한다.

---

## 핵심 요약

- `.mcp.json`은 프로젝트 루트에 위치
- `type`은 `stdio` (로컬), `http` (HTTP), `sse` (스트림) 3가지
- API 키는 `$ENV_VAR` 형식으로 환경 변수 참조
- `/mcp` 명령으로 연결 상태 확인
- 처음 실행 시 `npx -y`가 패키지를 자동 설치
