# 8-3. MCP 설정 범위

## 4가지 설정 범위

MCP 서버 설정은 적용 범위에 따라 4가지로 구분된다.

| 범위 | 위치 | 적용 대상 | 버전 관리 |
|------|------|-----------|-----------|
| **Local** | `./.mcp.json` | 현재 디렉토리만 | gitignore 권장 |
| **Project** | `./.claude/settings.json` 내 | 프로젝트 전체 | 포함 가능 |
| **User** | `~/.claude/settings.json` 내 | 모든 프로젝트 | 개인 설정 |
| **Managed** | 조직 정책 배포 | 팀/조직 전체 | IT 관리 |

---

## Local — 현재 디렉토리

**위치**: 프로젝트 루트의 `.mcp.json`

현재 디렉토리에서만 적용된다. API 키 등 민감한 정보가 포함될 때 사용한다.

```json
{
  "mcpServers": {
    "github": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "$GITHUB_TOKEN"
      }
    }
  }
}
```

**.gitignore에 추가:**

```bash
# .gitignore
.mcp.json
```

API 키가 포함될 수 있으므로 git에 포함하지 않는 것이 안전하다.

---

## Project — 버전 관리 포함

**위치**: `.claude/settings.json`의 `mcpServers` 섹션

팀원과 공유해야 하는 서버 설정을 여기에 작성한다. git으로 관리되므로 팀 전체에 일관된 설정을 제공할 수 있다.

```json
{
  "mcpServers": {
    "filesystem": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "./docs", "./data"]
    }
  }
}
```

**이 범위에 적합한 것:**
- API 키가 없는 서버 (filesystem 등)
- 팀 전체가 동일하게 사용해야 하는 서버
- 프로젝트 문서화를 위해 설정을 공유해야 할 때

---

## User — 모든 프로젝트

**위치**: `~/.claude/settings.json`의 `mcpServers` 섹션

어떤 프로젝트를 열든 항상 사용할 수 있는 서버를 설정한다.

```json
{
  "mcpServers": {
    "personal-notes": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/home/user/notes"]
    },
    "brave-search": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-brave-search"],
      "env": {
        "BRAVE_API_KEY": "$BRAVE_API_KEY"
      }
    }
  }
}
```

**이 범위에 적합한 것:**
- 개인 노트/문서 접근
- 웹 검색 등 범용 도구
- 개인 API 키가 필요한 서버

---

## Managed — 조직 배포

조직 IT 부서가 중앙에서 관리하고 배포하는 설정이다.

**특징:**
- 개별 설정보다 우선순위가 높을 수 있음
- 보안 정책 적용 가능
- 사용자가 임의로 변경 불가
- Claude Code Enterprise 환경에서 주로 사용

---

## 우선순위 및 충돌 해결

동일한 이름의 서버가 여러 범위에 정의된 경우 우선순위가 높은 설정이 사용된다.

```
우선순위: Managed > User > Project > Local
```

실제로는 **범위가 좁을수록 우선순위가 높다**.

```
Local (.mcp.json) > Project (.claude/settings.json) > User (~/.claude/settings.json)
```

### 예시

User 범위에 `github` 서버가 있고, Project 범위에도 `github` 서버가 있으면 **Project 설정이 우선** 적용된다.

---

## 언제 어느 범위를 써야 하나

### API 키가 포함된 서버 → Local

```json
// .mcp.json (gitignore에 추가)
{
  "mcpServers": {
    "openai": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "mcp-server-openai"],
      "env": {
        "OPENAI_API_KEY": "$OPENAI_API_KEY"
      }
    }
  }
}
```

이유: `.mcp.json`을 gitignore하면 키가 저장소에 올라가지 않는다.

### 팀이 공유해야 하는 서버 → Project

```json
// .claude/settings.json
{
  "mcpServers": {
    "project-docs": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "./docs"]
    }
  }
}
```

이유: git으로 공유하면 팀 전체가 동일한 서버에 접근한다.

### 개인 범용 도구 → User

```json
// ~/.claude/settings.json
{
  "mcpServers": {
    "my-notes": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/Users/나/Documents/Notes"]
    }
  }
}
```

이유: 어느 프로젝트에서나 개인 노트에 접근할 수 있다.

## settings.json에서 mcpServers 설정

`.mcp.json`뿐만 아니라 `settings.json`에서도 MCP 서버를 직접 설정할 수 있다.

### 프로젝트 범위 (`.claude/settings.json`)

```json
{
  "mcpServers": {
    "filesystem": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "./"],
      "disabled": false
    }
  },
  "hooks": {
    "PostToolUse": []
  }
}
```

### 사용자 범위 (`~/.claude/settings.json`)

```json
{
  "mcpServers": {
    "search": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-brave-search"],
      "env": {
        "BRAVE_API_KEY": "$BRAVE_API_KEY"
      }
    }
  }
}
```

**`.mcp.json` vs `settings.json`:**

| 비교 | `.mcp.json` | `settings.json` |
|------|-------------|-----------------|
| 목적 | MCP 전용 | 일반 설정 + MCP |
| 위치 | 프로젝트 루트 | `.claude/` 또는 `~/.claude/` |
| 직관성 | MCP 설정 전용이라 명확 | 다른 설정과 혼재 |
| gitignore | 쉽게 제외 가능 | 부분 제외 불가 |

민감한 키를 포함하지 않는다면 어느 쪽이든 무방하다.

---

## 실습: 범위별 설정 직접 구성

### 목표

Local, Project, User 3가지 범위에 각각 서버를 설정하고 동작 차이를 확인한다.

### 1단계 — User 범위 설정

```bash
mkdir -p ~/.claude
```

`~/.claude/settings.json`:
```json
{
  "mcpServers": {
    "user-filesystem": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/tmp"]
    }
  }
}
```

### 2단계 — Project 범위 설정

`.claude/settings.json`:
```json
{
  "mcpServers": {
    "project-docs": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "./docs"]
    }
  }
}
```

### 3단계 — Local 범위 설정

`.mcp.json` (gitignore에 추가):
```json
{
  "mcpServers": {
    "local-private": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "./private"]
    }
  }
}
```

`.gitignore`:
```
.mcp.json
```

### 4단계 — 확인

```
/mcp
```

3개의 서버(`user-filesystem`, `project-docs`, `local-private`)가 모두 나타나면 성공이다.

---

## 핵심 요약

- **Local** (`.mcp.json`): API 키 포함 → gitignore 필수
- **Project** (`.claude/settings.json`): 팀 공유 → git에 포함
- **User** (`~/.claude/settings.json`): 개인 범용 도구
- 우선순위: Local > Project > User > Managed
- `settings.json`의 `mcpServers`와 `.mcp.json`은 동일한 역할
