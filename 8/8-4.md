# 8-4. 실전 MCP 활용

## 개요

이 파일은 실제 개발에서 자주 사용하는 MCP 서버 연동 예제 모음이다. 각 예제는 `.mcp.json`에 바로 추가할 수 있다.

---

## 1. GitHub MCP

GitHub 저장소와 직접 상호작용한다. 이슈 관리, PR 조회, 코드 검색 등을 Claude가 직접 수행한다.

### 설정

```json
{
  "mcpServers": {
    "github": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "$GITHUB_TOKEN"
      }
    }
  }
}
```

### GitHub Token 발급

```bash
# GitHub → Settings → Developer settings → Personal access tokens → Fine-grained tokens
# 필요 권한: Contents(Read), Issues(Read/Write), Pull requests(Read/Write)

export GITHUB_TOKEN="ghp_xxxxxxxxxxxxxxxxxxxx"
```

### 주요 활용 예제

**이슈 조회:**
```
"내 저장소의 열린 이슈 목록 보여줘"
"priority: high 라벨이 붙은 이슈들 요약해줘"
```

**PR 관리:**
```
"현재 리뷰 대기 중인 PR 목록 보여줘"
"PR #42의 변경사항 요약해줘"
```

**코드 검색:**
```
"이 저장소에서 TODO 주석이 있는 파일 찾아줘"
"useEffect가 사용된 컴포넌트 목록 뽑아줘"
```

**이슈 생성:**
```
"로그인 페이지 로딩 속도 문제에 대한 이슈 만들어줘. 라벨은 bug와 performance 추가해줘"
```

## 2. Filesystem MCP

Claude Code의 기본 파일 접근 범위 밖에 있는 디렉토리를 읽고 쓸 수 있게 한다.

### 설정

```json
{
  "mcpServers": {
    "filesystem": {
      "type": "stdio",
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/home/user/documents",
        "/home/user/notes",
        "/tmp"
      ]
    }
  }
}
```

여러 경로를 args에 나열하면 모두 접근 가능해진다.

### 주요 활용 예제

**외부 문서 참조:**
```
"~/documents/API-spec.md 파일 읽어서 현재 구현과 비교해줘"
```

**노트 활용:**
```
"~/notes/2026-03 폴더의 내용을 바탕으로 이번 달 작업 요약해줘"
```

**공유 디렉토리 접근:**
```
"/shared/team-docs/coding-standards.md 규칙을 현재 코드에 적용해줘"
```

## 3. PostgreSQL MCP

데이터베이스 스키마를 파악하고 SQL 쿼리를 실행한다.

### 설정

```json
{
  "mcpServers": {
    "postgres": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres"],
      "env": {
        "DATABASE_URL": "$DATABASE_URL"
      }
    }
  }
}
```

### 환경 변수 설정

```bash
export DATABASE_URL="postgresql://username:password@localhost:5432/mydb"
```

### 주요 활용 예제

**스키마 파악:**
```
"현재 DB의 테이블 목록과 각 테이블의 컬럼 구조 알려줘"
```

**쿼리 실행:**
```
"최근 7일간 가입한 사용자 수를 일별로 보여줘"
```

**성능 분석:**
```
"슬로우 쿼리가 발생할 수 있는 테이블 관계 파악해줘"
```

**마이그레이션 도움:**
```
"users 테이블에 last_login_at 컬럼을 추가하는 마이그레이션 SQL 작성해줘"
```

> **주의**: 프로덕션 DB에는 읽기 전용 사용자로 연결하는 것을 강력 권장한다.

## 4. Slack MCP

Slack 워크스페이스와 직접 상호작용한다.

### 설정

```json
{
  "mcpServers": {
    "slack": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-slack"],
      "env": {
        "SLACK_BOT_TOKEN": "$SLACK_BOT_TOKEN",
        "SLACK_TEAM_ID": "$SLACK_TEAM_ID"
      }
    }
  }
}
```

### Slack Bot Token 발급

1. https://api.slack.com/apps 에서 앱 생성
2. OAuth & Permissions에서 Bot Token Scopes 추가:
   - `channels:read`, `chat:write`, `channels:history`
3. 워크스페이스에 앱 설치 후 Bot User OAuth Token 복사

```bash
export SLACK_BOT_TOKEN="xoxb-xxxxxxxxxxxx"
export SLACK_TEAM_ID="T0XXXXXXXX"
```

### 주요 활용 예제

**메시지 전송:**
```
"#backend-team 채널에 배포 완료 메시지 보내줘"
```

**채널 히스토리 분석:**
```
"#general 채널에서 오늘 중요한 공지사항 있었어?"
```

## 5. Notion MCP

Notion 페이지와 데이터베이스를 읽고 쓴다.

### 설정

```json
{
  "mcpServers": {
    "notion": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@notionhq/notion-mcp-server"],
      "env": {
        "NOTION_API_KEY": "$NOTION_API_KEY"
      }
    }
  }
}
```

### Notion API 키 발급

1. https://www.notion.so/my-integrations 에서 통합 생성
2. 생성된 "Internal Integration Secret" 복사
3. 사용할 페이지/DB에 통합 연결 (페이지 우측 상단 → Connections)

```bash
export NOTION_API_KEY="secret_xxxxxxxxxxxxxxxxxx"
```

### 주요 활용 예제

**문서 조회:**
```
"Notion의 프로젝트 로드맵 페이지 내용 요약해줘"
```

**데이터베이스 조회:**
```
"Task 데이터베이스에서 In Progress 상태인 항목 목록 보여줘"
```

**페이지 생성:**
```
"오늘 작업한 내용을 Daily Log 데이터베이스에 새 항목으로 추가해줘"
```

## MCP 도구 권한 설정

특정 MCP 서버의 특정 도구만 허용하거나 금지할 수 있다.

### allowedTools로 허용

`.claude/settings.json`:

```json
{
  "allowedTools": [
    "mcp__github__list_issues",
    "mcp__github__get_issue",
    "mcp__filesystem__read_file"
  ]
}
```

### 패턴으로 허용

```json
{
  "allowedTools": [
    "mcp__github__*",
    "mcp__filesystem__read_*"
  ]
}
```

`mcp__서버명__도구명` 형식으로 MCP 도구를 지정한다. `*` 와일드카드를 사용해 패턴 매칭이 가능하다.

### 특정 도구 금지

```json
{
  "disallowedTools": [
    "mcp__github__delete_*",
    "mcp__postgres__execute_query"
  ]
}
```

**실용 예시 — GitHub 읽기 전용 허용:**

```json
{
  "allowedTools": [
    "mcp__github__list_issues",
    "mcp__github__get_issue",
    "mcp__github__list_pull_requests",
    "mcp__github__get_pull_request",
    "mcp__github__search_code"
  ],
  "disallowedTools": [
    "mcp__github__create_issue",
    "mcp__github__update_issue",
    "mcp__github__merge_pull_request"
  ]
}
```

## 통합 설정 예제

실제 프로젝트에서 사용하는 완전한 `.mcp.json` 예제다.

```json
{
  "mcpServers": {
    "github": {

      "command": "npx",

      "type": "stdio",

      "args": ["-y", "@modelcontextprotocol/server-postgres"],
      "env": {
        "DATABASE_URL": "$DEV_DATABASE_URL"
      }
    }
  }
}
```

## 핵심 요약

- **GitHub**: 이슈/PR 관리, 코드 검색에 필수
- **Filesystem**: 프로젝트 외부 파일 접근
- **PostgreSQL**: DB 스키마 파악, 쿼리 실행 (프로덕션은 읽기 전용 권장)
- **Slack/Notion**: 팀 협업 도구 연동
- `mcp__서버명__*` 패턴으로 도구 권한 세밀하게 제어
- 레지스트리에서 더 많은 서버를 찾을 수 있다
