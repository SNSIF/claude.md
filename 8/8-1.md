# 8-1. MCP 개념

## MCP란 무엇인가

**MCP(Model Context Protocol)**는 Claude와 외부 서비스를 연결하는 **표준 오픈 프로토콜**이다.

Anthropic이 개발하고 커뮤니티가 서버를 제공하는 방식으로 운영되며, USB-C 표준처럼 어떤 외부 도구든 동일한 방식으로 Claude에 연결할 수 있게 한다.

### 개발 배경

Claude는 기본적으로 대화만 가능하다. 하지만 실제 개발 작업에는 데이터베이스 조회, GitHub PR 관리, 외부 파일 접근 등이 필요하다. MCP는 이런 외부 능력을 표준화된 방법으로 Claude에게 제공한다.

---

## MCP 없이 vs MCP 있을 때

### MCP 없을 때

```
사용자: "이 프로젝트의 GitHub 이슈 목록 보여줘"

Claude: "죄송합니다. 저는 GitHub에 직접 접근할 수 없습니다.
         이슈 목록을 복사해서 붙여넣어 주시면 분석해드리겠습니다."
```

매번 사용자가 데이터를 복사해야 한다. 비효율적이고 최신 데이터를 보장할 수 없다.

### MCP 있을 때

```
사용자: "이 프로젝트의 GitHub 이슈 목록 보여줘"

Claude: [GitHub MCP 서버 호출]
        → 실시간 이슈 목록 조회
        → "현재 열린 이슈 12개입니다. 우선순위 높은 순으로..."
```

Claude가 직접 GitHub에 접근해 최신 데이터를 가져온다.

---

## MCP로 할 수 있는 것

### 데이터베이스 쿼리
```
사용자: "users 테이블에서 최근 가입한 사용자 10명 보여줘"
Claude: [PostgreSQL MCP 서버 → SELECT 쿼리 실행]
        → 실시간 데이터 반환
```

### GitHub API 호출
```
사용자: "버그 수정 완료했어. PR 만들어줘"
Claude: [GitHub MCP 서버 → PR 생성 API 호출]
        → PR 번호와 링크 반환
```

### 파일 시스템 외부 접근
```
사용자: "/home/user/documents의 프로젝트 문서 요약해줘"
Claude: [Filesystem MCP 서버 → 파일 읽기]
        → 문서 내용 분석 및 요약
```

### 외부 서비스 제어
```
사용자: "팀 Slack 채널에 배포 완료 메시지 보내줘"
Claude: [Slack MCP 서버 → 메시지 전송 API]
        → 전송 완료
```

## MCP 아키텍처

```
┌─────────────────────────────────────────────────────┐
│                    Claude Code                       │
│              (MCP 클라이언트)                         │
└─────────────────┬───────────────────────────────────┘
                  │  JSON-RPC (요청/응답)
        ┌─────────┴──────────────────────────────┐
        │                                        │
        ▼                                        ▼
┌───────────────┐                    ┌───────────────────┐
│  GitHub MCP   │                    │  PostgreSQL MCP   │
│  서버         │                    │  서버             │
│  (stdio)      │                    │  (stdio)          │
└───────┬───────┘                    └────────┬──────────┘
        │                                     │
        ▼                                     ▼
   GitHub API                          PostgreSQL DB
```

### 구성 요소

**Claude Code (클라이언트)**
- MCP 프로토콜을 이해하는 클라이언트
- 도구 목록을 서버로부터 받아 Claude에게 제공
- 사용자 요청에 맞는 서버 도구를 호출

**MCP 서버**
- 특정 외부 서비스와의 통신을 담당
- 도구(Tools), 리소스(Resources), 프롬프트(Prompts) 제공
- 누구든 직접 만들 수 있음

**프로토콜**
- **JSON-RPC 2.0** 기반
- 통신 방식: stdio(표준 입출력), HTTP, SSE(Server-Sent Events)

---

## MCP가 제공하는 3가지 기능

### Tools (도구)

Claude가 **실행할 수 있는 함수**. 액션 수행에 사용된다.

```
예시:
- create_issue(title, body, labels) → GitHub 이슈 생성
- run_query(sql) → DB 쿼리 실행
- send_message(channel, text) → Slack 메시지 전송
```

### Resources (리소스)

Claude가 **읽을 수 있는 데이터**. 컨텍스트 제공에 사용된다.

```
예시:
- github://repos/{owner}/{repo}/README → 저장소 README
- postgres://tables → 테이블 목록
- file:///path/to/document → 파일 내용
```

### Prompts (프롬프트)

서버가 제공하는 **재사용 가능한 프롬프트 템플릿**.

```
예시:
- review_pr → PR 코드 리뷰 템플릿
- analyze_query → SQL 쿼리 분석 템플릿
```

### 첫 MCP 서버 연결 준비

다음 파일에서 실제 서버 설정 방법을 배운다. 테스트용으로 `filesystem` 서버를 먼저 설정해볼 예정이다. 이 서버는 API 키가 필요 없어 즉시 사용할 수 있다.

```bash
# 테스트 디렉토리 생성
mkdir -p ~/mcp-test-docs
echo "# 테스트 문서\n이것은 MCP 테스트용 파일입니다." > ~/mcp-test-docs/readme.md
```

## 핵심 요약

- MCP는 Claude와 외부 서비스를 연결하는 **표준 프로토콜**이다
- Anthropic 개발, 커뮤니티가 서버 제공
- JSON-RPC over stdio/http 방식으로 통신
- **Tools** (실행), **Resources** (데이터), **Prompts** (템플릿) 3가지 기능 제공
- 수십 개의 공식/커뮤니티 서버가 이미 존재한다
