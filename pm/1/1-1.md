# 1. 계약서 → 기능정의서

## 흐름

```
계약서 / 기획서 (PDF, 문서)
        ↓
  Claude가 문서 읽기
        ↓
  기능 목록 추출
        ↓
  기능정의서 작성
```

---

## 문서 읽히는 방법

### VSCode에서 @-mention으로 첨부

채팅 입력창에서 `@파일명`으로 파일을 직접 첨부합니다.
PDF, Word, 텍스트 문서 모두 가능합니다.

```
@contract.pdf 이 계약서에서 개발이 필요한 기능 목록을 뽑아줘
```

### 여러 문서 동시에

```
@requirements.pdf @meeting-notes.md
두 문서를 합쳐서 기능 목록을 정리해줘. 중복은 하나로 합치고
```

---

## 프롬프트 패턴

### Step 1 — 기능 목록 먼저 추출

전체를 한 번에 기능정의서로 만들면 누락이 생깁니다.
먼저 기능 목록만 뽑아서 확인하는 것이 효과적입니다.

```
@contract.pdf
이 문서에서 개발이 필요한 기능을 전부 목록으로 뽑아줘.
기능명과 한 줄 설명만. 빠진 게 없도록 꼼꼼하게.
```

### Step 2 — 목록 검토 후 기능정의서 작성

목록을 확인하고 빠진 기능을 추가한 뒤 진행합니다.

```
위 목록에서 [기능명] 의 기능정의서를 작성해줘.

아래 항목을 포함해줘:
- 기능 개요 (목적, 대상 사용자)
- 기능 상세 (사용자 동작 흐름)
- 필요한 데이터 (입력값, 저장값)
- 엣지 케이스 (예외 상황)
- 완료 기준
- 제외 범위 (이번 버전에서 안 할 것)
```

### 전체 한 번에 (문서가 단순할 때)

```
@contract.pdf
이 문서의 모든 기능에 대해 기능정의서를 작성해줘.
기능별로 섹션을 나눠서, 각각 개요 / 상세 / 데이터 / 엣지케이스 / 완료기준을 포함해줘.
```

## 기능정의서 출력 형식

Claude에게 아래 형식으로 달라고 지정할 수 있습니다.

```markdown
## [기능명]

### 개요
- 목적:
- 대상 사용자:
- 우선순위: P0 / P1 / P2

### 기능 상세
1. 사용자가 ~하면
2. 시스템이 ~한다
3. 결과로 ~가 표시된다

### 필요한 데이터
| 항목 | 설명 | 필수 |
|------|------|------|
| | | |

### 엣지 케이스
| 상황 | 처리 방식 |
|------|-----------|
| | |

### 완료 기준
- [ ]

### 제외 범위
-
```

## 실전 팁

**"빠진 게 없도록"을 명시하세요.**
계약서에는 당연한 듯 쓰여 있는 기능이 있습니다.
Claude에게 "빠진 기능이 있으면 추가로 알려줘" 라고 요청하면
문서에 명시는 안 됐지만 필요한 기능(로그인, 권한 관리 등)을 제안해줍니다.

```
위 기능 목록 외에 이 서비스를 구현하려면 필요한데
문서에 빠진 기능이 있으면 추가로 알려줘 (예: 인증, 관리자 페이지 등)
```

**우선순위를 함께 요청하세요.**

```
각 기능에 P0(필수) / P1(중요) / P2(있으면 좋음) 우선순위도 붙여줘.
MVP 기준으로 판단해줘.
```
