# 2. 기능정의서 → 페이지 분해

## 흐름

```
기능정의서
     ↓
  페이지 목록 도출
     ↓
  페이지별 기능 배치
     ↓
  각 페이지 상세 정의
```

---

## Step 1 — 페이지 목록 도출

기능정의서를 Claude에게 주고 필요한 화면 목록을 먼저 뽑습니다.

```
위 기능정의서를 바탕으로 필요한 페이지 목록을 뽑아줘.
페이지명, URL 경로, 한 줄 설명만. 사용자 유형별로 나눠줘.
```

**출력 예시:**

```
## 일반 사용자
- /login              로그인
- /dashboard          대시보드 (메인)
- /schedule           일정 목록
- /schedule/new       일정 등록
- /schedule/:id       일정 상세

## 관리자
- /admin              관리자 홈
- /admin/users        회원 관리
- /admin/schedule     전체 일정 관리
```

---

## Step 2 — 페이지별 기능 배치

페이지 목록을 확인한 뒤, 각 페이지에 어떤 기능이 들어가는지 정의합니다.

```
위 페이지 목록을 기준으로, 각 페이지에 들어갈 기능을 정리해줘.

각 페이지마다:
- 이 페이지의 목적
- 주요 기능 목록
- 필요한 데이터 (API에서 뭘 가져와야 하나)
- 사용자 동작 (이 페이지에서 뭘 할 수 있나)
```

---

## 페이지 정의 출력 형식

```markdown
## [페이지명] — [URL]

### 목적
[이 페이지가 존재하는 이유]

### 주요 기능
- [기능 1]
- [기능 2]

### 사용자 동작
- [할 수 있는 것 1]
- [할 수 있는 것 2]

### 필요한 데이터
- [데이터 1]: [설명]
- [데이터 2]: [설명]

### 진입 경로
- [어디서 이 페이지로 오는지]

### 이탈 경로
- [이 페이지에서 어디로 갈 수 있는지]
```

---

## Step 3 — 전체 페이지맵 정리

개별 페이지 정의가 끝나면 전체를 한눈에 보는 페이지맵으로 정리합니다.

```
위 내용을 바탕으로 전체 페이지맵을 표 형식으로 정리해줘.
페이지명 / URL / 접근 권한 / 주요 기능 / 연결 페이지 컬럼으로.
```

**출력 예시:**

| 페이지명 | URL | 권한 | 주요 기능 | 연결 페이지 |
|----------|-----|------|-----------|-------------|
| 로그인 | /login | 비로그인 | 이메일 로그인, 소셜 로그인 | 대시보드 |
| 대시보드 | /dashboard | 로그인 | 오늘 일정 요약, 알림 | 일정 목록, 일정 등록 |
| 일정 목록 | /schedule | 로그인 | 일정 조회, 필터, 검색 | 일정 상세, 일정 등록 |

---

## 실전 팁

**한 번에 다 뽑지 말고 단계를 나누세요.**

```
// 1단계: 목록만
페이지 목록 뽑아줘

// 2단계: 확인 후 상세 작업
/dashboard 페이지 상세 정의해줘

// 3단계: 전체 정리
전체 페이지맵 표로 정리해줘
```

한 번에 너무 많이 요청하면 얕게 나옵니다.
페이지별로 하나씩 상세화하는 게 훨씬 정밀한 결과를 줍니다.
