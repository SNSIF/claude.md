# Deepdive — 계약서 스크린샷 → 전체 기획 명세 한 번에

## 사용법

계약서나 기획서 스크린샷을 VSCode 채팅에 붙여넣거나 @파일로 첨부하고
아래 프롬프트를 그대로 사용합니다.

---

## 완전체 프롬프트

```
첨부한 문서를 분석해서 아래 순서대로 전체 기획 명세를 작성해줘.
단계별로 완료되면 다음 단계로 넘어가고, 각 단계 시작 전에 번호와 제목을 명시해줘.

---

## 1단계: 서비스 파악
- 서비스 이름 및 한 줄 설명
- 주요 사용자 유형 (예: 일반 사용자 / 관리자 / 게스트)
- 핵심 목적 (이 서비스가 해결하는 문제)

---

## 2단계: 기능 목록 추출
문서에 명시된 기능을 전부 목록으로 뽑아줘.
문서에 명시되진 않았지만 서비스 구현에 필요한 기능(인증, 권한, 알림 등)도 함께 추가해줘.
각 기능에 우선순위를 붙여줘: P0(MVP 필수) / P1(중요) / P2(있으면 좋음)

출력 형식:
- [기능명] — [한 줄 설명] / [P0|P1|P2]

---

## 3단계: 기능정의서
2단계의 P0 기능을 대상으로 각 기능의 정의서를 작성해줘.
P1/P2는 기능명과 한 줄 설명만.

P0 기능 각각에 대해 아래 항목을 작성해줘:
- 목적 및 대상 사용자
- 기능 상세 (사용자 동작 흐름 1~5단계)
- 필요한 데이터 (입력값, 저장값)
- 엣지 케이스 (예외 상황과 처리 방식)
- 완료 기준
- 제외 범위 (이번 버전에서 안 할 것)

---

## 4단계: 페이지 목록
전체 기능을 구현하는 데 필요한 페이지를 사용자 유형별로 정리해줘.

출력 형식:
[사용자 유형]
- [페이지명] — [URL] — [한 줄 설명]

---

## 5단계: 페이지별 기능 정의
4단계의 각 페이지에 대해 아래 항목을 작성해줘:
- 목적
- 주요 기능 목록
- 사용자가 이 페이지에서 할 수 있는 것
- 필요한 데이터
- 진입/이탈 경로

---

## 6단계: 전체 페이지맵
아래 컬럼으로 표 하나로 정리해줘.
페이지명 | URL | 접근 권한 | 주요 기능 | 연결 페이지
```

---

## 스킬로 등록하기

`.claude/skills/spec-from-contract.md` 에 등록하면 `/spec-from-contract` 으로 호출 가능합니다.

```markdown
# spec-from-contract

첨부된 계약서 또는 기획서 문서를 분석해서 전체 기획 명세를 작성합니다.

아래 순서로 진행합니다. 각 단계 시작 전에 번호와 제목을 명시합니다.

1단계: 서비스 파악
- 서비스 이름, 한 줄 설명, 사용자 유형, 핵심 목적

2단계: 기능 목록 추출
- 문서에 명시된 기능 전부 + 빠진 기능(인증/권한 등) 추가
- 각 기능에 P0(MVP 필수) / P1(중요) / P2(있으면 좋음) 우선순위 표시

3단계: 기능정의서
- P0 기능: 목적, 동작 흐름, 필요한 데이터, 엣지케이스, 완료기준, 제외범위
- P1/P2 기능: 기능명 + 한 줄 설명만

4단계: 페이지 목록
- 사용자 유형별로 페이지명 / URL / 한 줄 설명

5단계: 페이지별 기능 정의
- 각 페이지의 목적, 주요 기능, 사용자 동작, 필요 데이터, 진입/이탈 경로

6단계: 전체 페이지맵
- 표 형식: 페이지명 / URL / 접근 권한 / 주요 기능 / 연결 페이지

$ARGUMENTS 가 있으면 해당 내용을 추가 컨텍스트로 반영합니다.
```

**사용법:**
```
# 스크린샷 붙여넣고
/spec-from-contract

# 추가 컨텍스트가 있을 때
/spec-from-contract MVP는 로그인과 일정 등록만
```

---

## 분량이 많을 때

문서가 복잡하거나 기능이 많으면 단계를 나눠서 실행합니다.

```
# 1~2단계만 먼저
첨부 문서로 1단계(서비스 파악)와 2단계(기능 목록)만 작성해줘.
확인 후 이어서 진행할게.
```

```
# 특정 기능만 상세화
위 목록에서 [기능명]만 3단계(기능정의서) 작성해줘.
```
