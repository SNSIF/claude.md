# 1-1. .claude/ 디렉터리 구조

## .claude/ 란?

프로젝트 루트에 `.claude/` 폴더를 만들면 Claude Code가 자동으로 인식합니다.
프로젝트별 지침, 스킬, 에이전트, 설정을 여기에 모아둡니다.

```
your-project/
├── .claude/
│   ├── CLAUDE.md        # 프로젝트 지침 (핵심)
│   ├── settings.json    # 프로젝트 권한·설정
│   ├── skills/          # 커스텀 슬래시 커맨드
│   │   └── deploy.md
│   └── agents/          # 커스텀 서브에이전트
│       └── reviewer.md
└── ...
```

---

## 각 파일·폴더 역할

| 경로 | 역할 |
|------|------|
| `.claude/CLAUDE.md` | Claude에게 줄 프로젝트 지침. 대화 시작 시 자동 로드 |
| `.claude/settings.json` | 이 프로젝트에서만 적용할 권한·환경변수 설정 |
| `.claude/skills/` | `/스킬명` 으로 호출할 커스텀 커맨드 모음 |
| `.claude/agents/` | 특정 역할을 맡길 커스텀 에이전트 모음 |

---

## 전역 vs 프로젝트

`.claude/`는 두 곳에 존재합니다.

| 위치 | 경로 | 적용 범위 |
|------|------|-----------|
| 전역 | `~/.claude/` | 모든 프로젝트에 공통 적용 |
| 프로젝트 | `[프로젝트 루트]/.claude/` | 해당 프로젝트에서만 적용 |

> 같은 항목이 있으면 **프로젝트 설정이 전역 설정을 덮어씁니다.**

---

## 시작하는 법

새 프로젝트에서 Claude Code를 처음 세팅할 때:

```bash
mkdir .claude
touch .claude/CLAUDE.md
```

`CLAUDE.md`에 프로젝트 지침을 작성하는 방법은 **[1-2. CLAUDE.md 작성법](1-2.md)** 에서 다룹니다.
