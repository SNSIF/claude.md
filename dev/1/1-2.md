# 1-2. CLAUDE.md 작성법

## CLAUDE.md 란?

프로젝트 루트 또는 `.claude/` 폴더에 두는 지침 파일입니다.
Claude Code가 대화를 시작할 때 자동으로 읽어 컨텍스트로 사용합니다.

---

## 기본 구조

```markdown
# 프로젝트명

## 개요
이 프로젝트는 ~를 하는 서비스입니다.

## 기술 스택
- Frontend: React + TypeScript
- Backend: FastAPI
- DB: PostgreSQL

## 디렉터리 구조
\`\`\`
src/
├── components/
├── pages/
└── utils/
\`\`\`

## 코드 규칙
- 함수명은 camelCase
- 컴포넌트명은 PascalCase
- 커밋 메시지는 한국어로

## 자주 쓰는 명령
- 개발 서버: `npm run dev`
- 테스트: `npm test`
- 빌드: `npm run build`
```

---

## 로드 우선순위

Claude Code는 여러 위치의 CLAUDE.md를 합쳐서 읽습니다.

```
~/.claude/CLAUDE.md          # 전역 (모든 프로젝트 공통)
[프로젝트 루트]/CLAUDE.md     # 프로젝트 전체
[프로젝트 루트]/.claude/CLAUDE.md  # 동일 (어느 쪽이든 OK)
src/CLAUDE.md                # src/ 안에서 작업할 때만 추가 로드
```

> 하위 디렉터리의 CLAUDE.md는 그 디렉터리 안에서 작업할 때만 추가로 읽힙니다.

---

## 실전 팁

**짧게 써도 효과적입니다.** Claude는 이미 코드를 읽을 수 있으므로, 코드에서 알 수 없는 것만 적으면 됩니다.

```markdown
## 주의사항
- `legacy/` 폴더는 건드리지 말 것
- API 변경 시 반드시 `docs/api.md` 업데이트
- DB 마이그레이션은 직접 실행하지 말고 알려줄 것
```

**경로별로 다른 지침**이 필요하면 `@` 임포트를 활용합니다. → [1-3. @ 임포트](1-3.md)
