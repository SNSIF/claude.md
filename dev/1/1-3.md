# 1-3. @ 임포트

## @ 임포트란?

CLAUDE.md 안에서 다른 파일의 내용을 끌어올 수 있는 기능입니다.
공통 지침을 한 파일에 모아두고 여러 CLAUDE.md에서 재사용할 때 유용합니다.

```markdown
@./docs/conventions.md
@.claude/common-rules.md
```

파일 경로는 CLAUDE.md 기준 **상대 경로**로 작성합니다.

---

## 활용 예시

### 공통 규칙 분리

```
.claude/
├── CLAUDE.md          # 메인 지침
├── code-style.md      # 코드 스타일 규칙
└── git-rules.md       # 커밋 규칙
```

**CLAUDE.md**
```markdown
# 프로젝트 지침

@.claude/code-style.md
@.claude/git-rules.md

## 프로젝트 개요
...
```

---

### 모노레포에서 영역별 지침 분리

```
project/
├── CLAUDE.md              # 전체 공통 지침
├── frontend/
│   └── CLAUDE.md          # @../../CLAUDE.md 임포트 후 프론트 전용 규칙
└── backend/
    └── CLAUDE.md          # @../../CLAUDE.md 임포트 후 백엔드 전용 규칙
```

**frontend/CLAUDE.md**
```markdown
@../CLAUDE.md

## 프론트엔드 전용 규칙
- 컴포넌트는 src/components/ 에만 생성
- CSS는 Tailwind만 사용
```

---

## 주의사항

- 임포트한 파일이 존재하지 않으면 무시됩니다 (에러 없음)
- 순환 임포트는 피합니다
- 임포트 깊이가 깊어질수록 컨텍스트가 커지므로 필요한 것만 포함합니다
