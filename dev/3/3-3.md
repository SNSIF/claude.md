# 3-3. Edit — 코드 수정

## 기본 사용

기존 파일의 특정 부분만 수정할 때 사용합니다.

```
src/components/Button.tsx에서 primary 색상을 blue-500에서 indigo-600으로 바꿔줘
```

---

## 동작 방식

Edit은 전체 파일을 덮어쓰지 않고 **변경되는 부분만 교체**합니다.
VSCode에서는 인라인 diff로 변경 전/후를 바로 확인할 수 있습니다.

```
변경 전: className="bg-blue-500"
변경 후: className="bg-indigo-600"
```

---

## 여러 곳 동시에

```
Button.tsx에서
1. primary 색상 blue-500 → indigo-600
2. 호버 효과 추가: hover:opacity-90
3. disabled 상태에 cursor-not-allowed 추가
```

Claude는 하나의 파일에서 여러 군데를 한번에 수정합니다.

---

## replace_all 패턴

같은 문자열을 파일 전체에서 바꿀 때:

```
Button.tsx에서 'onClick' 을 전부 'handleClick' 으로 바꿔줘
```

---

## 실전 팁

**리팩터링에 강합니다.**

```
useAuth 훅에서 token을 localStorage 대신 sessionStorage로 바꿔줘
```

```
api/index.ts의 모든 fetch 호출에 에러 핸들링 추가해줘
```

파일을 먼저 읽고 맥락을 파악한 뒤 수정하므로, 주변 코드 스타일에 맞게 작성됩니다.
