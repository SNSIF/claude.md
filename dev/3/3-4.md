# 3-4. Glob — 파일 검색

## 기본 사용

파일 이름이나 경로 패턴으로 파일을 찾습니다.

```
테스트 파일이 어디 있어?
컴포넌트 파일 전부 보여줘
```

내부적으로 Glob 도구가 `**/*.test.ts`, `src/components/**/*.tsx` 같은 패턴을 실행합니다.

---

## 자주 쓰는 패턴

| 패턴 | 의미 |
|------|------|
| `**/*.ts` | 모든 TypeScript 파일 |
| `src/**/*.tsx` | src 하위 모든 TSX 파일 |
| `**/*.test.*` | 모든 테스트 파일 |
| `**/index.ts` | 모든 index.ts |
| `*.config.*` | 루트의 설정 파일들 |

---

## 직접 쓸 일은 거의 없습니다

Glob은 Claude가 내부적으로 알아서 사용합니다.

```
// 이렇게 물으면 Claude가 스스로 Glob을 사용해 찾습니다
"프로젝트에 환경변수 관련 파일이 뭐가 있어?"
"라우터 설정 파일 어디 있어?"
```

직접 패턴을 지정하고 싶다면:

```
**/*.config.ts 패턴으로 파일 찾아줘
```

---

## Grep과의 차이

| 도구 | 기준 | 예시 |
|------|------|------|
| Glob | 파일 경로/이름 | `**/*.ts` → TypeScript 파일 목록 |
| Grep | 파일 내용 | `useAuth` → 해당 텍스트가 있는 파일 |

파일 이름으로 찾을 땐 Glob, 코드 내용으로 찾을 땐 **[3-5. Grep](3-5.md)**.
