# 8-2. Extended Thinking

## Extended Thinking이란?

Claude가 응답 전에 **내부적으로 깊이 생각하는 시간**을 갖는 기능입니다.
복잡한 문제에서 더 정확하고 신중한 답변을 받을 수 있습니다.

---

## 활성화 방법

명시적으로 요청하면 됩니다:

```
깊게 생각해서 답해줘
천천히 생각해보고 최선의 접근법 알려줘
think hard
```

또는 복잡한 문제를 던지면 Claude가 자동으로 판단합니다.

---

## 언제 효과적인가

**아키텍처 결정:**
```
이 프로젝트에서 상태관리를 Context API로 유지할지 Zustand로 바꿀지 깊게 분석해줘
```

**버그 원인 추적:**
```
이 에러가 왜 발생하는지 모든 가능성을 고려해서 분석해줘
```

**트레이드오프 비교:**
```
REST vs GraphQL을 이 프로젝트 상황에 맞게 비교해줘. 양쪽 장단점 다 고려해서
```

**복잡한 알고리즘:**
```
이 정렬 로직을 최적화할 수 있는 방법 생각해줘
```

---

## 노력 수준 조정

```
간단하게 생각해줘     → 빠른 응답
보통으로             → 기본 (default)
깊게 / 최대한 꼼꼼히  → 더 오래, 더 신중하게
```

---

## 주의사항

- 응답 시간이 길어집니다 (수십 초~분 단위).
- 단순한 작업(파일 수정, 포맷 변경 등)에는 불필요합니다.
- 복잡한 설계 결정이나 버그 분석에 집중적으로 활용합니다.
