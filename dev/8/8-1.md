# 8-1. Plan Mode

## Plan Mode란?

실제 파일 수정이나 명령 실행 없이 **계획만 세우는 모드**입니다.
Claude가 무엇을 어떻게 할지 먼저 보여주고, 승인 후 실행합니다.

---

## 활성화 방법

채팅에서:
```
/plan
```

또는 settings.json에서 기본 모드로 설정:
```json
{
  "defaultMode": "plan"
}
```

---

## 사용 흐름

```
1. Plan Mode 진입
2. Claude에게 작업 요청
3. Claude가 실행 계획 작성 (파일 변경 없음)
4. 계획 확인 → 승인 or 수정 요청
5. 승인 시 실제 작업 실행
```

---

## 언제 쓰나

**큰 작업 전 범위 파악:**
```
결제 모듈 전체를 Stripe에서 Toss로 교체하는 계획 잡아줘
```

**리팩터링 영향 범위 확인:**
```
UserContext를 Zustand로 교체하면 어떤 파일이 영향받는지 계획으로 보여줘
```

**낯선 코드베이스에서 신중하게 작업:**
```
이 기능 추가하기 전에 어떤 파일을 건드려야 하는지 먼저 보여줘
```

---

## Plan Mode 해제

```
/mode default
```

또는 작업 승인 후 자동으로 해제됩니다.

---

## Plan Mode vs Explore 에이전트

| | Plan Mode | Explore 에이전트 |
|--|-----------|-----------------|
| 목적 | 실행 계획 수립 | 코드 분석·탐색 |
| 결과물 | 단계별 실행 계획 | 분석 보고서 |
| 실행 전환 | 승인 후 즉시 실행 가능 | 별도 요청 필요 |
