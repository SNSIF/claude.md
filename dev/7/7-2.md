# 7-2. 메모리 관리

## 메모리 확인

```
/memory
```

저장된 메모리 목록을 보여줍니다.

---

## 메모리 편집

```
/memory
```

커맨드 실행 후 VSCode에서 메모리 파일을 직접 열어 편집할 수 있습니다.
또는 Claude에게 직접 요청합니다:

```
저장된 메모리 중 커밋 메시지 관련된 거 보여줘
```

---

## 메모리 삭제

```
커밋 메시지 영어로 쓴다는 메모리 지워줘
```

또는 메모리 파일을 직접 삭제합니다.

---

## 메모리 무시

특정 대화에서 메모리를 참조하지 않게 하려면:

```
이전 메모리 무시하고, 이번엔 이렇게 해줘: ...
```

---

## 실전 팁

**메모리에 저장할 만한 것:**
- 팀 규칙 ("PR은 항상 squash merge")
- 작업 스타일 ("설명보다 코드 먼저 보여줘")
- 프로젝트 특이사항 ("legacy/ 폴더는 건드리지 말 것")
- 자주 하는 피드백 ("요약 없이 바로 결과만")

**메모리에 저장하지 않아도 되는 것:**
- 코드 패턴, 아키텍처 → CLAUDE.md에 작성
- 임시 작업 상태 → 대화 내에서만 유효
