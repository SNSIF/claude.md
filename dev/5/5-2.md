# 5-2. 실전 훅 레시피

## 1. 파일 수정 후 자동 포맷터 실행

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "prettier --write \"$CLAUDE_TOOL_INPUT_FILE_PATH\" 2>/dev/null || true"
          }
        ]
      }
    ]
  }
}
```

> `|| true` 를 붙이면 포맷터 오류가 있어도 Claude 작업이 중단되지 않습니다.

---

## 2. TypeScript 파일 수정 시 타입 체크

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "if [[ \"$CLAUDE_TOOL_INPUT_FILE_PATH\" == *.ts || \"$CLAUDE_TOOL_INPUT_FILE_PATH\" == *.tsx ]]; then npx tsc --noEmit 2>&1 | head -20; fi"
          }
        ]
      }
    ]
  }
}
```

---

## 3. 특정 파일 보호 (수정 차단)

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "if [[ \"$CLAUDE_TOOL_INPUT_FILE_PATH\" == *\".env\"* ]]; then echo '.env 파일은 직접 수정하지 마세요' && exit 1; fi"
          }
        ]
      }
    ]
  }
}
```

> `PreToolUse` + `exit 1` 로 실행 자체를 막을 수 있습니다.

---

## 4. 작업 완료 시 알림

```json
{
  "hooks": {
    "Stop": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "osascript -e 'display notification \"Claude 작업 완료\" with title \"Claude Code\"'"
          }
        ]
      }
    ]
  }
}
```

> Windows에서는 `osascript` 대신 PowerShell 알림 명령을 사용합니다.

---

## 5. 세션 시작 시 환경 상태 출력

```json
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "echo \"Branch: $(git branch --show-current)\" && echo \"Node: $(node -v)\""
          }
        ]
      }
    ]
  }
}
```
