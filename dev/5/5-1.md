# 5-1. 훅 개념 & 이벤트

## 훅(Hook)이란?

Claude Code가 특정 행동을 할 때 **자동으로 끼어드는 쉘 명령**입니다.
"파일 수정 후 자동으로 포맷터 실행" 같은 결정적 자동화를 만들 수 있습니다.

---

## 설정 위치

`.claude/settings.json` 또는 `~/.claude/settings.json`

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "prettier --write $CLAUDE_TOOL_INPUT_FILE_PATH"
          }
        ]
      }
    ]
  }
}
```

---

## 주요 이벤트

| 이벤트 | 발생 시점 |
|--------|-----------|
| `PreToolUse` | 도구 실행 **직전** |
| `PostToolUse` | 도구 실행 **직후** |
| `Stop` | Claude가 응답을 **완료했을 때** |
| `SessionStart` | 새 세션이 **시작될 때** |

---

## matcher — 어떤 도구에 반응할까

| matcher 값 | 반응 도구 |
|------------|-----------|
| `"Edit"` | 파일 수정 시 |
| `"Write"` | 파일 생성 시 |
| `"Edit\|Write"` | 수정 또는 생성 시 |
| `"Bash"` | 터미널 명령 실행 시 |
| `""` (빈 문자열) | 모든 도구 |

---

## 환경변수

훅 명령 안에서 사용할 수 있는 변수들:

| 변수 | 의미 |
|------|------|
| `$CLAUDE_TOOL_INPUT_FILE_PATH` | 수정/생성된 파일 경로 |
| `$CLAUDE_TOOL_NAME` | 실행된 도구 이름 |
| `$CLAUDE_SESSION_ID` | 현재 세션 ID |

실전 레시피는 **[5-2. 실전 훅 레시피](5-2.md)** 에서 다룹니다.
