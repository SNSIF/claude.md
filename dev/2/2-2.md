# 2-2. allow / deny 규칙

## 왜 필요한가?

기본적으로 Claude Code는 파일 수정, 명령 실행 등 위험할 수 있는 작업마다 승인을 요청합니다.
`allow` 규칙을 설정하면 자주 쓰는 작업을 자동 허용해 승인 횟수를 줄일 수 있습니다.

---

## 기본 형식

```json
{
  "permissions": {
    "allow": [
      "Bash(npm run *)",
      "Edit",
      "Write"
    ],
    "deny": [
      "Bash(rm -rf *)"
    ]
  }
}
```

---

## 규칙 문법

| 패턴 | 의미 |
|------|------|
| `"Edit"` | Edit 도구 전체 허용 |
| `"Bash(npm *)"` | `npm`으로 시작하는 Bash 명령 허용 |
| `"Bash(git commit *)"` | git commit 허용 |
| `"Read(/src/*)"` | src/ 안의 파일 읽기만 허용 |
| `"WebFetch(https://api.example.com/*)"` | 특정 도메인 fetch 허용 |

---

## 실전 예시

### 개발 프로젝트용 `.claude/settings.json`

```json
{
  "permissions": {
    "allow": [
      "Bash(npm run *)",
      "Bash(npm install *)",
      "Bash(git status)",
      "Bash(git diff *)",
      "Bash(git add *)",
      "Bash(git commit *)",
      "Edit",
      "Write",
      "Glob",
      "Grep"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Bash(git push --force *)"
    ]
  }
}
```

---

## deny 우선순위

allow와 deny가 겹치면 **deny가 우선**입니다.

```json
"allow": ["Bash(git *)"],
"deny":  ["Bash(git push --force *)"]
// → git push --force 는 차단, 나머지 git 명령은 허용
```

---

## 어디에 쓸까?

| 규칙 | 위치 |
|------|------|
| 팀 전체 공통 허용 | `.claude/settings.json` (git 포함) |
| 나만 쓰는 로컬 허용 | `.claude/settings.local.json` (git 제외) |
| 모든 프로젝트 공통 | `~/.claude/settings.json` |
