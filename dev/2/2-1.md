# 2-1. 설정 계층 구조

## 설정 파일 위치

Claude Code의 설정은 4개 계층으로 나뉩니다.

```
~/.claude/settings.json              # User (전역)
[프로젝트]/.claude/settings.json     # Project
[프로젝트]/.claude/settings.local.json  # Local (git 제외 권장)
Managed (관리자가 배포, 수정 불가)
```

---

## 우선순위

충돌 시 아래로 갈수록 우선순위가 높습니다.

```
Managed < User < Project < Local
```

> 예: Project에서 `npm test` 를 deny해도 Local에서 allow하면 허용됩니다.

---

## 각 계층 용도

| 계층 | 파일 | 언제 쓰나 |
|------|------|-----------|
| User | `~/.claude/settings.json` | 모든 프로젝트에 공통 적용할 규칙 |
| Project | `.claude/settings.json` | 팀 전체가 따를 프로젝트 규칙 (git에 포함) |
| Local | `.claude/settings.local.json` | 나만의 로컬 오버라이드 (git 제외) |

---

## 기본 구조

```json
{
  "permissions": {
    "allow": [],
    "deny": []
  },
  "env": {}
}
```

권한 설정 방법은 **[2-2. allow / deny 규칙](2-2.md)** 에서 다룹니다.
