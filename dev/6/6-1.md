# 6-1. 커스텀 스킬 만들기

## 스킬이란?

`.claude/skills/` 폴더에 Markdown 파일로 만드는 **나만의 슬래시 커맨드**입니다.
반복되는 작업 절차를 커맨드 하나로 등록해두면 매번 긴 프롬프트를 치지 않아도 됩니다.

---

## 파일 구조

```
.claude/
└── skills/
    ├── commit.md       # /commit 으로 호출
    ├── review.md       # /review 로 호출
    └── deploy-check.md # /deploy-check 으로 호출
```

파일 이름이 곧 커맨드 이름입니다.

---

## 스킬 파일 작성법

```markdown
# commit

변경된 파일의 diff를 분석하고, 다음 규칙에 따라 커밋합니다:
- 커밋 메시지는 한국어로 작성
- prefix: feat / fix / refactor / docs / chore
- 본문에 변경 이유 한 줄 추가
- 스테이징되지 않은 파일은 포함하지 않음
```

---

## 호출 방법

```
/commit
```

Claude가 스킬 파일의 지침을 읽고 그대로 실행합니다.

---

## 인자 전달

```markdown
# review

$ARGUMENTS 에 지정된 파일을 코드 리뷰합니다.
지정하지 않으면 최근 변경된 파일을 리뷰합니다.

리뷰 항목:
- 버그 가능성
- 성능 이슈
- 가독성
```

```
/review src/api/auth.ts
```

`$ARGUMENTS`에 `src/api/auth.ts`가 전달됩니다.

---

## 실전 스킬 예시

```markdown
# deploy-check

배포 전 체크리스트를 실행합니다:
1. console.log 가 남아있는지 Grep으로 확인
2. TODO 주석 목록 출력
3. .env.example 과 실제 .env 키 비교
4. 테스트 실행: npm test
5. 빌드 확인: npm run build
이슈가 있으면 목록으로 정리해서 보고합니다.
```
