# 6-3. 내장 에이전트 활용

## 내장 에이전트 3종

Claude Code에는 별도 설정 없이 바로 쓸 수 있는 에이전트가 있습니다.

---

## Explore — 코드베이스 탐색

코드를 읽고 분석하는 데 특화된 에이전트입니다.
파일 수정 없이 탐색만 하므로 안전합니다.

```
이 프로젝트 인증 흐름이 어떻게 되어 있는지 분석해줘
API 엔드포인트 전체 목록 정리해줘
```

**언제 쓰나:** 낯선 코드베이스를 처음 파악할 때, 영향 범위를 파악할 때

---

## Plan — 계획 수립

실제 작업 없이 **무엇을 어떻게 할지 계획만** 세우는 에이전트입니다.
계획을 확인하고 승인한 뒤 실행할 수 있습니다.

```
결제 모듈 추가하는 계획 잡아줘. 실제로 코드는 건드리지 말고
```

**언제 쓰나:** 큰 기능 구현 전, 리팩터링 범위 파악 전

> Plan Mode와 연결됩니다 → [8-1. Plan Mode](../8/8-1.md)

---

## general-purpose — 범용 에이전트

파일 읽기/쓰기/검색/실행 등 모든 도구를 사용할 수 있는 기본 에이전트입니다.
복잡한 멀티스텝 작업에 적합합니다.

```
// 여러 단계가 필요한 작업
"로그인 기능 전체를 처음부터 구현해줘. DB 스키마부터 API, 프론트엔드까지"
```

---

## 에이전트를 명시적으로 지정하는 법

```
Explore 에이전트로 이 프로젝트 구조 파악해줘
Plan 에이전트로 리팩터링 계획 잡아줘
```

명시하지 않으면 Claude가 상황에 따라 적절한 에이전트를 자동 선택합니다.
