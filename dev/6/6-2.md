# 6-2. 커스텀 에이전트 만들기

## 에이전트란?

`.claude/agents/` 폴더에 Markdown 파일로 만드는 **역할을 가진 서브에이전트**입니다.
특정 도메인에 특화된 지침, 도구 제한, 권한을 가진 전용 에이전트를 만들 수 있습니다.

---

## 파일 구조

```
.claude/
└── agents/
    ├── reviewer.md    # 코드 리뷰 전문 에이전트
    ├── tester.md      # 테스트 작성 전문 에이전트
    └── docs-writer.md # 문서 작성 전문 에이전트
```

---

## 에이전트 파일 작성법

```markdown
---
name: reviewer
description: 코드 리뷰를 요청할 때 자동으로 사용됩니다. PR 리뷰, 코드 품질 검토에 특화됩니다.
tools: Read, Glob, Grep
---

당신은 시니어 개발자로서 코드 리뷰를 담당합니다.

리뷰 시 다음 항목을 반드시 확인합니다:
- 버그 및 엣지 케이스
- 성능 병목 가능성
- 보안 취약점 (XSS, SQL Injection 등)
- 코드 중복 및 리팩터링 기회
- 네이밍 일관성

심각도를 [🔴 critical / 🟡 warning / 🟢 suggestion] 으로 표시합니다.
```

---

## 프론트매터 옵션

| 키 | 설명 |
|----|------|
| `name` | 에이전트 이름 |
| `description` | Claude가 언제 이 에이전트를 쓸지 판단하는 기준 |
| `tools` | 허용할 도구 목록 (제한하면 더 안전) |
| `model` | 사용할 모델 (생략 시 기본 모델) |

---

## 에이전트 호출 방법

### 자동 위임
`description`을 잘 쓰면 Claude가 상황에 맞게 알아서 에이전트를 씁니다.

```
이 PR 코드 리뷰해줘  →  reviewer 에이전트 자동 사용
```

### 명시적 호출
```
@reviewer 이 파일 리뷰해줘
```

---

## 스킬 vs 에이전트

| | 스킬 | 에이전트 |
|--|------|----------|
| 용도 | 반복 절차 자동화 | 역할 특화 |
| 도구 제한 | 불가 | 가능 |
| 자율성 | 낮음 (지침 실행) | 높음 (스스로 판단) |
| 호출 | `/스킬명` | `@에이전트명` 또는 자동 |
