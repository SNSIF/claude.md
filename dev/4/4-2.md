# 4-2. 브랜치 & PR

## 브랜치 작업

```
feature/login-validation 브랜치 만들어줘
현재 브랜치에서 main으로 머지해줘
브랜치 목록 보여줘
```

---

## PR 초안 작성

```
지금까지 작업한 내용으로 PR 초안 만들어줘
```

Claude가 diff를 분석해서 제목과 본문을 작성합니다.

**생성 결과 예시:**
```markdown
## PR 제목
feat: 로그인 폼 유효성 검사 추가

## 변경 내용
- 이메일 형식 검사 추가
- 비밀번호 최소 길이(8자) 검사
- 에러 메시지 UI 컴포넌트 추가

## 테스트
- [ ] 유효하지 않은 이메일 입력 시 에러 표시 확인
- [ ] 비밀번호 8자 미만 입력 시 에러 표시 확인
```

---

## GitHub CLI 연동

`gh` 명령이 설치되어 있으면 PR을 직접 생성합니다.

```
PR 만들어서 올려줘
```

→ 내부적으로 `gh pr create` 실행

```
열려있는 PR 목록 보여줘
PR #42 리뷰해줘
```

---

## 실전 활용

```
// 브랜치 전략대로 브랜치 생성
"이슈 #15 작업할 브랜치 만들어줘. 이름은 fix/issue-15로"

// 커밋 후 PR까지 한번에
"작업 내용 커밋하고 PR 초안까지 만들어줘"
```
