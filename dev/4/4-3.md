# 4-3. Worktree

## Worktree란?

하나의 git 저장소에서 **여러 브랜치를 동시에 별도 폴더로 체크아웃**하는 기능입니다.
브랜치를 전환하지 않고도 두 작업을 병렬로 진행할 수 있습니다.

---

## 어떤 상황에 유용한가?

- 큰 기능 개발 중 긴급 버그 수정이 생겼을 때
- Claude 에이전트 여러 개를 독립된 브랜치에서 동시에 돌릴 때
- A/B로 다른 접근법을 동시에 실험할 때

---

## 기본 사용법

```bash
# 새 worktree 생성
git worktree add ../my-project-hotfix hotfix/urgent-bug

# worktree 목록 확인
git worktree list

# worktree 제거
git worktree remove ../my-project-hotfix
```

---

## Claude Code에서 활용

Claude Code의 Worktree 격리 기능을 사용하면 에이전트가 독립된 복사본에서 작업합니다.

```
// 현재 작업을 건드리지 않고 별도 worktree에서 리팩터링 진행
"auth 모듈 리팩터링을 별도 worktree에서 진행해줘"
```

에이전트가 변경사항을 만들면 worktree 경로와 브랜치를 반환하고,
변경이 없으면 자동으로 정리됩니다.

---

## 주의사항

- 같은 브랜치를 두 worktree에서 동시에 체크아웃할 수 없습니다.
- `node_modules` 같은 의존성은 각 worktree에서 별도로 설치해야 합니다.
- 작업이 끝나면 `git worktree remove`로 정리합니다.
