# 4-1. 기본 Git 흐름

## 자연어로 Git 명령 실행

Claude Code는 자연어 요청을 Git 명령으로 처리합니다.
직접 터미널을 열지 않아도 됩니다.

---

## 자주 쓰는 흐름

### 현재 상태 확인
```
지금 변경된 파일 뭐야?
스테이징 안 된 게 뭐 있어?
```
→ `git status`, `git diff`

### 변경 내용 확인
```
방금 내가 뭘 바꿨는지 보여줘
Button.tsx 에서 바뀐 부분 보여줘
```
→ `git diff`, `git diff [파일]`

### 커밋
```
지금까지 작업한 내용 커밋해줘
커밋 메시지는 "feat: 로그인 폼 유효성 검사 추가"로
```
→ `git add`, `git commit`

### 로그 확인
```
최근 커밋 10개 보여줘
어제 내가 뭘 커밋했어?
```
→ `git log`

---

## Claude가 커밋 메시지를 대신 써줍니다

```
변경 내용 보고 적절한 커밋 메시지로 커밋해줘
```

diff를 분석해서 의미 있는 커밋 메시지를 제안합니다.

---

## 주의사항

- `git push` 같이 **외부에 영향을 주는 명령**은 기본적으로 승인을 요청합니다.
- `git reset --hard`, `git push --force` 같은 위험한 명령은 deny 목록에 추가해두는 것을 권장합니다. → [2-2. allow/deny 규칙](../2/2-2.md)
