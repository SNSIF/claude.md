# 7-2. 주요 훅 이벤트

## 전체 이벤트 목록

Claude Code는 다음 이벤트 시점에 훅을 실행할 수 있다.

| 이벤트 | 실행 시점 |
|--------|-----------|
| `SessionStart` | 세션 시작 또는 재개 시 |
| `SessionEnd` | 세션 종료 시 |
| `UserPromptSubmit` | 사용자 프롬프트 제출 직후 |
| `PreToolUse` | 도구 실행 직전 |
| `PostToolUse` | 도구 실행 성공 후 |
| `PostToolUseFailure` | 도구 실행 실패 후 |
| `PermissionRequest` | 권한 다이얼로그 표시 전 |
| `Stop` | Claude 응답 완료 시 |
| `Notification` | Claude 알림 발송 시 |
| `SubagentStart` | 서브에이전트 시작 시 |
| `SubagentStop` | 서브에이전트 완료 시 |
| `PreCompact` | 컨텍스트 압축 직전 |
| `PostCompact` | 컨텍스트 압축 완료 후 |
| `ConfigChange` | 설정 파일 변경 감지 시 |

---

## 세션 관련 이벤트

### SessionStart — 세션 시작/재개

세션이 처음 열리거나 중단 후 재개될 때 실행된다.

**활용 예시:**
- 환경 상태 점검
- 작업 일지 시작
- 필요한 서비스가 실행 중인지 확인

```json
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "echo \"[$(date)] 세션 시작\" >> ~/.claude/session-log.txt"
          }
        ]
      }
    ]
  }
}
```

**사용 가능한 환경 변수:**
- `$SESSION_ID` — 세션 식별자
- `$WORKSPACE_DIR` — 현재 작업 디렉토리

---

### SessionEnd — 세션 종료

세션이 완전히 종료될 때 실행된다.

**활용 예시:**
- 작업 시간 집계
- 임시 파일 정리
- 세션 요약 저장

```json
{
  "hooks": {
    "SessionEnd": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "echo \"[$(date)] 세션 종료\" >> ~/.claude/session-log.txt"
          }
        ]
      }
    ]
  }
}
```

## 프롬프트 관련 이벤트

### UserPromptSubmit — 사용자 프롬프트 제출

사용자가 메시지를 전송한 직후, Claude가 처리하기 전에 실행된다.

**활용 예시:**
- 프롬프트 로깅
- 금지 키워드 필터링
- 컨텍스트 자동 주입

```json
{
  "hooks": {
    "UserPromptSubmit": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "echo \"[$(date)] 프롬프트: $PROMPT_TEXT\" >> ~/.claude/prompt-log.txt"
          }
        ]
      }
    ]
  }
}
```

**사용 가능한 환경 변수:**
- `$PROMPT_TEXT` — 사용자가 입력한 프롬프트 전문

## 도구 관련 이벤트

### PreToolUse — 도구 실행 전 (차단 가능)

Claude가 도구를 실행하기 **직전**에 실행된다. **exit 코드 2**를 반환하면 해당 도구 실행을 차단할 수 있다.

**활용 예시:**
- 위험한 파일 수정 차단
- 실행 전 사용자 확인
- 도구 사용 감사 로그

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "/bin/bash -c 'if [[ \"$FILE_PATH\" == *\".env\"* ]]; then echo \".env 파일 수정 차단\"; exit 2; fi'"
          }
        ]
      }
    ]
  }
}
```

**차단 메커니즘:**

| exit 코드 | 동작 |
|-----------|------|
| `0` | 정상 진행 |
| `1` | 오류 발생, 경고 출력 후 진행 |
| `2` | **도구 실행 차단** |

**사용 가능한 환경 변수:**
- `$TOOL_NAME` — 실행 예정 도구 이름
- `$FILE_PATH` — 대상 파일 경로 (파일 관련 도구)
- `$COMMAND` — 실행 예정 명령 (Bash 도구)

---

### PostToolUse — 도구 실행 성공 후

도구가 성공적으로 실행된 **직후**에 실행된다.

**활용 예시:**
- 저장 후 포맷터 실행
- 파일 수정 후 테스트 실행
- 변경사항 자동 스테이징

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "npx prettier --write \"$FILE_PATH\" 2>/dev/null || true"
          }
        ]
      }
    ]
  }
}
```

**사용 가능한 환경 변수:**
- `$TOOL_NAME` — 실행된 도구 이름
- `$FILE_PATH` — 수정된 파일 경로
- `$TOOL_OUTPUT` — 도구 실행 결과 (stdout)

---

### PostToolUseFailure — 도구 실행 실패 후

도구 실행이 **실패했을 때** 실행된다.

**활용 예시:**
- 실패 알림
- 오류 로깅
- 자동 복구 시도

```json
{
  "hooks": {
    "PostToolUseFailure": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "echo \"[$(date)] 도구 실패: $TOOL_NAME — $ERROR_MESSAGE\" >> .claude/error-log.txt"
          }
        ]
      }
    ]
  }
}
```

**사용 가능한 환경 변수:**
- `$TOOL_NAME` — 실패한 도구 이름
- `$ERROR_MESSAGE` — 오류 메시지
- `$EXIT_CODE` — 종료 코드

---

### PermissionRequest — 권한 다이얼로그 표시 전

Claude가 권한 요청 다이얼로그를 표시하기 전에 실행된다.

**활용 예시:**
- 특정 권한 자동 승인
- 권한 요청 로깅
- 보안 정책 적용

```json
{
  "hooks": {
    "PermissionRequest": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "echo \"[$(date)] 권한 요청: $PERMISSION_TYPE\" >> .claude/permission-log.txt"
          }
        ]
      }
    ]
  }
}
```

## Claude 관련 이벤트

### Stop — Claude 응답 완료

Claude가 하나의 응답을 완전히 완료했을 때 실행된다.

**활용 예시:**
- 작업 완료 알림 (알림음, 푸시 알림)
- 자동 git 커밋
- 완료 시간 기록

```json
{
  "hooks": {
    "Stop": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "osascript -e 'display notification \"Claude 작업 완료\" with title \"Claude Code\"'"
          }
        ]
      }
    ]
  }
}
```

---

### Notification — Claude 알림 발송

Claude가 사용자에게 알림을 보낼 때 실행된다.

**활용 예시:**
- 알림 내용 외부 전달
- Slack/Discord 알림 연동

```json
{
  "hooks": {
    "Notification": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "curl -s -X POST $SLACK_WEBHOOK -d \"{\\\"text\\\": \\\"$NOTIFICATION_TEXT\\\"}\""
          }
        ]
      }
    ]
  }
}
```

**사용 가능한 환경 변수:**
- `$NOTIFICATION_TEXT` — 알림 메시지 내용
- `$NOTIFICATION_TYPE` — 알림 유형

## 실습: PreToolUse로 파일 차단하기

### 목표
`.env` 파일이 수정되려 할 때 차단하고 경고 메시지 출력하기

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Write|Edit|MultiEdit",
        "hooks": [
          {
            "type": "command",
            "command": "/bin/bash -c 'case \"$FILE_PATH\" in *.env|*.env.*) echo \"경고: 환경 변수 파일($FILE_PATH) 수정이 차단되었습니다.\"; exit 2 ;; esac'"
          }
        ]
      }
    ]
  }
}
```

### 테스트 방법
1. 위 설정을 `.claude/settings.json`에 추가
2. Claude Code에서 `.env` 파일 수정 요청
3. 차단 메시지 확인

---

## 핵심 요약

- **SessionStart/End**: 세션 생명주기 추적
- **PreToolUse**: 도구 실행 전 차단 가능 (exit 2)
- **PostToolUse**: 도구 실행 후 자동화의 핵심
- **Stop**: 작업 완료 알림에 최적
- 각 이벤트마다 사용 가능한 환경 변수가 다르다
