# 7-4. 실전 훅 예제

## 개요

이 파일은 실제 개발 워크플로에서 바로 사용할 수 있는 훅 예제 모음이다. 각 예제를 `.claude/settings.json`에 추가하거나 조합해 사용한다.

---

## 1. 코드 자동 포맷팅 (PostToolUse)

파일이 저장될 때마다 자동으로 코드 스타일을 정리한다.

### Prettier 자동 실행

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit|MultiEdit",
        "hooks": [
          {
            "type": "command",
            "command": "/bin/bash -c 'if [[ \"$FILE_PATH\" =~ \\.(js|jsx|ts|tsx|css|json|md)$ ]]; then npx prettier --write \"$FILE_PATH\" 2>/dev/null; fi; exit 0'",
            "timeout": 15000
          }
        ]
      }
    ]
  }
}
```

### ESLint --fix 자동 실행

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "/bin/bash -c 'if [[ \"$FILE_PATH\" =~ \\.(js|jsx|ts|tsx)$ ]]; then npx eslint --fix \"$FILE_PATH\" 2>/dev/null; fi; exit 0'",
            "timeout": 20000
          }
        ]
      }
    ]
  }
}
```

### Python black + isort 자동 실행

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "/bin/bash -c 'if [[ \"$FILE_PATH\" =~ \\.py$ ]]; then black \"$FILE_PATH\" 2>/dev/null && isort \"$FILE_PATH\" 2>/dev/null; fi; exit 0'",
            "timeout": 20000
          }
        ]
      }
    ]
  }
}
```

### Prettier + ESLint 조합 (언어별 분기)

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit|MultiEdit",
        "hooks": [

            "type": "command",
            "command": "/bin/bash -c 'EXT=\"${FILE_PATH##*.}\"; case $EXT in js|jsx|ts|tsx) npx eslint --fix \"$FILE_PATH\" 2>/dev/null; npx prettier --write \"$FILE_PATH\" 2>/dev/null;; py) black \"$FILE_PATH\" 2>/dev/null; isort \"$FILE_PATH\" 2>/dev/null;; css|scss) npx prettier --write \"$FILE_PATH\" 2>/dev/null;; esac; exit 0'",

          }
        ]

    ]
  }
}
```

## 2. 파일 보호 (PreToolUse)

중요한 파일이 실수로 수정되는 것을 방지한다.

### .env 파일 수정 차단

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Write|Edit|MultiEdit",
        "hooks": [
          {
            "type": "command",
            "command": "/bin/bash -c 'FILENAME=$(basename \"$FILE_PATH\"); if [[ \"$FILENAME\" == .env || \"$FILENAME\" == .env.* ]]; then echo \"오류: .env 파일($FILE_PATH)은 Claude가 직접 수정할 수 없습니다. 직접 편집하세요.\"; exit 2; fi; exit 0'"
          }
        ]
      }
    ]
  }
}
```

### package-lock.json 직접 수정 차단

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "/bin/bash -c 'FILENAME=$(basename \"$FILE_PATH\"); if [[ \"$FILENAME\" == \"package-lock.json\" || \"$FILENAME\" == \"yarn.lock\" || \"$FILENAME\" == \"pnpm-lock.yaml\" ]]; then echo \"오류: 락 파일은 직접 수정하지 마세요. npm install 등 패키지 매니저를 사용하세요.\"; exit 2; fi; exit 0'"
          }
        ]
      }
    ]
  }
}
```

### 프로덕션 설정 파일 보호

```bash
# .claude/protect-prod.sh
#!/bin/bash

PROTECTED_PATTERNS=(
    "config/production.json"
    "config/prod.yml"
    "k8s/production"
    "terraform/prod"
    ".github/workflows"
)

for pattern in "${PROTECTED_PATTERNS[@]}"; do
    if [[ "$FILE_PATH" == *"$pattern"* ]]; then
        echo "경고: 프로덕션 파일($FILE_PATH) 수정이 차단되었습니다."
        echo "직접 편집하거나 관리자에게 문의하세요."
        exit 2
    fi
done

exit 0
```

```json
  "hooks": {

        "hooks": [
          {
            "type": "command",
            "command": "bash .claude/protect-prod.sh"
          }
        ]
      }
    ]
  }
}
```

## 3. 자동 알림 (Stop)

Claude가 작업을 완료했을 때 알림을 받는다. 긴 작업을 맡기고 다른 일을 할 때 유용하다.

### macOS: osascript 알림

```json
{
  "hooks": {
    "Stop": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "osascript -e 'display notification \"작업이 완료되었습니다\" with title \"Claude Code\" sound name \"Glass\"'"
          }
        ]
      }
    ]
  }
}
```

### Windows: PowerShell 알림

```json
{
  "hooks": {
    "Stop": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "powershell -Command \"[System.Reflection.Assembly]::LoadWithPartialName('System.Windows.Forms'); [System.Windows.Forms.MessageBox]::Show('Claude 작업 완료', 'Claude Code')\"",
            "timeout": 10000
          }
        ]
      }
    ]
  }
}
```

### Linux: notify-send

```json
{
  "hooks": {
    "Stop": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "notify-send 'Claude Code' '작업이 완료되었습니다' --icon=dialog-information 2>/dev/null; exit 0"
          }
        ]
      }
    ]
  }
}
```

### Slack 웹훅 알림

## 4. 자동 테스트 (PostToolUse)

파일 저장 후 관련 테스트를 자동으로 실행한다.

### 변경 파일 관련 테스트 자동 실행

```bash
# .claude/run-related-tests.sh
#!/bin/bash

FILE_PATH="${FILE_PATH:-}"
if [ -z "$FILE_PATH" ]; then exit 0; fi

# 테스트 파일이면 그 자체를 실행
if [[ "$FILE_PATH" =~ \.(test|spec)\.(js|ts|jsx|tsx)$ ]]; then
    npx jest "$FILE_PATH" --no-coverage 2>&1 | tail -20
    exit 0
fi

# 소스 파일이면 같은 이름의 테스트 파일 검색
BASENAME=$(basename "$FILE_PATH" | sed 's/\.[^.]*$//')
TESTFILE=$(find . -name "${BASENAME}.test.*" -o -name "${BASENAME}.spec.*" 2>/dev/null | head -1)

if [ -n "$TESTFILE" ]; then
    echo "관련 테스트 실행: $TESTFILE"
    npx jest "$TESTFILE" --no-coverage 2>&1 | tail -20
fi

exit 0
```

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "bash .claude/run-related-tests.sh",
            "timeout": 60000
          }
        ]
      }
    ]
  }
}
```

### Python pytest 자동 실행

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "/bin/bash -c 'if [[ \"$FILE_PATH\" =~ \\.py$ ]]; then TESTFILE=$(echo \"$FILE_PATH\" | sed \"s/\\.py$/_test.py/\"); if [ -f \"$TESTFILE\" ]; then python -m pytest \"$TESTFILE\" -v --tb=short 2>&1 | tail -20; fi; fi; exit 0'",
            "timeout": 60000
          }
        ]
      }
    ]
  }
}
```

## 5. 로깅 (SessionStart/End)

세션과 도구 사용 이력을 파일에 기록한다.

### 세션 시작/종료 시간 기록

```json
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "echo \"=== 세션 시작: $(date '+%Y-%m-%d %H:%M:%S') | 디렉토리: $(pwd) ===\" >> ~/.claude/session-history.log"
          }
        ]
      }
    ],
    "SessionEnd": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "echo \"=== 세션 종료: $(date '+%Y-%m-%d %H:%M:%S') ===\" >> ~/.claude/session-history.log"
          }
        ]
      }
    ]
  }
}
```

### 도구 사용 이력 저장

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "echo \"$(date '+%Y-%m-%d %H:%M:%S') | $TOOL_NAME | $FILE_PATH\" >> ~/.claude/tool-history.log"
          }
        ]
      }
    ]
  }
}
```

### 일별 작업 요약 생성

```bash
# .claude/daily-summary.sh
#!/bin/bash

LOG_FILE=~/.claude/tool-history.log
TODAY=$(date '+%Y-%m-%d')
SUMMARY_FILE=~/.claude/summary-${TODAY}.txt

echo "=== $TODAY 작업 요약 ===" > "$SUMMARY_FILE"
echo "" >> "$SUMMARY_FILE"
echo "수정된 파일:" >> "$SUMMARY_FILE"
grep "$TODAY" "$LOG_FILE" 2>/dev/null | awk '{print $3}' | sort -u >> "$SUMMARY_FILE"
echo "" >> "$SUMMARY_FILE"
echo "도구 사용 횟수:" >> "$SUMMARY_FILE"
grep "$TODAY" "$LOG_FILE" 2>/dev/null | awk '{print $2}' | sort | uniq -c | sort -rn >> "$SUMMARY_FILE"

echo "요약 저장됨: $SUMMARY_FILE"
```

```json
{
  "hooks": {
    "SessionEnd": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "bash .claude/daily-summary.sh 2>/dev/null; exit 0"
          }
        ]
      }
    ]
  }
}
```

## 6. Git 자동화 (Stop)

Claude 응답 완료 후 변경사항을 자동으로 스테이징한다.

### 자동 git add

```json
{
  "hooks": {
    "Stop": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "/bin/bash -c 'if git rev-parse --git-dir > /dev/null 2>&1; then CHANGED=$(git status --porcelain | wc -l); if [ \"$CHANGED\" -gt 0 ]; then git add -A && echo \"변경 파일 $CHANGED개 스테이징 완료\"; fi; fi; exit 0'",
            "timeout": 10000
          }
        ]
      }
    ]
  }
}
```

### 자동 커밋 (주의: 신중히 사용)

```json
{
  "hooks": {
    "Stop": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "/bin/bash -c 'if git rev-parse --git-dir > /dev/null 2>&1; then CHANGED=$(git status --porcelain | wc -l); if [ \"$CHANGED\" -gt 0 ]; then git add -A && git commit -m \"auto: Claude Code 변경사항 [$(date +%H:%M)]\" && echo \"자동 커밋 완료\"; fi; fi; exit 0'",
            "timeout": 15000
          }
        ]
      }
    ]
  }
}
```

## 핵심 요약

- **포맷팅**: PostToolUse + command 조합이 가장 실용적
- **파일 보호**: PreToolUse + exit 2로 즉시 차단
- **알림**: Stop 이벤트 + 플랫폼별 명령 사용
- **테스트**: PostToolUse에서 관련 테스트 자동 탐색 및 실행
- **로깅**: SessionStart/End + PostToolUse로 완전한 이력 관리
- **Git**: Stop 이벤트에서 자동 스테이징 (커밋은 신중히)
- 모든 command 훅은 마지막에 `exit 0` 또는 `|| true`를 추가해 실패가 워크플로를 막지 않도록 한다
