# 7-1. 훅(Hook) 개념

## 훅이란 무엇인가

훅(Hook)은 Claude Code 라이프사이클의 특정 지점에서 자동으로 실행되는 **사용자 정의 셸 명령**이다.

Claude가 파일을 저장할 때, 응답을 완료할 때, 세션을 시작할 때 등 — 정해진 이벤트 시점에 여러분이 직접 작성한 명령이 자동으로 실행된다.

---

## LLM이 아닌 결정적(Deterministic) 자동화

훅의 핵심 특성은 **결정적**이라는 것이다.

| 구분 | 설명 |
|------|------|
| Claude의 일반 행동 | 확률적. 같은 요청도 매번 다른 결과 가능 |
| 훅 | 결정적. 조건이 충족되면 항상 동일하게 실행 |

즉, 훅은 LLM 판단을 거치지 않는다. 이벤트가 발생하면 설정된 명령이 **무조건** 실행된다.

---

## 훅이 필요한 이유

아래 두 접근 방식의 차이를 이해하면 훅의 가치가 명확해진다.

### 방식 A — Claude에게 부탁하기
> "파일 수정하고 나서 Prettier 실행해줘"

- Claude가 깜빡할 수 있다
- Claude가 포맷팅 명령을 잘못 실행할 수 있다
- 매번 상기시켜야 한다
- 프롬프트 컨텍스트를 소비한다

### 방식 B — 훅으로 강제하기
> PostToolUse 훅: Write 도구 실행 후 → `npx prettier --write $FILE_PATH`

- Claude가 파일을 저장하는 순간 **자동으로** 포맷터가 실행된다
- Claude가 모르는 사이에도 실행된다
- 한 번 설정하면 영원히 작동한다
- 컨텍스트를 소비하지 않는다

**"Claude에게 X를 해달라"** 는 요청이고, **"Claude가 Y를 할 때마다 반드시 X를 실행"** 은 보장이다.

훅은 보장을 제공한다.

---

## 훅 설정 위치

훅은 `settings.json`의 `hooks` 블록에 정의한다.

설정 파일 경로:
- **프로젝트 범위**: `.claude/settings.json`
- **사용자 범위**: `~/.claude/settings.json`

```bash
# 설정 파일 열기 (프로젝트)
cat .claude/settings.json

# Claude Code에서 설정 열기
# /config 명령어 입력
```

---

## 훅 기본 구조

```json
{
  "hooks": {
    "이벤트명": [
      {
        "matcher": "도구명 패턴",
        "hooks": [
          {
            "type": "command",
            "command": "실행할 명령"
          }
        ]
      }
    ]
  }
}
```

### 구조 분해

```json
{
  "hooks": {
    "PostToolUse": [          // 1) 이벤트: 도구 실행 성공 후
      {
        "matcher": "Write",   // 2) 매처: Write 도구일 때만
        "hooks": [
          {
            "type": "command",                          // 3) 유형: 셸 명령
            "command": "npx prettier --write $FILE_PATH" // 4) 명령: 포맷터 실행
          }
        ]
      }
    ]
  }
}
```

각 레이어의 역할:
1. **이벤트명** — 언제 실행할지 (PostToolUse, Stop, SessionStart 등)
2. **matcher** — 어떤 도구/조건일 때만 실행할지
3. **type** — 훅 유형 (command, prompt, agent, http)
4. **command** — 실제로 실행할 내용

---

## matcher 상세 설명

`matcher`는 **정규표현식(RegEx)**으로 특정 도구만 필터링한다.

```json
// Write 도구만 매칭
"matcher": "Write"

// Write 또는 Edit 도구 매칭
"matcher": "Write|Edit"

// Bash 도구만 매칭
"matcher": "Bash"

// 모든 도구 매칭 (빈 문자열)
"matcher": ""

// 파일 이름으로 필터링 (환경 변수 포함)
"matcher": "Write"
```

matcher가 비어 있으면 해당 이벤트의 모든 도구 호출에 반응한다.

### 여러 매처 사용

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write",
        "hooks": [{ "type": "command", "command": "echo 'JS 파일 저장됨'" }]
      },
      {
        "matcher": "Edit",
        "hooks": [{ "type": "command", "command": "echo 'Edit 실행됨'" }]
      }
    ]
  }
}
```

## 훅 실행 순서

동일 이벤트에 여러 훅이 등록된 경우 **배열 순서대로** 순차 실행된다.

```json
{
  "hooks": {
    "PostToolUse": [
      { "matcher": "Write", "hooks": [{ "type": "command", "command": "echo '1번 실행'" }] },
      { "matcher": "Write", "hooks": [{ "type": "command", "command": "echo '2번 실행'" }] },
      { "matcher": "Write", "hooks": [{ "type": "command", "command": "echo '3번 실행'" }] }
    ]
  }
}
```

실행 순서: 1번 → 2번 → 3번

단, `PreToolUse`에서 `exit 2`로 차단한 경우 이후 훅은 실행되지 않는다.

## 실습: 첫 번째 훅 만들어보기

### 목표
파일이 저장될 때마다 로그를 기록하는 훅 만들기

### 1단계 — 설정 파일 생성

프로젝트 루트에서:

```bash
mkdir -p .claude
```

`.claude/settings.json` 파일 생성:

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "echo \"[$(date '+%Y-%m-%d %H:%M:%S')] 파일 수정됨\" >> .claude/hook-log.txt"
          }
        ]
      }
    ]
  }
}
```

### 2단계 — 동작 확인

Claude Code에서 파일을 수정한 뒤:

```bash
cat .claude/hook-log.txt
# [2026-03-23 14:30:00] 파일 수정됨
# [2026-03-23 14:30:15] 파일 수정됨
```

### 3단계 — 환경 변수 추가

어떤 파일이 수정됐는지 기록하려면 `$FILE_PATH` 환경 변수를 활용한다:

```json
{
  "type": "command",
  "command": "echo \"[$(date '+%Y-%m-%d %H:%M:%S')] $FILE_PATH\" >> .claude/hook-log.txt"
}
```

이제 어떤 파일이 언제 수정됐는지 로그에 남는다.

---

## 핵심 요약

- 훅은 Claude Code 이벤트에 반응하는 **자동화 명령**이다
- LLM 판단 없이 **항상 동일하게** 실행된다
- `settings.json`의 `hooks` 블록에 정의한다
- `matcher`로 특정 도구만 선택적으로 처리할 수 있다
- 배열 순서대로 순차 실행된다
