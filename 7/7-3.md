# 7-3. 훅 유형

## 4가지 훅 유형 개요

훅은 실행 방식에 따라 4가지 유형으로 나뉜다.

| 유형 | 설명 | 주요 용도 |
|------|------|-----------|
| `command` | 셸 명령 실행 | 포맷팅, 테스트, 파일 조작 |
| `prompt` | Claude 모델에 판단 위임 | 안전성 검사, 정책 결정 |
| `agent` | 서브에이전트 실행 | 복잡한 검증, 멀티스텝 작업 |
| `http` | 외부 HTTP 엔드포인트 호출 | 웹훅, CI/CD 연동, 알림 |

---

## 1. command — 셸 명령

가장 기본적이고 많이 사용되는 유형이다. 운영체제의 셸 명령을 직접 실행한다.

### 기본 구조

```json
{
  "type": "command",
  "command": "npx prettier --write $FILE_PATH",
  "timeout": 30000
}
```

### 옵션 설명

| 필드 | 필수 | 설명 |
|------|------|------|
| `type` | 필수 | `"command"` |
| `command` | 필수 | 실행할 셸 명령 |
| `timeout` | 선택 | 타임아웃 (밀리초, 기본 30000) |

### stdin/stdout 데이터 교환

`command` 훅은 Claude와 stdin/stdout을 통해 데이터를 교환할 수 있다.

**stdin으로 입력 받기:**
Claude는 훅 실행 시 관련 데이터를 stdin으로 전달한다.

```bash
#!/bin/bash
# stdin에서 도구 정보 읽기
INPUT=$(cat)
echo "받은 데이터: $INPUT" >> .claude/debug.txt
```

**stdout으로 출력:**
stdout에 출력한 내용은 Claude의 컨텍스트에 포함될 수 있다.

```json
{
  "type": "command",
  "command": "echo '린트 검사 완료: 오류 없음'"
}
```

### exit 코드로 차단 제어

`PreToolUse` 이벤트에서만 동작한다.

```bash
#!/bin/bash
# protect-env.sh

if [[ "$FILE_PATH" == *".env"* ]]; then
    echo ".env 파일 수정은 허용되지 않습니다."
    exit 2   # 도구 실행 차단
fi

exit 0   # 정상 진행
```

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "bash .claude/protect-env.sh",
            "timeout": 5000
          }
        ]
      }
    ]
  }
}
```

### 실용 예제

```json
{
  "type": "command",
  "command": "npx eslint --fix \"$FILE_PATH\" 2>/dev/null; exit 0"
}
```

오류가 발생해도 `exit 0`으로 강제 종료해 훅 실패가 워크플로를 막지 않도록 한다.

## 2. prompt — Claude 모델 판단 위임

또 다른 Claude 모델 인스턴스에게 특정 판단을 맡기는 유형이다.

### 기본 구조

```json
{
  "type": "prompt",
  "prompt": "변경사항이 보안 취약점을 유발하는지 확인해줘",
  "model": "haiku"
}
```

### 옵션 설명

| 필드 | 필수 | 설명 |
|------|------|------|
| `type` | 필수 | `"prompt"` |
| `prompt` | 필수 | Claude에게 전달할 질문 |
| `model` | 선택 | 사용할 모델 (`haiku`, `sonnet`, `opus`) |
| `timeout` | 선택 | 타임아웃 (밀리초) |

### 모델 선택 지침

간단한 체크에는 속도가 빠르고 비용이 저렴한 `haiku` 사용을 권장한다.

```json
{
  "type": "prompt",
  "prompt": "이 코드 변경이 SQL 인젝션 취약점을 포함하는지 yes/no로만 답해줘",
  "model": "haiku"
}
```

복잡한 판단이 필요할 때만 `sonnet` 또는 `opus`를 사용한다.

### 활용 예제

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "prompt",
            "prompt": "방금 저장된 코드에 명백한 보안 문제가 있으면 지적해줘. 없으면 '문제 없음'이라고만 답해줘.",
            "model": "haiku"
          }
        ]
      }
    ]
  }
}
```

## 3. agent — 서브에이전트 실행

독립된 Claude 서브에이전트를 실행해 복잡한 작업을 처리하는 유형이다.

### 기본 구조

```json
{
  "type": "agent",
  "prompt": "변경된 파일과 관련된 테스트를 실행하고 결과를 보고해줘",
  "timeout": 120000
}
```

### 옵션 설명

| 필드 | 필수 | 설명 |
|------|------|------|
| `type` | 필수 | `"agent"` |
| `prompt` | 필수 | 서브에이전트에게 전달할 지시 |
| `timeout` | 선택 | 타임아웃 (밀리초, 기본 120000) |
| `model` | 선택 | 서브에이전트 모델 |

### command와의 차이

| 구분 | command | agent |
|------|---------|-------|
| 실행 주체 | 운영체제 | Claude 서브에이전트 |
| 도구 사용 | 불가 | 가능 (Read, Bash 등) |
| 복잡도 | 단순 명령 | 멀티스텝 작업 |
| 비용 | 없음 | API 호출 비용 발생 |

### 활용 예제 — 자동 테스트 실행

```json
{
  "hooks": {
    "Stop": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "agent",
            "prompt": "방금 수정된 파일들의 관련 테스트를 찾아서 실행하고, 실패한 테스트가 있으면 어떤 테스트가 왜 실패했는지 요약해줘.",
            "timeout": 120000
          }
        ]
      }
    ]
  }
}
```

### 주의사항

- 서브에이전트는 **별도 컨텍스트**에서 실행된다 (메인 대화 내용 미공유)
- 비용이 발생하므로 꼭 필요한 경우에만 사용한다
- 타임아웃을 충분히 설정한다 (복잡한 작업은 60초 이상)

## 4. http — 외부 HTTP 엔드포인트 호출

외부 웹 서비스나 API에 이벤트를 전송하는 유형이다.

### 기본 구조

```json
{
  "type": "http",
  "url": "http://localhost:8080/webhook",
  "method": "POST",
  "headers": {
    "Content-Type": "application/json",
    "Authorization": "Bearer $MY_TOKEN"
  },
  "body": "{\"event\": \"$TOOL_NAME\", \"file\": \"$FILE_PATH\"}"
}
```

### 옵션 설명

| 필드 | 필수 | 설명 |
|------|------|------|
| `type` | 필수 | `"http"` |
| `url` | 필수 | 요청을 보낼 URL |
| `method` | 선택 | HTTP 메서드 (기본 `POST`) |
| `headers` | 선택 | 요청 헤더 |
| `body` | 선택 | 요청 본문 |
| `timeout` | 선택 | 타임아웃 (밀리초) |

### 활용 예제 — Slack 웹훅

```json
{
  "hooks": {
    "Stop": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "http",
            "url": "https://hooks.slack.com/services/YOUR/WEBHOOK/URL",
            "method": "POST",
            "headers": {
              "Content-Type": "application/json"
            },
            "body": "{\"text\": \"Claude 작업 완료 — 확인해주세요\"}"
          }
        ]
      }
    ]
  }
}
```

### 활용 예제 — 로컬 로깅 서버

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "http",
            "url": "http://localhost:3000/log",
            "method": "POST",
            "headers": { "Content-Type": "application/json" },
            "body": "{\"tool\": \"$TOOL_NAME\", \"file\": \"$FILE_PATH\", \"time\": \"$(date)\"}"
          }
        ]
      }
    ]
  }
}
```

### CI/CD 연동 예제

```json
{
  "hooks": {
    "Stop": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "http",
            "url": "https://api.github.com/repos/OWNER/REPO/dispatches",
            "method": "POST",
            "headers": {
              "Authorization": "Bearer $GITHUB_TOKEN",
              "Content-Type": "application/json"
            },
            "body": "{\"event_type\": \"claude-code-complete\"}"
          }
        ]
      }
    ]
  }
}
```

## 4가지 유형 비교 표

| 항목 | command | prompt | agent | http |
|------|---------|--------|-------|------|
| 실행 속도 | 빠름 | 보통 | 느림 | 빠름 |
| API 비용 | 없음 | 있음 | 있음 | 없음 |
| 복잡도 | 낮음 | 중간 | 높음 | 낮음 |
| 도구 사용 | 불가 | 불가 | 가능 | 불가 |
| 인터넷 필요 | 불필요 | 필요 | 필요 | 필요 |
| 차단 기능 | 가능 | 가능 | 가능 | 불가 |
| 주요 용도 | 포맷팅/린팅 | 안전성 검사 | 복잡한 검증 | 외부 알림 |

## 실습: 4가지 유형 조합하기

아래는 파일 저장 시 여러 유형을 조합한 완전한 예제다.

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "npx prettier --write \"$FILE_PATH\" 2>/dev/null; exit 0",
            "timeout": 15000
          },
          {
            "type": "http",
            "url": "http://localhost:4000/track",
            "method": "POST",
            "headers": { "Content-Type": "application/json" },
            "body": "{\"file\": \"$FILE_PATH\", \"action\": \"saved\"}"
          }
        ]
      }
    ],
    "Stop": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "echo '작업 완료' | wall 2>/dev/null; exit 0"
          }
        ]
      }
    ]
  }
}
```

### 테스트 방법

1. `.claude/settings.json`에 설정 저장
2. 로컬 추적 서버 실행 (선택):
   ```bash
   python3 -c "
   from http.server import HTTPServer, BaseHTTPRequestHandler
   class H(BaseHTTPRequestHandler):
       def do_POST(self):
           print('추적 수신:', self.headers.get('Content-Length'), 'bytes')
           self.send_response(200); self.end_headers()
   HTTPServer(('', 4000), H).serve_forever()
   "
   ```
3. Claude Code에서 파일 수정
4. 각 훅이 순서대로 실행되는지 확인

---

## 핵심 요약

- **command**: 가장 일반적. 포맷팅, 린팅, 파일 조작에 사용
- **prompt**: Claude에게 판단 위임. 단순 체크에는 haiku 사용
- **agent**: 복잡한 멀티스텝 작업. 비용 발생 주의
- **http**: 외부 서비스 연동. Slack, CI/CD, 로깅 서버 등에 활용
- 하나의 이벤트에 여러 유형을 조합할 수 있다
