# 6-1. 설정 계층 구조

Claude Code의 설정은 여러 파일에 분산되어 있으며, 위치에 따라 적용 범위와 우선순위가 다릅니다. 이 계층 구조를 이해하면 개인 설정과 팀 공유 설정을 효과적으로 분리할 수 있습니다.

---

## settings.json 파일 위치

### 1. 사용자 수준 — 모든 프로젝트에 적용

```
~/.claude/settings.json
```

개인 전역 설정입니다. 어떤 프로젝트를 열어도 항상 적용됩니다. 개인 선호 사항(기본 모델, 개인 MCP 서버 등)을 설정합니다.

### 2. 프로젝트 수준 — 해당 프로젝트에 적용

```
<프로젝트 루트>/.claude/settings.json
```

프로젝트 전용 설정입니다. Git에 포함하여 팀 전체가 같은 설정을 공유합니다. 프로젝트의 허용 명령어, MCP 서버, 훅 등을 정의합니다.

### 3. 로컬 전용 — 개인 로컬 설정

```
<프로젝트 루트>/.claude/settings.local.json
```

프로젝트 설정과 같은 위치이지만 `.gitignore`에 추가하여 팀과 공유하지 않는 개인 설정입니다. 로컬 API 키, 개인 환경 변수, 로컬 경로 등을 저장합니다.

```bash
# .gitignore에 추가
echo ".claude/settings.local.json" >> .gitignore
```

### 4. 관리 설정 — 조직 IT 배포

```
# macOS / Linux
/etc/claude/settings.json

# Windows
C:\ProgramData\claude\settings.json
```

기업 환경에서 IT 부서가 모든 사용자 머신에 배포하는 설정입니다. 보안 정책(특정 도구 금지, 네트워크 제한 등)을 강제합니다.

---

## 우선순위

낮은 번호가 높은 우선순위를 가집니다. 충돌이 생기면 높은 우선순위의 설정이 적용됩니다.

```
1순위 (최강): Managed (관리 설정)
2순위:        CLI 플래그 (실행 시 직접 지정)
3순위:        Local (.claude/settings.local.json)
4순위:        Project (.claude/settings.json)
5순위 (최약): User (~/.claude/settings.json)
```

### 우선순위 예시

```
~/.claude/settings.json
  defaultMode: "acceptEdits"

.claude/settings.json
  defaultMode: "plan"

→ 적용 결과: "plan" (프로젝트 설정이 우선)
```

```
.claude/settings.json
  env.NODE_ENV: "production"

.claude/settings.local.json
  env.NODE_ENV: "development"

→ 적용 결과: "development" (로컬 설정이 프로젝트보다 우선)
```

## 충돌 해결 방식

설정 충돌은 키별로 다르게 처리됩니다.

### 단순 값 — 높은 우선순위가 덮어씀

```json
// User settings.json
{ "defaultModel": "claude-haiku-3-5" }

// Project settings.json
{ "defaultModel": "claude-sonnet-4-5" }

// 결과: claude-sonnet-4-5
```

### permissions — allow/deny 목록은 병합됨

```json
// User settings.json
{
  "permissions": {
    "allow": ["Bash(git *)"]
  }
}

// Project settings.json
{
  "permissions": {
    "allow": ["Bash(npm *)"]
  }
}

// 결과: 두 규칙 모두 적용 (git * AND npm * 허용)
```

단, 같은 레벨에서 충돌 시 `deny`가 `allow`보다 우선합니다.

```json
{
  "permissions": {
    "allow": ["Bash(npm *)"],
    "deny": ["Bash(npm publish)"]
  }
}
// npm publish는 npm * 범위에 포함되지만 deny에 있으므로 거부됨
```

## 설정 파일 기본 구조

```json
{
  "permissions": {
    "allow": [],
    "deny": [],
    "ask": []
  },
  "hooks": {
    "PreToolUse": [],
    "PostToolUse": [],
    "Notification": []
  },
  "env": {},
  "mcpServers": {},
  "defaultMode": "default",
  "defaultModel": "claude-sonnet-4-5"
}
```

| 키 | 역할 |
|----|------|
| `permissions` | 도구 사용 허용/거부 규칙 |
| `hooks` | 이벤트 기반 자동 실행 스크립트 |
| `env` | 환경 변수 설정 |
| `mcpServers` | MCP 서버 연결 설정 |
| `defaultMode` | 기본 권한 모드 |
| `defaultModel` | 기본 AI 모델 |

## `/config` 커맨드로 설정 열기

Claude Code 안에서 설정 파일을 바로 열 수 있습니다.

```
/config
```

선택 메뉴가 표시됩니다.

```
설정 파일 선택:
1. 사용자 설정  (~/.claude/settings.json)
2. 프로젝트 설정 (.claude/settings.json)
3. 로컬 설정    (.claude/settings.local.json)
```

선택하면 기본 에디터로 해당 파일이 열립니다. 저장 후 Claude Code를 재시작하지 않아도 변경 사항이 적용됩니다.

## 설정 파일 생성

파일이 없는 경우 직접 만듭니다.

```bash
# 프로젝트 설정 초기화
mkdir -p .claude
touch .claude/settings.json

# 최소 구조로 시작
cat > .claude/settings.json << 'EOF'
{
  "permissions": {
    "allow": [],
    "deny": []
  }
}
EOF
```

## 실습: 프로젝트용 settings.json 만들기

### 1단계: 디렉토리 생성

```bash
mkdir -p .claude
```

### 2단계: 프로젝트 설정 파일 작성

아래 내용으로 `.claude/settings.json`을 만듭니다.

```json
{
  "permissions": {
    "allow": [
      "Bash(npm run *)",
      "Bash(git status)",
      "Bash(git diff *)",
      "Bash(git log *)",
      "Bash(git add *)",
      "Bash(git commit *)"
    ],
    "deny": [
      "Bash(git push --force*)",
      "Bash(rm -rf *)",
      "Edit(.env)"
    ]
  },
  "env": {
    "NODE_ENV": "development"
  },
  "defaultMode": "acceptEdits"
}
```

### 3단계: 로컬 전용 설정 파일 작성

`.claude/settings.local.json` (gitignore에 추가):

```json
{
  "env": {
    "DATABASE_URL": "postgresql://localhost:5432/mydb_dev",
    "REDIS_URL": "redis://localhost:6379"
  }
}
```

```bash
echo ".claude/settings.local.json" >> .gitignore
```

### 4단계: 적용 확인

```
/permissions
```

두 파일에서 병합된 권한 규칙이 표시되는지 확인합니다.
