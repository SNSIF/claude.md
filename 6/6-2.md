# 6-2. 권한 설정

Claude Code는 파일 편집, 명령 실행, 웹 요청 등 강력한 작업을 수행합니다. 권한 설정은 어떤 작업을 자동으로 허용하고, 어떤 작업에 승인을 요청하며, 어떤 작업을 완전히 금지할지 정의합니다.

---

## 권한 설정 개요

권한 설정이 없으면 Claude Code는 민감한 작업마다 승인을 요청합니다. 권한을 적절히 설정하면 반복적인 승인 클릭을 줄이면서도 위험한 작업을 차단할 수 있습니다.

```json
{
  "permissions": {
    "allow": [...],
    "deny": [...],
    "ask": [...]
  }
}
```

| 블록 | 동작 |
|------|------|
| `allow` | 승인 없이 자동 실행 |
| `deny` | 실행 자체를 차단 |
| `ask` | 실행 전 반드시 승인 요청 (기본 동작) |

처리 순서: `deny` → `allow` → `ask`(기본)

같은 명령이 `allow`와 `deny`에 모두 매칭되면 `deny`가 우선합니다.

---

## 권한 규칙 문법

### 도구 전체 허용/거부

```json
"allow": ["Read"]
```

`Read` 도구의 모든 파일 읽기를 허용합니다.

```json
"deny": ["WebSearch"]
```

웹 검색 자체를 금지합니다.

---

### 와일드카드 패턴

```json
"allow": ["Bash(npm run *)"]
```

`npm run dev`, `npm run build`, `npm run test` 등 `npm run`으로 시작하는 모든 명령을 허용합니다.

```json
"deny": ["Bash(rm *)"]
```

`rm`으로 시작하는 모든 명령을 차단합니다.

---

### 정확한 명령

```json
"allow": ["Bash(npm test)"]
```

`npm test`와 정확히 일치하는 명령만 허용합니다. `npm test --watch`는 매칭되지 않습니다.

---

### 특정 파일 제한

```json
"deny": ["Edit(.env)"]
```

`.env` 파일 편집을 차단합니다.

```json
"allow": ["Read(package.json)"]
```

`package.json` 읽기만 허용합니다 (다른 파일은 기본값 적용).

---

### 도메인 필터

```json
"allow": ["WebFetch(domain:github.com)"]
```

`github.com` 도메인에서만 웹 가져오기를 허용합니다.

```json
"deny": ["WebFetch(domain:internal.company.com)"]
```

내부 서버 접근을 차단합니다.

---

### MCP 도구 패턴

```json
"allow": ["mcp__github__*"]
```

GitHub MCP 서버의 모든 도구를 허용합니다.

```json
"allow": ["mcp__github__create_pr", "mcp__github__list_prs"]
```

GitHub MCP의 특정 도구만 허용합니다.

---

## 파일 경로 패턴

파일 경로 지정 방식은 세 가지입니다.

### 상대 경로

```json
"deny": ["Read(*.env)", "Edit(*.env*)"]
```

현재 프로젝트 루트 기준 상대 경로입니다. `.env`, `.env.local`, `.env.production` 등이 매칭됩니다.

```json
"allow": ["Read(src/**/*.ts)"]
```

`src` 디렉토리 아래 모든 `.ts` 파일 읽기를 허용합니다.

### 절대 경로

```json
"deny": ["Edit(//home/user/config/nginx.conf)"]
```

이중 슬래시(`//`)로 시작하면 절대 경로입니다.

### 홈 디렉토리

```json
"allow": ["Read(~/.zshrc)"]
```

`~`은 홈 디렉토리를 나타냅니다.

## 실전 예제: 안전한 기본 설정

### 일반 웹 프로젝트

```json
{
  "permissions": {
    "allow": [
      "Bash(npm run dev)",
      "Bash(npm run build)",
      "Bash(npm test)",
      "Bash(npm run lint)",
      "Bash(npm run lint:fix)",
      "Bash(git status)",
      "Bash(git diff *)",
      "Bash(git log *)",
      "Bash(git add *)",
      "Bash(git commit *)",
      "Bash(git branch *)",
      "Bash(git checkout *)",
      "Bash(git stash *)"
    ],
    "deny": [
      "Bash(git push --force*)",
      "Bash(git reset --hard*)",
      "Bash(rm -rf *)",
      "Bash(sudo *)",
      "Edit(.env)",
      "Edit(.env.production)",
      "Edit(.env.local)"
    ]
  }
}
```

### FastAPI 백엔드 프로젝트

```json
{
  "permissions": {
    "allow": [
      "Bash(uvicorn *)",
      "Bash(python -m pytest *)",
      "Bash(alembic *)",
      "Bash(pip install *)",
      "Bash(black *)",
      "Bash(isort *)",
      "Bash(mypy *)",
      "Bash(git *)"
    ],
    "deny": [
      "Bash(git push --force*)",
      "Bash(rm -rf *)",
      "Bash(DROP *)",
      "Edit(.env)",
      "Edit(alembic.ini)"
    ],
    "ask": [
      "Bash(alembic upgrade head)",
      "Bash(alembic downgrade *)"
    ]
  }
}
```

`ask` 블록에 마이그레이션 명령을 넣어 실행 전 반드시 확인을 받습니다.

### CI/CD 자동화 환경

```json
{
  "permissions": {
    "allow": [
      "Bash(*)",
      "Read(*)",
      "Write(*)",
      "Edit(*)"
    ]
  },
  "defaultMode": "bypassPermissions"
}
```

자동화 파이프라인에서는 모든 권한을 허용하고 `bypassPermissions` 모드를 사용합니다.

## 권한 규칙 우선순위 예시

복잡한 규칙이 겹칠 때 동작 방식을 이해합니다.

```json
{
  "permissions": {
    "allow": ["Bash(npm *)"],
    "deny": ["Bash(npm publish)", "Bash(npm version *)"]
  }
}
```

- `npm test` → allow 매칭 → 자동 실행
- `npm run dev` → allow 매칭 → 자동 실행
- `npm publish` → allow와 deny 모두 매칭 → deny 우선 → 차단
- `npm version patch` → deny 매칭 → 차단
- `git status` → 어디도 매칭 안 됨 → ask (승인 요청)

## `/permissions` UI에서 확인

설정 파일을 직접 편집하는 대신 UI로 관리할 수 있습니다.

```
/permissions
```

현재 적용된 모든 권한 규칙이 소스(User/Project/Local)와 함께 표시됩니다. 충돌 규칙도 시각적으로 확인할 수 있습니다.
