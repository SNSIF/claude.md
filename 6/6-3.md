# 6-3. 환경 변수

Claude Code는 settings.json의 `env` 블록을 통해 환경 변수를 설정하고, 이를 실행되는 도구(Bash 명령, MCP 서버 등)에 자동으로 전달합니다. 민감한 정보는 로컬 전용 파일에 분리하는 방식을 사용합니다.

---

## 기본 문법

```json
{
  "env": {
    "NODE_ENV": "development",
    "API_BASE_URL": "http://localhost:3000",
    "DEBUG": "true",
    "LOG_LEVEL": "info"
  }
}
```

`env` 블록에 키-값 쌍으로 정의합니다. 값은 모두 문자열입니다. 숫자나 불리언을 사용하고 싶어도 따옴표로 감싸야 합니다.

Claude Code가 `Bash` 도구로 명령을 실행할 때 이 환경 변수들이 자동으로 주입됩니다.

```bash
# settings.json에 NODE_ENV: "development"가 설정된 경우
node -e "console.log(process.env.NODE_ENV)"
# 출력: development
```

---

## 외부 변수 참조

시스템 환경 변수 값을 settings.json에서 참조할 수 있습니다. `"$변수명"` 형식을 사용합니다.

```json
{
  "env": {
    "GITHUB_TOKEN": "$MY_GITHUB_TOKEN",
    "AWS_PROFILE": "$AWS_DEFAULT_PROFILE",
    "HOME_DIR": "$HOME"
  }
}
```

`MY_GITHUB_TOKEN`이 시스템에 설정되어 있으면 그 값을 읽어 `GITHUB_TOKEN`에 전달합니다. settings.json 파일 자체에 실제 토큰값이 저장되지 않으므로 안전합니다.

참조한 변수가 시스템에 없으면 빈 문자열로 처리됩니다.

---

## 범위별 환경 변수

설정 계층 구조에 따라 환경 변수도 범위별로 분리됩니다.

### 사용자 전역 (`~/.claude/settings.json`)

모든 프로젝트에 공통으로 적용됩니다.

```json
{
  "env": {
    "EDITOR": "code",
    "PAGER": "cat",
    "LANG": "ko_KR.UTF-8"
  }
}
```

### 프로젝트 (`<프로젝트>/.claude/settings.json`)

해당 프로젝트에서만 적용됩니다. Git에 포함하여 팀과 공유합니다.

```json
{
  "env": {
    "NODE_ENV": "development",
    "API_BASE_URL": "http://localhost:3000",
    "POSTGRES_DB": "myapp_dev"
  }
}
```

### 로컬 전용 (`<프로젝트>/.claude/settings.local.json`)

개인 로컬 환경에만 적용됩니다. `.gitignore`에 추가하여 공유하지 않습니다.

```json
{
  "env": {
    "DATABASE_URL": "postgresql://john:password@localhost:5432/myapp_dev",
    "REDIS_URL": "redis://localhost:6379",
    "STRIPE_SECRET_KEY": "sk_test_abc123..."
  }
}
```

## 민감한 값 처리

API 키, 데이터베이스 비밀번호 등 민감한 정보를 안전하게 관리하는 방법입니다.

### 원칙

1. 민감한 값은 `.local.json`에만 저장한다
2. `.local.json`은 반드시 `.gitignore`에 추가한다
3. 팀 공유 설정에는 플레이스홀더나 외부 변수 참조를 사용한다

### 팀 공유용 설정 예시

```json
// .claude/settings.json (Git에 포함)
{
  "env": {
    "NODE_ENV": "development",
    "API_BASE_URL": "http://localhost:3000",
    "DATABASE_URL": "$DATABASE_URL"
  }
}
```

```json
// .claude/settings.local.json (Git 제외)
{
  "env": {
    "DATABASE_URL": "postgresql://dev_user:dev_pass@localhost:5432/myapp"
  }
}
```

로컬 설정이 프로젝트 설정보다 우선순위가 높으므로 `DATABASE_URL`은 로컬 값으로 대체됩니다.

### .gitignore 설정

```bash
# .gitignore에 추가
echo ".claude/settings.local.json" >> .gitignore

# 이미 추적 중인 경우 제거
git rm --cached .claude/settings.local.json
```

## 주요 Claude Code 환경 변수

Claude Code 자체의 동작을 제어하는 내장 환경 변수입니다.

### `CLAUDE_CODE_DEBUG`

디버그 로그를 활성화합니다. 내부 API 요청, 도구 실행 세부 정보가 출력됩니다.

```json
{
  "env": {
    "CLAUDE_CODE_DEBUG": "true"
  }
}
```

또는 실행 시:

```bash
CLAUDE_CODE_DEBUG=true claude
```

---

### `CLAUDE_CODE_DISABLE_AUTO_MEMORY`

자동 메모리 저장을 비활성화합니다.

```json
{
  "env": {
    "CLAUDE_CODE_DISABLE_AUTO_MEMORY": "true"
  }
}
```

CI 환경이나 민감한 정보를 다루는 세션에서 유용합니다.

---

### `CLAUDE_AUTOCOMPACT_PCT_OVERRIDE`

자동 컨텍스트 압축 임계값을 설정합니다. 기본값은 80(%)이며, 이 비율에 도달하면 자동으로 `/compact`를 실행합니다.

```json
{
  "env": {
    "CLAUDE_AUTOCOMPACT_PCT_OVERRIDE": "70"
  }
}
```

낮은 값일수록 더 자주 압축합니다. 토큰 비용을 최소화하려면 값을 낮게 설정합니다.

---

### `MAX_THINKING_TOKENS`

Extended Thinking 기능에서 사용할 최대 토큰 수를 제한합니다.

```json
{
  "env": {
    "MAX_THINKING_TOKENS": "10000"
  }
}
```

복잡한 분석에는 높게, 비용 절약이 중요하면 낮게 설정합니다.

---

## 환경 변수 우선순위

```
1순위: 시스템 환경 변수 (터미널에서 export한 값)
2순위: settings.local.json의 env 블록
3순위: settings.json (프로젝트)의 env 블록
4순위: settings.json (사용자)의 env 블록
```

시스템 환경 변수가 항상 최우선입니다. 개발 중 일시적으로 다른 값을 테스트하려면 터미널에서 직접 `export`하면 됩니다.

```bash
# 일시적으로 프로덕션 DB에 연결 (settings.json보다 우선)
DATABASE_URL="postgresql://prod-host:5432/myapp" claude
```

## 실습

### 실습 1: 기본 환경 변수 설정

프로젝트 settings.json에 개발 환경 변수를 추가합니다.

```json
{
  "env": {
    "NODE_ENV": "development",
    "PORT": "3000",
    "LOG_LEVEL": "debug"
  }
}
```

적용 확인:

```
NODE_ENV 환경 변수 값을 출력하는 Node.js 명령 실행해줘
```

### 실습 2: 로컬 시크릿 분리

```bash
# settings.local.json 생성
cat > .claude/settings.local.json << 'EOF'
{
  "env": {
    "DATABASE_URL": "postgresql://localhost:5432/myapp_dev",
    "JWT_SECRET": "local-dev-secret-key"
  }
}
EOF

# gitignore 추가
echo ".claude/settings.local.json" >> .gitignore
```

### 실습 3: 외부 변수 참조

```bash
# 시스템 환경 변수 설정
export GITHUB_TOKEN="ghp_mytoken123"

# settings.json에서 참조
{
  "env": {
    "GITHUB_TOKEN": "$GITHUB_TOKEN"
  }
}
```

Claude Code에서 확인:

```
현재 GITHUB_TOKEN 환경 변수가 설정되어 있는지 확인해줘
```

### 실습 4: 자동 압축 임계값 조정

```json
{
  "env": {
    "CLAUDE_AUTOCOMPACT_PCT_OVERRIDE": "60"
  }
}
```

긴 대화를 진행하며 60% 도달 시 자동 압축이 실행되는지 확인합니다.
