# 6-4. 권한 모드 (Permission Modes)

Claude Code는 작업 상황에 맞게 4가지 권한 모드를 제공합니다. 권한 모드는 Claude가 파일 수정, 명령 실행, 도구 사용 시 사용자의 승인을 어느 수준까지 요구할지를 결정합니다.

---

## 4가지 권한 모드 상세

### 1. `default` — 기본 승인 모드

**개요**: 위험한 작업(파일 쓰기, 명령 실행 등)은 매번 사용자 승인을 요청합니다. 가장 안전한 기본값입니다.

**동작 방식**:
- 파일 읽기(Read, Grep, Glob): 자동 승인
- 파일 쓰기(Write, Edit): 매번 승인 요청
- 명령 실행(Bash): 매번 승인 요청
- 외부 서비스 호출: 매번 승인 요청

**적합한 상황**:
- 일반적인 개발 작업
- Claude에 처음 입문하거나 작업 내용을 꼼꼼히 확인하고 싶을 때
- 프로덕션 코드베이스 작업

**예시 흐름**:
```
사용자: "config.json 파일에 새 키를 추가해줘"

Claude: config.json을 수정하려 합니다.
        [수정 내용 미리보기]
        승인하시겠습니까? (y/n)
```

---

### 2. `acceptEdits` — 편집 자동 승인 모드

**개요**: 파일 편집(Edit/Write) 작업은 자동으로 승인하지만, 명령 실행(Bash) 등 나머지 위험 작업은 기본 모드처럼 승인을 요청합니다.

**동작 방식**:
- 파일 읽기: 자동 승인
- 파일 쓰기(Write, Edit): **자동 승인** (승인 창 없음)
- 명령 실행(Bash): 매번 승인 요청
- 외부 서비스 호출: 매번 승인 요청

**적합한 상황**:
- 리팩토링 등 파일 편집이 빈번한 작업
- 여러 파일을 한꺼번에 수정해야 하는 대규모 코드 변경
- 편집 내용을 신뢰하되 명령 실행은 여전히 확인하고 싶을 때

**예시 흐름**:
```
사용자: "모든 컴포넌트 파일에서 className을 class로 바꿔줘"

Claude: [파일 1 수정 완료 — 자동 승인]
        [파일 2 수정 완료 — 자동 승인]
        [파일 3 수정 완료 — 자동 승인]
        총 12개 파일 수정 완료
```

---

### 3. `plan` — 읽기 전용 / 계획 수립 모드

**개요**: 파일 수정과 명령 실행이 완전히 차단됩니다. 코드베이스를 탐색하고 계획을 수립할 때 사용합니다.

**동작 방식**:
- 파일 읽기(Read, Grep, Glob): 자동 승인
- 파일 쓰기(Write, Edit): **완전 차단** (실행 불가)
- 명령 실행(Bash): **완전 차단** (실행 불가)
- 외부 서비스 호출: **완전 차단** (실행 불가)

**적합한 상황**:
- 코드베이스 탐색 및 분석
- 구현 전 계획 초안 수립
- 코드 리뷰 (읽기만 하고 변경 없음)
- 신뢰하지 않는 프로젝트를 안전하게 탐색

**예시 흐름**:
```
사용자: "인증 모듈 개선 방법을 계획해줘"

Claude: [코드 탐색 중...]
        auth/ 폴더 구조를 분석했습니다.
        현재 문제점:
        1. JWT 만료 처리가 없음
        2. 리프레시 토큰 미구현

        개선 계획:
        - 1단계: refreshToken 테이블 추가
        - 2단계: /auth/refresh 엔드포인트 구현
        - 3단계: 클라이언트 인터셉터 추가

        (이 모드에서는 코드를 직접 수정하지 않습니다)
```

---

### 4. `bypassPermissions` — 모든 권한 확인 생략 모드

**개요**: 모든 권한 확인을 생략하고 Claude가 모든 작업을 자동으로 수행합니다. CI/CD 파이프라인이나 자동화 스크립트용입니다.

**동작 방식**:
- 파일 읽기: 자동 승인
- 파일 쓰기(Write, Edit): **자동 승인**
- 명령 실행(Bash): **자동 승인**
- 외부 서비스 호출: **자동 승인**

**적합한 상황**:
- CI/CD 파이프라인 자동화
- 스크립트 내 Claude API 활용
- 신뢰할 수 있는 격리된 환경에서의 배치 작업
- 반복적인 자동화 태스크

**예시 흐름 (CI 환경)**:
```bash
# GitHub Actions, Jenkins 등 CI 환경에서
claude --permission-mode bypassPermissions \
  "테스트 실패한 파일들 자동 수정하고 커밋해줘"
```

---

## 모드 전환 방법

### 방법 1: `Shift+Tab` — 대화 중 순환 전환

대화 중 언제든지 `Shift+Tab`을 누르면 모드가 순환 전환됩니다.

```
default → acceptEdits → plan → bypassPermissions → default → ...
```

현재 모드는 프롬프트 왼쪽에 아이콘으로 표시됩니다.

---

### 방법 2: `/permissions` — 권한 관리 UI

슬래시 커맨드로 권한 관리 인터페이스를 열 수 있습니다.

```
/permissions
```

실행하면 현재 권한 모드와 개별 도구별 허용/차단 상태를 시각적으로 확인하고 수정할 수 있습니다.

---

### 방법 3: `--permission-mode` CLI 플래그

Claude Code 실행 시 시작 모드를 지정합니다.

```bash
# plan 모드로 시작
claude --permission-mode plan

# acceptEdits 모드로 시작
claude --permission-mode acceptEdits

# bypassPermissions 모드로 시작 (CI 환경)
claude --permission-mode bypassPermissions "자동화 작업 내용"
```

---

### 방법 4: `settings.json` 기본값 설정

매번 지정하지 않고 기본 시작 모드를 고정할 수 있습니다.

**프로젝트별 설정** (`.claude/settings.json`):
```json
{
  "permissions": {
    "defaultMode": "acceptEdits"
  }
}
```

**전역 설정** (`~/.claude/settings.json`):
```json
{
  "permissions": {
    "defaultMode": "plan"
  }
}
```

---

## 모드별 사용 시나리오 요약

| 상황 | 권장 모드 | 이유 |
|------|-----------|------|
| 코드 탐색 / 리뷰만 | `plan` | 실수로 파일 변경되는 것 방지 |
| 파일 편집 작업이 많음 | `acceptEdits` | 매번 승인 없이 편집 속도 향상 |
| CI/CD 자동화 파이프라인 | `bypassPermissions` | 사람이 없으므로 자동 처리 필요 |
| 일반 개발 작업 | `default` | 가장 안전하고 권장되는 기본값 |
| 신뢰하지 않는 저장소 탐색 | `plan` | 코드가 실행되지 않도록 차단 |
| 대규모 리팩토링 | `acceptEdits` | 수십 개 파일 수정 시 편리 |

## 보안 고려사항: `bypassPermissions`의 위험성

`bypassPermissions` 모드는 강력하지만 신중하게 사용해야 합니다.

**위험 요소**:
```
1. 실수로 중요 파일 삭제 가능
2. 의도치 않은 외부 API 호출 (비용 발생)
3. 악성 CLAUDE.md가 있는 저장소에서 프롬프트 인젝션 공격 가능
4. DB 초기화, 환경 변수 노출 등 파괴적 명령 실행 가능
```

**안전한 사용 원칙**:
```bash
# 1. 격리된 환경(Docker, VM)에서만 사용
docker run --rm my-ci-image claude --permission-mode bypassPermissions "..."

# 2. 읽기 전용 마운트로 소스 코드 보호
docker run -v $(pwd):/workspace:ro ...

# 3. 네트워크 격리
docker run --network none ...

# 4. 항상 git으로 변경 사항 추적 가능하게 유지
git status && claude --permission-mode bypassPermissions "..."
```

**절대 사용하지 말아야 할 상황**:
- 프로덕션 서버에서 직접 실행
- 신뢰할 수 없는 외부 저장소에서 실행
- 중요 자격증명(AWS 키, DB 비밀번호 등)이 있는 환경

---

## settings.json 전체 예시

```json
{
  "permissions": {
    "defaultMode": "acceptEdits",
    "allow": [
      "Bash(git *)",
      "Bash(npm run *)",
      "Bash(python *)"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Bash(DROP TABLE *)"
    ]
  }
}
```

- `defaultMode`: 시작 시 기본 권한 모드
- `allow`: 특정 패턴의 명령은 항상 자동 승인
- `deny`: 특정 패턴의 명령은 항상 차단 (모드와 무관)

## 실습: 각 모드를 직접 전환해보기

### 실습 1: plan 모드로 코드 탐색

```bash
# plan 모드로 시작
claude --permission-mode plan

# 아래 명령을 입력해 보세요
> "src 폴더 구조를 분석해줘"
> "인증 관련 파일들을 찾아줘"
# 읽기만 하고 파일 변경 없이 분석만 수행됩니다
```

### 실습 2: Shift+Tab으로 모드 전환

```
1. claude 실행
2. "현재 권한 모드가 뭐야?" 라고 물어보기
3. Shift+Tab 눌러서 모드 변경
4. 다시 "현재 권한 모드가 뭐야?" 라고 물어보기
5. 모드가 바뀌었는지 확인
```

### 실습 3: settings.json으로 기본 모드 설정

```bash
# 프로젝트 루트에 .claude 디렉토리 생성
mkdir -p .claude

# settings.json 작성
cat > .claude/settings.json << 'EOF'
{
  "permissions": {
    "defaultMode": "acceptEdits"
  }
}
EOF

# 다시 claude 실행 — acceptEdits 모드로 시작되는지 확인
claude
```

### 실습 4: 모드 차이 체감하기

```
1. default 모드에서 "README.md 파일에 빈 줄 하나 추가해줘" 실행
   → 승인 요청 창이 뜨는지 확인

2. Shift+Tab으로 acceptEdits로 전환 후 동일 요청
   → 승인 없이 바로 수정되는지 확인

3. Shift+Tab으로 plan 모드로 전환 후 동일 요청
   → "이 모드에서는 파일을 수정할 수 없습니다" 메시지 확인
```
