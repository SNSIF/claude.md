# 2-4. 파일 검색 (Glob)

## Glob 도구 개요

Glob 도구는 파일명 패턴(와일드카드)을 사용해 파일을 찾는 도구입니다. 내용이 아닌 **파일 경로와 이름**을 기준으로 검색합니다.

**핵심 특징:**
- 와일드카드 패턴으로 파일명 매칭
- 결과는 **최근 수정 시간 순**으로 정렬
- 권한 없이 자동 실행 (읽기 전용)
- 재귀 검색(`**`) 지원

---

## 지원 패턴 상세

### `*` — 단일 디렉토리 내 와일드카드

현재 디렉토리의 파일만 매칭합니다 (하위 디렉토리 미포함).

```
*.py          → 현재 디렉토리의 모든 Python 파일
*.{js,ts}     → 현재 디렉토리의 JS 또는 TS 파일
test_*.py     → test_로 시작하는 Python 파일
*.config.js   → .config.js로 끝나는 파일
```

### `**` — 재귀 와일드카드

모든 하위 디렉토리를 포함해 매칭합니다.

```
**/*.ts           → 모든 하위 경로의 TypeScript 파일
src/**/*.tsx      → src 하위 모든 TSX 파일
**/*.test.js      → 모든 Jest 테스트 파일
**/node_modules   → node_modules 디렉토리들
```

### `{}` — 복수 확장자 또는 패턴

```
**/*.{js,ts}          → JS 또는 TS
**/*.{test,spec}.ts   → .test.ts 또는 .spec.ts
src/**/*.{jsx,tsx}    → React 컴포넌트
```

### 특정 위치 패턴

```
src/**/*.py           → src 하위 모든 Python 파일
tests/test_*.py       → tests 폴더의 test_로 시작하는 파일
**/migrations/*.sql   → 모든 마이그레이션 SQL 파일
config/*.{yaml,yml}   → config 폴더의 YAML 파일
```

---

## Glob vs Grep 비교

| 항목 | Glob | Grep |
|------|------|------|
| 검색 기준 | 파일명/경로 패턴 | 파일 내용 |
| 용도 | "*.py 파일이 어디에 있지?" | "이 함수가 어디서 쓰이지?" |
| 속도 | 빠름 | 상대적으로 느림 |
| 결과 | 파일 경로 목록 | 매칭된 라인 |

**Glob 사용 예:**
```
> 프로젝트에 있는 모든 설정 파일을 찾아줘 → **/*.{json,yaml,yml,toml}
> migrations 파일이 몇 개야? → **/migrations/*.py
```

**Grep 사용 예:**
```
> TODO 주석이 어디에 있어? → "TODO" 패턴으로 검색
> authenticate 함수가 어디서 호출돼? → "authenticate(" 패턴으로 검색
```

## Claude에게 Glob 요청하기

Claude는 자연어 요청을 Glob 패턴으로 변환해 실행합니다.

```
> 이 프로젝트의 모든 TypeScript 파일을 찾아줘
> 테스트 파일이 어디에 있어?
> src 폴더 아래 React 컴포넌트 파일들을 목록으로 보여줘
> __init__.py 파일들이 어디에 있어?
> 마이그레이션 파일들을 모두 찾아줘
```

## 실습 예제들

### 실습 1: 프로젝트 파일 구조 파악

```bash
cd ~/my-project
claude
```

```
> 이 프로젝트에 있는 파일 유형들을 파악해줘.
  Python, JavaScript, YAML 파일이 각각 몇 개야?
```

Claude가 내부적으로:
- `**/*.py` 실행
- `**/*.{js,ts}` 실행
- `**/*.{yaml,yml}` 실행
- 각 결과 수 집계 후 보고

### 실습 2: 테스트 파일 찾기

```
> 테스트 파일들을 모두 찾아줘. pytest 형식과 Jest 형식 둘 다.
```

Claude가 처리:
```
test_*.py, **/tests/*.py  → pytest
**/*.test.{js,ts}         → Jest
**/*.spec.{js,ts}         → Jest
```

### 실습 3: 설정 파일 목록

```
> 환경 설정과 관련된 파일들을 모두 찾아줘.
  (.env, config.*, settings.* 등)
```

### 실습 4: 특정 모듈의 파일들

```
> auth 관련 파일들을 모두 찾아줘.
  파일명에 'auth'가 포함된 것들.
```

Claude가 패턴 적용:
```
**/auth*.py
**/auth*.ts
**/*auth*.{py,ts,js}
```

### 실습 5: 최근 수정 파일

Glob 결과는 수정 시간 순으로 정렬되므로:

```
> 최근에 수정된 Python 파일 5개를 보여줘.
```

## 실제 패턴 예시 모음

```
# 프론트엔드
src/**/*.tsx          → React TypeScript 컴포넌트
src/**/*.module.css   → CSS 모듈 파일
public/**/*.svg       → SVG 에셋

# 백엔드
app/**/*.py           → 모든 Python 파일
**/models/*.py        → 모델 파일
**/routers/*.py       → 라우터 파일
**/migrations/V*.sql  → Flyway 마이그레이션

# 설정
**/.env*              → 환경변수 파일
**/*.config.{js,ts}   → 설정 파일
**/Dockerfile*        → Docker 파일
**/*.{yaml,yml}       → YAML 파일

# 테스트
tests/**/*.py         → Python 테스트
**/__tests__/**/*.ts  → TypeScript 테스트
**/*.{test,spec}.tsx  → React 컴포넌트 테스트

# 무시 패턴 (Glob 자체는 exclude 미지원, Grep의 --glob 제외 패턴 활용)
!**/node_modules/**   → node_modules 제외
!**/.git/**           → .git 제외
```

---

## 주의사항

- Glob은 파일 **내용을 읽지 않습니다**. 파일 목록만 반환합니다.
- `node_modules`, `.git` 등 대형 디렉토리를 포함하면 결과가 많아질 수 있습니다.
- 결과가 너무 많으면 더 구체적인 패턴을 사용하세요.
  ```
  # 너무 광범위
  **/*

  # 적절
  src/**/*.ts
  ```
