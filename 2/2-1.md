# 2-1. 파일 읽기 (Read)

## Read 도구 개요

Read 도구는 파일 시스템에서 파일 내용을 읽어 대화 컨텍스트에 포함시키는 도구입니다.

**핵심 특징:**
- **별도 승인 불필요** — 읽기 전용이므로 Claude가 자동으로 실행
- 텍스트, 이미지, PDF, Jupyter 노트북 등 다양한 형식 지원
- 라인 범위 지정으로 대용량 파일의 일부만 읽기 가능
- 여러 파일을 동시에 읽어 비교/분석 가능

---

## 파일 읽기 방법

### 방법 1: 자연어 요청

```
> src/main.py 파일을 읽어줘
> config/settings.py의 내용을 보여줘
> .env.example 파일이 어떻게 생겼는지 확인해줘
```

### 방법 2: @-mention (권장)

`@` 뒤에 파일 경로를 입력합니다. Tab 자동완성을 지원합니다.

```
> @src/main.py 이 파일에서 라우터 설정 부분을 찾아줘
> @package.json 에 어떤 의존성이 있어?
> @.gitignore 에 node_modules가 포함되어 있어?
```

### 방법 3: 절대 경로 사용

```
> /home/user/projects/myapp/src/app.py 를 읽어줘
> C:\Users\USER\projects\myapp\src\app.py 분석해줘
```

## 지원 파일 형식

### 텍스트 파일 (코드, 설정 등)

모든 텍스트 기반 파일을 읽을 수 있습니다.

```
> @src/auth/jwt.py 이 파일 분석해줘
> @docker-compose.yml 서비스 구성을 설명해줘
> @nginx.conf 현재 설정의 문제점을 찾아줘
> @tsconfig.json strict 모드가 활성화되어 있어?
```

지원 형식 예시:
- `.py`, `.js`, `.ts`, `.jsx`, `.tsx`
- `.json`, `.yaml`, `.yml`, `.toml`, `.ini`
- `.md`, `.txt`, `.csv`, `.xml`, `.html`
- `.sh`, `.bash`, `.zsh`
- `.sql`, `.graphql`
- `.Dockerfile`, `.env.example`

### 이미지 파일 (멀티모달)

Claude는 이미지를 시각적으로 분석할 수 있습니다.

```
> @screenshot.png 이 화면에서 어떤 에러가 발생했어?
> @ui-mockup.jpg 이 디자인을 HTML/CSS로 구현해줘
> @diagram.png 이 아키텍처 다이어그램을 설명해줘
> @error-screenshot.jpg 이 에러를 어떻게 해결해?
```

지원 형식: `.png`, `.jpg`, `.jpeg`, `.gif`, `.webp`

**활용 사례:**
- UI 스크린샷 → 코드 구현
- 에러 화면 → 디버깅
- 다이어그램 → 문서화 또는 코드 생성

### PDF 파일

PDF를 직접 읽고 내용을 분석합니다. 페이지 범위를 지정할 수 있습니다.

```
> @spec.pdf 이 사양서의 핵심 요구사항을 정리해줘
> @api-docs.pdf 3-5페이지를 읽어줘
> @contract.pdf 이 계약서에서 중요한 조항을 찾아줘
```

페이지 범위 지정:
```
> @large-report.pdf 의 10-20페이지만 읽어서 요약해줘
```

### Jupyter 노트북 (.ipynb)

노트북의 모든 셀(코드, 텍스트, 출력)을 읽습니다.

```
> @analysis.ipynb 이 분석의 결론이 뭐야?
> @model-training.ipynb 학습 파라미터를 최적화해줘
> @data-exploration.ipynb 데이터 전처리 단계를 설명해줘
```

## 라인 범위 지정

대용량 파일에서 특정 부분만 읽을 때 유용합니다.

```
> main.py의 50-100번째 줄만 읽어줘
> src/api/routes.py 의 200번 라인부터 250번 라인을 보여줘
> database.py에서 100번째 줄 근처를 읽어줘
```

직접 요청 예시:
```
> @src/models/user.py 를 읽되, User 클래스 정의 부분만 봐줘 (약 20-60번 줄)
```

## 여러 파일 동시 읽기

Claude는 여러 파일을 동시에 읽어 비교하거나 연관 분석을 할 수 있습니다.

```
> @src/services/auth.py 와 @src/services/user.py 의 차이점을 설명해줘
> @frontend/src/api.js 와 @backend/app/main.py 가 서로 호환되는지 확인해줘
> @test/test_auth.py 와 @src/auth.py 를 비교해서 테스트 커버리지를 분석해줘
```

프로젝트 전체 구조 파악:
```
> @src/models/user.py @src/services/user.py @src/routers/user.py
  이 세 파일의 관계를 설명하고 개선점을 찾아줘
```

## 디렉토리 탐색과 함께 사용

Read와 Bash 도구를 조합해 디렉토리 구조를 파악한 후 읽을 수 있습니다.

```
> 이 프로젝트의 전체 구조를 파악하고 중요한 파일들을 읽어서 설명해줘
```

Claude가 자동으로:
1. `ls` 또는 `find`로 파일 목록 파악
2. 핵심 파일 식별 및 읽기
3. 전체 구조 설명

## 실습: 현재 디렉토리 파일 읽어보기

### 실습 1: 기본 파일 읽기

먼저 실습용 파일을 만들어 두겠습니다.

```bash
# 실습 디렉토리 준비
mkdir ~/claude-read-practice
cd ~/claude-read-practice

# 예시 파일 생성
cat > app.py << 'EOF'
from fastapi import FastAPI

app = FastAPI()

@app.get("/")
def read_root():
    return {"message": "Hello, World!"}

@app.get("/users/{user_id}")
def read_user(user_id: int):
    return {"user_id": user_id}
EOF
```

Claude에게 읽기 요청:

```
> @app.py 이 파일을 분석해줘. 어떤 엔드포인트가 있어?
```

### 실습 2: 이미지 분석

```
> @screenshot.png 이 화면에서 레이아웃 문제가 보여?
```

### 실습 3: 여러 파일 비교

```bash
cat > models.py << 'EOF'
class User:
    def __init__(self, id: int, name: str, email: str):
        self.id = id
        self.name = name
        self.email = email
EOF

cat > schemas.py << 'EOF'
from pydantic import BaseModel

class UserCreate(BaseModel):
    name: str
    email: str

class UserResponse(BaseModel):
    id: int
    name: str
    email: str
EOF
```

```
> @models.py 와 @schemas.py 가 서로 일치하는지 확인해줘
```

### 실습 4: 라인 범위 지정

```
> @app.py 의 1번부터 5번 라인만 읽어줘
```

## 주의사항

- 매우 큰 파일(수만 줄)은 토큰 한도를 초과할 수 있으므로 라인 범위를 지정하세요.
- 바이너리 파일(`.exe`, `.bin`, `.zip` 등)은 읽을 수 없습니다.
- 이미지는 파일 크기가 클수록 더 많은 토큰을 소비합니다.
- PDF는 텍스트 레이어가 있어야 읽을 수 있습니다 (스캔된 이미지 PDF는 OCR 불가).
