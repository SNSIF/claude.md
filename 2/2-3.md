# 2-3. 파일 편집 (Edit)

## Edit 도구 개요

Edit 도구는 파일의 특정 부분만 정확하게 수정하는 도구입니다. 변경할 텍스트(old_string)와 새 텍스트(new_string)를 지정해 교체하는 방식으로 동작합니다.

**핵심 특징:**
- **diff 방식** — 변경된 부분만 전송하여 토큰 효율적
- 파일 전체가 아닌 필요한 부분만 교체
- 고유한 문자열을 기준으로 교체하므로 정확성 보장
- 파일 읽기 후에만 사용 가능 (Claude가 먼저 파일을 Read해야 함)

---

## Write와의 차이

| 항목 | Edit | Write |
|------|------|-------|
| 교체 범위 | 지정된 부분만 | 파일 전체 |
| 전송 크기 | 작음 (변경분만) | 큼 (전체 파일) |
| 나머지 내용 | 그대로 보존 | 삭제됨 |
| 용도 | 수정, 버그 수정, 기능 추가 | 새 파일 생성, 전면 교체 |

---

## 작동 방식

내부적으로 `old_string → new_string` 교체를 수행합니다.

```
old_string: "def add(a, b):\n    return a - b"
new_string: "def add(a, b):\n    return a + b"
```

Claude가 파일에서 `old_string`을 찾아 `new_string`으로 교체합니다. `old_string`이 파일 내에서 **유일해야** 합니다. 같은 문자열이 여러 곳에 있으면 Claude가 더 넓은 컨텍스트를 포함해 고유하게 만들어 처리합니다.

## 권한 필요

Edit도 파일을 변경하므로 처음 실행 시 승인이 필요합니다.

```
Claude가 다음 편집을 실행하려 합니다:

  파일: src/utils/math.py

  - def add(a, b):
  -     return a - b    ← 삭제
  + def add(a, b):
  +     return a + b    ← 추가

허용하시겠습니까? [y/n]
```

## 여러 부분 동시 편집

Claude는 한 파일의 여러 부분을 연속으로 Edit할 수 있습니다.

```
> @src/config.py 에서 다음 세 가지를 수정해줘:
  1. DEBUG = True 를 False로 변경
  2. DATABASE_URL에서 localhost를 db로 변경 (Docker 환경)
  3. SECRET_KEY를 os.environ.get()으로 읽도록 변경
```

Claude가 세 번의 Edit 호출을 연속으로 실행합니다.


## 실습: 기존 파일의 특정 함수 수정

### 실습 준비: 예제 파일 만들기

```bash
mkdir ~/claude-edit-practice
cd ~/claude-edit-practice
```

아래 내용으로 `calculator.py`를 만들어 두세요.

```python
# calculator.py

class Calculator:
    """간단한 계산기 클래스"""

    def add(self, a, b):
        return a - b  # 버그: 빼기로 잘못 구현됨

    def subtract(self, a, b):
        return a - b

    def multiply(self, a, b):
        return a * b

    def divide(self, a, b):
        return a / b  # 0으로 나누기 예외 처리 없음

    def power(self, base, exp):
        result = 1
        for i in range(exp):
            result = result * base  # 비효율적 구현
        return result
```

### 실습 1: 버그 수정

```
> @calculator.py 의 add 함수에 버그가 있어. 빼기 대신 더하기를 해야 해.
```

### 실습 2: 예외 처리 추가

```
> @calculator.py 의 divide 함수에 0으로 나누기 예외 처리를 추가해줘.
  ZeroDivisionError를 발생시키는 대신 ValueError를 사용해줘.
```

### 실습 3: 리팩토링

```
> @calculator.py 의 power 함수를 ** 연산자를 사용하도록 리팩토링해줘.
```

### 실습 4: 기능 추가

```
> @calculator.py 의 Calculator 클래스에 다음 메서드를 추가해줘:
  - modulo(a, b): 나머지 연산, 0으로 나누기 처리 포함
  - is_even(n): 짝수 판별
```

### 실습 5: 문서화

```
> @calculator.py 의 모든 메서드에 Args, Returns, Raises 형식의 docstring을 추가해줘.
```

## Edit 실패 케이스 처리

### 문제: old_string이 파일에 없음

Claude가 파일을 다시 읽고 정확한 문자열을 찾아 재시도합니다.

```
> @app.py 에서 app = Flask(__name__) 을 FastAPI로 바꿔줘
```

만약 실제로 `Flask`가 없다면 Claude가 알려줍니다.

### 문제: 같은 문자열이 여러 곳에 있음

Claude가 더 넓은 컨텍스트를 포함해 고유하게 식별합니다.

```python
# 파일 내에 같은 함수명이 두 번 있는 경우
# Claude가 클래스 이름 등 주변 컨텍스트를 포함해 구분
```
