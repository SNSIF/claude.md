# 10-3. 상태바 & 단축키

## Claude Code 상태바 (VS Code)

VS Code 하단 상태바에 Claude Code의 현재 상태가 표시됩니다. 이를 통해 작업 흐름을 방해하지 않고 Claude의 상태를 한눈에 파악할 수 있습니다.

```
[VS Code 상태바 예시]
┌─────────────────────────────────────────────────────────┐
│  ⎇ main  │ claude-sonnet-4-6 │ $0.023 │ 🔵 auto │ ...  │
└─────────────────────────────────────────────────────────┘
     Git      현재 모델          비용     권한 모드
```

### 상태바 항목 설명

#### 현재 모델 표시

```
claude-sonnet-4-6     → 현재 사용 중인 Claude 모델
claude-opus-4-6       → Opus 모델 사용 중
claude-haiku-4-5-...  → Haiku 모델 사용 중 (빠른 작업)
```

클릭하면 모델 전환 메뉴가 표시됩니다.

#### 비용/토큰 카운터

```
$0.023   → 현재 세션의 누적 비용 (USD)
1.2k tok → 사용된 토큰 수
```

- 세션 종료 시 누적 비용 초기화
- 클릭하면 상세 토큰 분석 팝업 표시:
  ```
  Input tokens:  8,432
  Output tokens: 1,245
  Cache read:    6,103
  Cache write:   2,329
  ─────────────────────
  Estimated cost: $0.023
  ```

#### 권한 모드 인디케이터

| 표시 | 모드 | 설명 |
|------|------|------|
| `🔵 auto` | Auto | 모든 작업 자동 승인 |
| `🟡 plan` | Plan | 읽기만 가능, 실행 불가 |
| `🔴 manual` | Manual | 모든 작업에 수동 승인 필요 |

`Shift+Tab` 으로 모드를 순환 전환합니다.

### 상태바 커스터마이징

```json
// settings.json
{
  // 상태바에 표시할 항목 제어
  "claude.statusBar.showModel": true,
  "claude.statusBar.showCost": true,
  "claude.statusBar.showTokenCount": false,
  "claude.statusBar.showPermissionMode": true,

  // 상태바 위치 (left/right)
  "claude.statusBar.alignment": "right",

  // 비용 표시 단위
  "claude.statusBar.costUnit": "usd",  // "usd" 또는 "krw"

  // 토큰 경고 임계값
  "claude.statusBar.warnAtTokens": 100000
}
```

## 키바인딩 커스터마이징

### 설정 파일 위치

```bash
~/.claude/keybindings.json
```

Windows:
```
C:\Users\<사용자명>\.claude\keybindings.json
```

macOS/Linux:
```
~/.claude/keybindings.json
```

### 기본 단축키 목록

| 단축키 | 명령 | 설명 |
|--------|------|------|
| `Ctrl+Enter` | `claude.submitPrompt` | 프롬프트 제출 |
| `Escape` | `claude.abortResponse` | 응답 중단 |
| `Ctrl+Shift+Del` | `claude.clearSession` | 세션 초기화 |
| `Shift+Tab` | `claude.cyclePermissionMode` | 권한 모드 전환 |
| `Ctrl+Shift+C` | `claude.togglePanel` | 패널 열기/닫기 |
| `Ctrl+K Ctrl+C` | `claude.compactContext` | 컨텍스트 압축 |
| `Alt+T` | `claude.toggleThinking` | Extended Thinking 토글 |
| `Ctrl+Up` | `claude.historyPrev` | 이전 프롬프트로 이동 |
| `Ctrl+Down` | `claude.historyNext` | 다음 프롬프트로 이동 |

### 커스텀 바인딩 추가 예제

```json
// ~/.claude/keybindings.json
[
  {
    "key": "ctrl+enter",
    "command": "claude.submitPrompt"
  },
  {
    "key": "ctrl+shift+r",
    "command": "claude.reviewCode",
    "when": "editorHasSelection"
  },
  {
    "key": "ctrl+shift+f",
    "command": "claude.fixError",
    "when": "editorHasSelection"
  },
  {
    "key": "ctrl+shift+t",
    "command": "claude.generateTests",
    "when": "editorTextFocus"
  },
  {
    "key": "alt+shift+d",
    "command": "claude.explainCode",
    "when": "editorHasSelection"
  }
]
```

### Chord(조합) 단축키 설정

두 키 조합을 순서대로 누르는 chord 단축키를 정의할 수 있습니다.

```json
[
  {
    "key": "ctrl+k ctrl+r",
    "command": "claude.reviewCode"
  },
  {
    "key": "ctrl+k ctrl+f",
    "command": "claude.fixAllErrors"
  },
  {
    "key": "ctrl+k ctrl+p",
    "command": "claude.togglePlanMode"
  },
  {
    "key": "ctrl+k ctrl+s",
    "command": "claude.summarizeChanges"
  }
]
```

**사용 방법**: `Ctrl+K` 를 누른 후 손을 떼고 `Ctrl+R` 을 누릅니다.

## 주요 기본 단축키 상세 설명

### 프롬프트 제출 (`Ctrl+Enter`)

```
일반 Enter     → 줄바꿈
Ctrl+Enter     → 프롬프트 제출 (Claude에게 전송)
Shift+Enter    → 줄바꿈 (Enter와 동일)
```

### 세션 초기화 (`Ctrl+Shift+Del`)

현재 대화 컨텍스트를 완전히 지웁니다. 새로운 주제나 다른 파일 작업 시작 전에 사용합니다.

```bash
# CLI에서도 동일하게 동작
/clear
```

### 권한 모드 전환 (`Shift+Tab`)

```
첫 번째 Shift+Tab → Plan 모드 (읽기 전용)
두 번째 Shift+Tab → Manual 모드 (수동 승인)
세 번째 Shift+Tab → Auto 모드 (자동 승인, 기본값)
```

### 컨텍스트 압축 (`Ctrl+K Ctrl+C`)

대화가 길어져 토큰이 부족해질 때 컨텍스트를 압축합니다.

```bash
# CLI에서 동일하게 동작
/compact

# 압축 전략 지정
/compact --strategy aggressive   # 더 많이 압축
/compact --strategy conservative # 중요 내용 최대한 보존
```

**압축 과정**:
1. Claude가 현재까지의 대화를 요약
2. 요약본으로 컨텍스트 교체
3. 이후 대화는 요약 기반으로 진행

### Extended Thinking 토글 (`Alt+T`)

```
Alt+T         → Extended Thinking 켜기/끄기
Alt+Shift+T   → Thinking 수준 선택 메뉴
```

## 단축키 충돌 해결

기존 VS Code / IDE 단축키와 충돌 시:

```json
// 기존 단축키 비활성화
[
  {
    "key": "ctrl+enter",
    "command": "-editor.action.insertLineAfter",
    "when": "editorTextFocus && !editorReadonly"
  },
  {
    "key": "ctrl+enter",
    "command": "claude.submitPrompt",
    "when": "claude.panelFocused"
  }
]
```

`when` 조건을 활용하면 특정 상황에서만 단축키가 동작하도록 제한할 수 있습니다.

## 실습

### 실습 1: 상태바 확인 및 모델 전환

1. VS Code에서 Claude Code 패널 열기
2. 하단 상태바에서 모델명 클릭
3. 목록에서 다른 모델 선택 (예: claude-haiku-4-5)
4. 간단한 질문을 보내고 응답 속도 차이 확인
5. 상태바의 비용 카운터 변화 확인

### 실습 2: 커스텀 단축키 설정

```bash
# keybindings.json 파일 생성 또는 편집
mkdir -p ~/.claude
cat > ~/.claude/keybindings.json << 'EOF'
[
  {
    "key": "ctrl+enter",
    "command": "claude.submitPrompt"
  },
  {
    "key": "ctrl+shift+r",
    "command": "claude.reviewCode",
    "when": "editorHasSelection"
  }
]
EOF
```

### 실습 3: 권한 모드 순환 체험

1. Claude Code 패널에서 다음 프롬프트 입력:
   ```
   현재 디렉토리의 모든 .js 파일에서 console.log를 제거해줘
   ```
2. `Shift+Tab` 으로 Plan 모드 전환
3. 같은 프롬프트 재입력 → 계획만 제시됨을 확인
4. `Shift+Tab` 으로 Auto 모드 복귀 후 실제 실행

### 실습 4: 컨텍스트 압축 실험

1. Claude와 긴 대화를 나누어 토큰을 소비
2. 상태바의 토큰 카운터가 증가함을 확인
3. `Ctrl+K Ctrl+C` 로 컨텍스트 압축
4. 압축 후 토큰 수가 감소했음을 확인
5. 이전 대화 내용에 대해 질문하여 요약이 유지되는지 확인
