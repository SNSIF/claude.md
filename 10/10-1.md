# 10-1. VS Code 확장

## Claude Code VS Code 확장 개요

Claude Code VS Code 확장은 터미널 기반의 Claude Code CLI를 Visual Studio Code 에디터 환경과 통합합니다. 에디터에서 코드를 작성하면서 별도의 터미널 창 전환 없이 Claude와 자연스럽게 상호작용할 수 있습니다. 인라인 diff, 파일 참조, 선택 영역 공유 등 에디터와 긴밀하게 통합된 기능이 특징입니다.

---

## 설치 방법

### 방법 1: Extensions 마켓플레이스에서 설치

1. VS Code를 열고 좌측 사이드바의 Extensions 아이콘 클릭 (`Ctrl+Shift+X`)
2. 검색창에 **"Claude Code"** 입력
3. Anthropic이 게시한 확장 선택 후 **Install** 클릭
4. VS Code 재시작 (선택사항)

### 방법 2: 명령줄에서 설치

```bash
code --install-extension anthropic.claude-code
```

### 방법 3: VSIX 직접 설치

```bash
# 최신 VSIX 파일 다운로드 후
code --install-extension claude-code-x.x.x.vsix
```

### 설치 확인

```bash
code --list-extensions | grep anthropic
# 출력: anthropic.claude-code
```

> **선행 조건**: Claude Code CLI가 시스템에 설치되어 있어야 합니다.
> ```bash
> npm install -g @anthropic-ai/claude-code
> ```

## 주요 기능

### 1. 사이드 패널에서 Claude와 대화

VS Code 좌측 또는 우측 사이드 패널에 Claude Code 대화창이 표시됩니다.

- 에디터 작업 중 패널을 열어 실시간으로 Claude에게 질문
- 현재 열린 파일의 컨텍스트를 자동으로 인식
- 다중 파일 프로젝트에서도 관련 파일을 파악하여 응답

```
[VS Code 레이아웃]
┌──────────────┬────────────────────┐
│  파일 탐색기   │   에디터 영역       │
│              │                    │
│              │    // 코드 작성 중   │
│              │                    │
├──────────────┴────────────────────┤
│ Claude Code 패널                   │
│ > 이 함수의 시간복잡도를 분석해줘      │
└───────────────────────────────────┘
```

### 2. 인라인 Diff 보기

Claude가 파일을 수정할 때 변경사항을 에디터 내에서 직접 확인할 수 있습니다.

- 추가된 라인: 초록색 배경
- 삭제된 라인: 빨간색 배경
- 변경 전/후 나란히 비교 (Side-by-side diff)
- **수락(Accept)** 또는 **거부(Reject)** 버튼으로 각 변경사항 제어
- 부분 수락: 특정 변경사항만 선택적으로 적용 가능

```
[인라인 Diff 예시]
- function getUserName(id) {        // 삭제 (빨간색)
+ function getUserName(userId: number): string {  // 추가 (초록색)
    const user = db.find(id);       // 변경 없음
-   return user.name;               // 삭제 (빨간색)
+   return user?.name ?? '';        // 추가 (초록색)
  }
```

### 3. 파일 @-mention

대화창에서 `@` 기호를 입력하면 현재 워크스페이스의 파일을 참조할 수 있습니다.

```
사용 예시:
> @src/utils/auth.ts 파일에서 토큰 만료 처리 로직을 개선해줘
> @package.json 의존성을 보고 업그레이드 필요한 패키지 알려줘
> @README.md 내용을 바탕으로 시작 가이드를 작성해줘
```

**자동 완성 지원**: `@` 입력 후 파일명 일부를 타이핑하면 후보 목록이 표시됩니다.

### 4. 선택 영역 공유

에디터에서 코드를 선택한 후 Claude에게 직접 전달할 수 있습니다.

**사용 방법**:
1. 에디터에서 분석할 코드 블록 선택 (드래그)
2. 우클릭 → **"Ask Claude about selection"** 선택
3. 또는 선택 후 Claude 패널에 자동으로 삽입

```python
# 아래 코드를 선택 후 "왜 이 함수가 느린지 분석해줘" 질문
def process_data(items):
    result = []
    for item in items:
        if item not in result:  # O(n) 탐색 — 비효율
            result.append(item)
    return result
```

### 5. 대화 이력

이전 세션과의 대화를 탐색하고 재개할 수 있습니다.

- 세션 목록 패널에서 날짜/프로젝트별 정렬
- 이전 대화 검색 기능
- 특정 세션 재개 (컨텍스트 복원)
- 세션 내보내기 (마크다운 형식)

## 설정

VS Code 설정 (`Ctrl+,`)에서 `claude` 검색:

```json
// settings.json
{
  // 사이드 패널 위치
  "claude.panel.position": "right",  // "left" 또는 "right"

  // 파일 변경 시 자동으로 diff 뷰 열기
  "claude.autoOpenDiff": true,

  // 인라인 diff 스타일
  "claude.diffStyle": "inline",  // "inline" 또는 "side-by-side"

  // 세션 자동 저장
  "claude.session.autoSave": true,

  // 파일 @-mention 최대 개수
  "claude.mention.maxFiles": 10,

  // 상태바 표시 항목
  "claude.statusBar.showModel": true,
  "claude.statusBar.showCost": true
}
```

## 단축키

### 기본 단축키

| 단축키 | 동작 |
|--------|------|
| `Ctrl+Shift+P` → "Claude Code" | 명령 팔레트에서 Claude 명령 검색 |
| `Ctrl+Shift+C` | Claude Code 패널 열기/닫기 |
| `Ctrl+Enter` | 프롬프트 제출 |
| `Shift+Tab` | 권한 모드 전환 |
| `Esc` | 현재 응답 중단 |

### 프롬프트 내 슬래시 명령

Claude Code 패널의 입력창에서 `/` 를 입력하면 사용 가능한 스킬 목록이 나타납니다:

```
/clear          — 현재 대화 초기화
/compact        — 컨텍스트 압축
/plan           — Plan Mode 활성화
/review         — 코드 리뷰 요청
/test           — 테스트 생성 요청
/explain        — 코드 설명 요청
/fix            — 오류 수정 요청
```

## 터미널에서 CLI와 함께 사용

VS Code의 통합 터미널(`Ctrl+`` `)과 사이드 패널을 동시에 활용할 수 있습니다.

```bash
# 터미널에서 CLI 실행
claude

# 특정 파일을 컨텍스트로 지정하여 시작
claude --add-dir ./src

# 헤드리스 모드로 스크립트 실행 후 패널에서 결과 확인
claude -p "테스트 실패 원인 분석" --output-format json
```

**권장 워크플로우**:
- 사이드 패널: 대화형 탐색, 설명 요청, 코드 리뷰
- 통합 터미널: 자동화 스크립트, CI/CD 연동, 배치 작업

## 실습

### 실습 1: 기본 대화 시작하기

1. VS Code에서 임의의 프로젝트 폴더 열기
2. `Ctrl+Shift+C` 로 Claude Code 패널 열기
3. 다음 프롬프트 입력:

```
현재 프로젝트 구조를 분석하고 개선 사항을 제안해줘
```

### 실습 2: 인라인 Diff 활용

1. 기존 JavaScript 파일 열기
2. 다음과 같이 입력:

```
@현재파일.js 의 모든 var 선언을 const/let으로 변환해줘
```

3. 에디터에서 제안된 변경사항 확인
4. 각 변경사항을 수락/거부

### 실습 3: 선택 영역 분석

```javascript
// 아래 코드를 선택하고 "이 코드의 문제점을 찾아줘" 입력
async function fetchUser(id) {
  const res = await fetch(`/api/users/${id}`)
  const data = await res.json()
  return data
}
```
