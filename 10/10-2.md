# 10-2. JetBrains 플러그인

## 개요

Claude Code JetBrains 플러그인은 JetBrains 계열 IDE와 Claude Code CLI를 통합합니다. IntelliJ IDEA의 강력한 정적 분석, 코드 완성, 리팩토링 도구와 Claude의 AI 기능을 결합하여 생산성을 극대화합니다.

---

## 설치 방법

### 방법 1: IDE 내 플러그인 마켓플레이스

1. **Settings** 열기 (`Ctrl+Alt+S` / macOS: `Cmd+,`)
2. **Plugins** 섹션으로 이동
3. **Marketplace** 탭 클릭
4. 검색창에 **"Claude Code"** 입력
5. Anthropic 게시 플러그인 선택 → **Install** 클릭
6. IDE 재시작

```
Settings → Plugins → Marketplace → "Claude Code" 검색 → Install
```

### 방법 2: 디스크에서 설치 (오프라인 환경)

1. JetBrains Marketplace 웹사이트에서 `.zip` 파일 다운로드
2. **Settings → Plugins → ⚙️ → Install Plugin from Disk...**
3. 다운로드한 `.zip` 선택

### 설치 확인

**Settings → Plugins → Installed** 탭에서 "Claude Code" 활성화 여부 확인

## 주요 기능

### 1. 사이드 패널 대화창

VS Code 확장과 유사하게 IDE 우측/하단에 Claude Code 대화창이 표시됩니다.

```
[IntelliJ 레이아웃]
┌──────────────────────────────────────────┐
│  메뉴바 / 툴바                            │
├───────────┬──────────────────────────────┤
│ 프로젝트  │   에디터 (코드 작성)          │
│ 탐색기    │                              │
│           │                              │
├───────────┴──────────────────────────────┤
│ Claude Code                    [탭]      │
│ > 이 클래스의 책임이 너무 많아. 분리해줘  │
└──────────────────────────────────────────┘
```

### 2. 인라인 Diff

Claude가 제안하는 코드 변경사항을 에디터 내에서 직접 확인합니다.

- JetBrains의 기본 diff 뷰어와 통합 (Unified / Side-by-side)
- 변경사항별 수락/거부 제어
- 전체 수락 또는 전체 거부 옵션
- 변경 이력에 Claude 세션 참조 포함

```java
// 변경 전 (빨간색)
- public String getUserName(int id) {
-     User user = userRepo.findById(id);
-     return user.getName();
- }

// 변경 후 (초록색)
+ public Optional<String> getUserName(long id) {
+     return userRepo.findById(id)
+         .map(User::getName);
+ }
```

### 3. 파일 참조 (@-mention)

대화창에서 `@`를 입력하면 프로젝트 파일을 참조할 수 있습니다.

```
> @UserService.java 와 @UserRepository.java 를 보고
  의존성 역전 원칙이 지켜지고 있는지 확인해줘
```

**자동 완성**: IDE의 파일 인덱스를 활용하여 정확한 파일명 완성 지원

### 4. 선택 영역 공유

```
사용 방법:
1. 에디터에서 코드 선택
2. 우클릭 메뉴 → "Ask Claude"
   또는 Alt+C (커스텀 단축키 설정 가능)
3. Claude 패널에 선택 코드 자동 삽입
```

## JetBrains 고유 기능

### IDE 인텔리전스 연동

JetBrains의 정적 분석 결과를 Claude가 함께 활용합니다.

**타입 정보 활용**:
```
Claude가 자동으로 인식하는 정보:
- 변수/함수의 타입 정보 (Java, Kotlin, Python 등)
- 임포트 관계 그래프
- 인터페이스 구현 관계
- 오버라이드 계층 구조
```

**검사(Inspection) 연동**:
```
// IDE가 발견한 경고를 Claude에게 자동 전달
⚠️ NullPointerException 가능성 → Claude: "이 문제를 어떻게 해결할까?"
⚠️ Unused variable 'result' → Claude: "이 변수가 필요 없는 이유는?"
```

**리팩토링 제안 연동**:
- IDE의 "Extract Method", "Rename" 등 리팩토링 제안을 Claude가 확인
- 리팩토링 후 영향받는 파일을 Claude가 함께 분석

### 데이터베이스 도구 연동 (DataGrip / IntelliJ Ultimate)

```sql
-- 선택한 쿼리를 Claude에게 분석 요청
SELECT u.*, COUNT(o.id) as order_count
FROM users u
LEFT JOIN orders o ON u.id = o.user_id
WHERE u.created_at > '2024-01-01'
GROUP BY u.id
-- → "이 쿼리의 인덱스 전략을 제안해줘"
```

## 설정 방법

**Settings → Tools → Claude Code** 경로에서 설정:

```
Claude Code 설정 항목:
┌─────────────────────────────────────────┐
│ API 설정                                │
│   ☑ CLI 경로 자동 감지                  │
│   CLI 경로: /usr/local/bin/claude       │
│                                         │
│ 패널 설정                               │
│   위치: ○ 하단  ● 우측                  │
│   ☑ 자동 diff 열기                      │
│   Diff 스타일: Unified ▼               │
│                                         │
│ 단축키                                  │
│   패널 열기: Alt+C                      │
│   선택 영역 질문: Alt+Shift+C           │
└─────────────────────────────────────────┘
```

**Keymap 설정** (`Settings → Keymap → Claude Code`):

| 동작 | 기본 단축키 |
|------|-------------|
| Claude 패널 열기/닫기 | `Alt+C` |
| 선택 영역에 대해 질문 | `Alt+Shift+C` |
| 프롬프트 제출 | `Ctrl+Enter` |
| 응답 중단 | `Escape` |
| 세션 초기화 | `Ctrl+Shift+Del` |

## CLI와 병행 사용

JetBrains의 내장 터미널 (`Alt+F12`)에서 CLI를 함께 사용합니다.

```bash
# JetBrains 내장 터미널에서 실행
claude

# 현재 프로젝트 루트를 자동으로 인식
# (JetBrains가 워킹 디렉토리를 프로젝트 루트로 설정)

# 특정 모듈만 대상으로 작업
claude --add-dir ./src/main/java/com/example/service
```

**추천 워크플로우**:
1. 플러그인 패널: 인터랙티브 탐색, 코드 설명, 리뷰
2. 내장 터미널 CLI: 자동화, 테스트 실행, 배치 변경

## 실습

### 실습 1: 플러그인 설치 및 기본 대화

1. IntelliJ IDEA 또는 PyCharm에서 플러그인 설치
2. 기존 프로젝트 열기
3. `Alt+C` 로 패널 열기
4. 입력:

```
이 프로젝트의 패키지 구조를 설명하고,
Clean Architecture 관점에서 개선점을 제안해줘
```

### 실습 2: 타입 안전성 개선 (Kotlin 예제)

```kotlin
// 아래 코드를 선택하고 Claude에게 전달
fun processOrder(orderId: String?): String {
    val order = orderRepo.findById(orderId)
    return order.status  // NullPointerException 위험
}
```

입력:
```
이 코드의 Null 안전성 문제를 Kotlin 관용적인 방식으로 수정해줘
```

### 실습 3: IDE 검사 결과 활용

1. 프로젝트에서 `Ctrl+Alt+Shift+I` 로 전체 검사 실행
2. 발견된 경고 목록에서 하나 선택
3. Claude 패널에서:

```
방금 IDE가 발견한 "Potential null dereference" 경고들을
일괄적으로 수정하는 방법을 제안해줘
```
