# 3-2. 브랜치 & PR

## 브랜치 생성

Claude에게 자연어로 요청하면 적절한 브랜치명을 만들어 체크아웃합니다.

```
> feature/login 브랜치 만들어줘
> 로그인 기능 개발을 위한 브랜치를 만들어줘
> bugfix/user-email-validation 브랜치를 만들고 체크아웃해줘
```

Claude가 실행:
```bash
git checkout -b feature/login
# 또는
git switch -c feature/login
```

### 브랜치 명명 규칙

Claude가 자연어 요청을 브랜치명으로 변환할 때 따르는 패턴:

| 요청 | 생성 브랜치명 |
|------|--------------|
| "로그인 기능 추가" | `feature/login` |
| "이메일 유효성 버그 수정" | `bugfix/email-validation` |
| "긴급 결제 오류 수정" | `hotfix/payment-error` |
| "코드 리팩토링" | `refactor/cleanup` |
| "이슈 #42 해결" | `fix/issue-42` |

## 브랜치 전환

```
> main 브랜치로 전환해줘
> develop 브랜치로 바꿔줘
> 이전 브랜치로 돌아가줘
```

```bash
git checkout main
git switch develop
git switch -   # 이전 브랜치로
```

## 브랜치 목록

```
> 브랜치 목록 보여줘
> 원격 브랜치도 포함해서 보여줘
> 어떤 브랜치에 있어?
```

```bash
git branch              # 로컬 브랜치
git branch -a           # 로컬 + 원격 브랜치
git branch --current    # 현재 브랜치
```

## 브랜치 삭제

```
> feature/login 브랜치 삭제해줘
> 이미 머지된 브랜치들을 정리해줘
```

```bash
git branch -d feature/login       # 머지된 브랜치만 삭제
git branch -D feature/experimental # 강제 삭제 (명시 요청 필요)
```

머지된 브랜치 일괄 삭제:
```bash
git branch --merged main | grep -v "main\|develop" | xargs git branch -d
```

## PR 생성 워크플로우

### 단계 1: 작업 완료 및 커밋

```
> 변경사항을 정리해서 커밋해줘. 로그인 기능 구현 완료.
```

### 단계 2: 원격에 푸시

```
> 현재 브랜치를 원격에 푸시해줘
```

```bash
git push -u origin feature/login
```

### 단계 3: 변경사항 요약

```
> main 브랜치 대비 이 브랜치에서 변경된 사항을 요약해줘
```

Claude가:
```bash
git diff main...HEAD --stat
git log main..HEAD --oneline
```

분석 결과 예시:
```
이 브랜치의 변경사항:
- 파일 6개 변경, 187줄 추가, 12줄 삭제
- 주요 변경: JWT 기반 로그인 API 구현, 사용자 인증 미들웨어 추가
- 테스트 파일 2개 추가 (커버리지 89%)
```

### 단계 4: PR 생성

```
> PR을 만들어줘
> GitHub PR을 생성해줘. 제목과 본문을 알아서 작성해줘.
```

Claude가 `gh` CLI를 사용해 PR을 생성합니다:

```bash
gh pr create \
  --title "feat: JWT 기반 로그인 기능 구현" \
  --body "$(cat <<'EOF'
## 변경 사항

- JWT 토큰 기반 로그인/로그아웃 API 구현
- 인증 미들웨어 추가 (모든 보호된 라우트에 적용)
- 비밀번호 해싱 로직 bcrypt으로 구현

## 테스트

- [ ] 로그인 성공/실패 케이스 확인
- [ ] 토큰 만료 처리 확인
- [ ] 보호된 라우트 접근 제한 확인

## 관련 이슈

closes #15
EOF
)"
```

## PR 제목 및 본문 작성 팁

### 좋은 PR 제목 예시

```
feat: 사용자 프로필 이미지 업로드 기능 추가
fix: 이메일 중복 가입 허용 버그 수정
refactor: 인증 서비스 의존성 주입 방식으로 변경
chore: GitHub Actions CI 파이프라인 구성
```

### PR 본문 구조

```
> 이 PR의 본문을 작성해줘. 변경사항, 테스트 방법, 관련 이슈를 포함해줘.
```

Claude가 생성하는 본문 템플릿:

```markdown
## 변경 사항
- [핵심 변경 1]
- [핵심 변경 2]

## 변경 이유
[왜 이 변경이 필요한지]

## 테스트 방법
- [ ] [테스트 단계 1]
- [ ] [테스트 단계 2]

## 스크린샷 (UI 변경 시)
[해당 없음]

## 관련 이슈
closes #[이슈 번호]
```

## PR 리뷰 및 머지

### PR 목록 확인

```
> 열린 PR 목록 보여줘
> 내가 리뷰해야 할 PR이 있어?
```

```bash
gh pr list
gh pr list --reviewer @me
```

### PR 내용 확인

```
> PR #42 내용을 분석해줘
> 이 PR의 코드 변경사항을 리뷰해줘
```

```bash
gh pr view 42
gh pr diff 42
```

### PR 체크아웃 (로컬에서 테스트)

```
> PR #42를 로컬에서 테스트해볼게. 체크아웃해줘.
```

```bash
gh pr checkout 42
```

### PR 머지

```
> PR #42를 머지해줘. squash 머지로.
```

```bash
gh pr merge 42 --squash --delete-branch
```

머지 방식:
| 방식 | 명령 | 특징 |
|------|------|------|
| 일반 머지 | `--merge` | 머지 커밋 생성 |
| Squash | `--squash` | 커밋 하나로 압축 |
| Rebase | `--rebase` | 선형 히스토리 유지 |

## `gh` CLI 도구 주요 명령

```bash
# 인증
gh auth login

# 저장소
gh repo view
gh repo clone owner/repo

# PR
gh pr create              # PR 생성
gh pr list                # PR 목록
gh pr view [번호]         # PR 상세
gh pr diff [번호]         # PR diff
gh pr checkout [번호]     # PR 체크아웃
gh pr merge [번호]        # PR 머지
gh pr close [번호]        # PR 닫기
gh pr comment [번호]      # 댓글 추가

# 이슈
gh issue create           # 이슈 생성
gh issue list             # 이슈 목록
gh issue view [번호]      # 이슈 상세
gh issue close [번호]     # 이슈 닫기
```
