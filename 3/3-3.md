# 3-3. Worktree

## Git Worktree 개념

Git Worktree는 하나의 Git 저장소에서 여러 브랜치를 동시에 서로 다른 디렉토리에 체크아웃할 수 있는 기능입니다.

**기존 방식의 한계:**
```
저장소 하나 → 브랜치 하나만 체크아웃 가능
feature/login 작업 중 → hotfix 요청 발생 → stash 또는 커밋 후 전환 필요
```

**Worktree 방식:**
```
저장소 하나 → 여러 디렉토리에 서로 다른 브랜치 동시 체크아웃
/project           → main 브랜치
/project-login     → feature/login 브랜치
/project-hotfix    → hotfix/payment 브랜치

모두 동일한 .git 저장소를 공유 → 디스크 효율적
```

**핵심 장점:**
- 브랜치 전환 없이 여러 작업 동시 진행
- 각 Worktree는 독립적인 작업 디렉토리
- 저장소 이력, 객체 데이터베이스는 공유 (디스크 효율)
- stash 없이 컨텍스트 완전히 분리

---

## Claude Code에서 Worktree 활용

Claude Code는 각 Worktree를 독립적인 프로젝트 컨텍스트로 인식합니다.

### 새 Worktree 생성하며 Claude 시작

```bash
# 기존 브랜치로 Worktree 생성
claude --worktree feature-auth

# 새 브랜치와 함께 Worktree 생성
claude --worktree bugfix-123
```

위 명령은 내부적으로:
1. `git worktree add ../프로젝트명-feature-auth feature/auth` 실행
2. 새 디렉토리에서 Claude 세션 시작
3. 해당 브랜치 컨텍스트에서 독립적으로 작업

---

## Worktree 생성/삭제 (Git 명령)

### 생성

```bash
# 기존 브랜치로 Worktree 추가
git worktree add ../my-project-feature feature/login

# 새 브랜치와 함께 Worktree 추가
git worktree add -b hotfix/payment ../my-project-hotfix main

# 현재 디렉토리 기준 경로 지정
git worktree add ./worktrees/feature-auth feature/auth
```

### 목록 확인

```bash
git worktree list
```

출력 예시:
```
/Users/user/projects/my-project         abc1234 [main]
/Users/user/projects/my-project-login   def5678 [feature/login]
/Users/user/projects/my-project-hotfix  ghi9012 [hotfix/payment]
```

### 삭제

```bash
# Worktree 디렉토리 삭제 후
git worktree remove ../my-project-login

# 강제 삭제 (수정사항 있어도)
git worktree remove --force ../my-project-login

# 더 이상 존재하지 않는 Worktree 정리
git worktree prune
```

## 각 Worktree의 독립적인 컨텍스트

각 Worktree 디렉토리에서 실행한 Claude 세션은 해당 Worktree에만 집중합니다.

```bash
# 터미널 1: 로그인 기능 개발
cd ~/projects/my-project-login
claude "로그인 API 구현 계속해줘"

# 터미널 2: 핫픽스 처리
cd ~/projects/my-project-hotfix
claude "결제 오류 원인 분석하고 수정해줘"
```

두 Claude 세션은:
- 각자의 파일 컨텍스트만 봄
- 각자의 브랜치에서만 작업
- 서로 간섭하지 않음

---

## 저장소 이력 공유 (디스크 효율)

Worktree는 `.git` 객체 데이터베이스를 공유합니다.

```
my-project/          ← 메인 Worktree (.git/ 포함)
my-project-login/    ← Linked Worktree (.git 파일만, 실제 데이터는 공유)
my-project-hotfix/   ← Linked Worktree
```

**실제 디스크 사용:**
- 저장소 히스토리: 한 번만 저장
- 각 브랜치의 작업 파일만 추가 저장
- 일반 브랜치 복사 대비 디스크 60-80% 절감

## 언제 Worktree를 사용해야 하나

### 1. 병렬 기능 개발

```
feature/auth      → 인증 시스템 개발
feature/payment   → 결제 모듈 개발
동시에 두 기능을 독립적으로 개발
```

```bash
git worktree add ../project-auth feature/auth
git worktree add ../project-payment feature/payment

# 터미널 1
cd ~/project-auth && claude "OAuth2 로그인 구현해줘"

# 터미널 2
cd ~/project-payment && claude "Stripe 결제 연동해줘"
```

### 2. 핫픽스와 기능 개발 동시 진행

```
상황: feature/dashboard 작업 중 프로덕션 버그 발생
```

```bash
# 기존 feature 작업 그대로 두고
# hotfix용 Worktree를 main에서 새로 생성
git worktree add -b hotfix/login-500 ../project-hotfix main

cd ~/project-hotfix
claude "로그인 500 에러 원인 찾아서 수정해줘"

# 핫픽스 완료 후 머지
git push origin hotfix/login-500
gh pr create --base main

# 다시 feature 작업으로 돌아오기
cd ~/project-dashboard
# stash 없이 바로 이어서 작업 가능
```

### 3. 코드 리뷰 중 작업 계속

```bash
# PR 리뷰를 위해 다른 브랜치 체크아웃
git worktree add ../project-review pr/123

cd ~/project-review
claude "이 PR의 코드를 리뷰해줘. 잠재적 문제점을 찾아줘."

# 내 작업 브랜치는 그대로 유지
cd ~/my-project
# 작업 계속
```

### 4. 서로 다른 버전 동시 실행

```bash
# v1 서버와 v2 서버 동시 실행
git worktree add ../project-v1 v1-stable
git worktree add ../project-v2 feature/v2

# 터미널 1: v1 서버 (포트 8000)
cd ~/project-v1 && uvicorn app.main:app --port 8000

# 터미널 2: v2 서버 (포트 8001)
cd ~/project-v2 && uvicorn app.main:app --port 8001

# 두 버전 동시 테스트 가능
```

## Claude Code 에이전트 병렬화

Worktree는 여러 Claude 에이전트를 병렬로 실행하는 데 최적입니다.

```bash
#!/bin/bash
# 세 가지 작업을 병렬로 처리

git worktree add ../project-backend feature/api-v2
git worktree add ../project-frontend feature/ui-v2
git worktree add ../project-tests feature/test-coverage

# 세 터미널에서 동시 실행
# 터미널 1
cd ~/project-backend
claude -p "REST API를 GraphQL로 마이그레이션해줘" --max-turns 20 &

# 터미널 2
cd ~/project-frontend
claude -p "React 컴포넌트를 TypeScript로 변환해줘" --max-turns 20 &

# 터미널 3
cd ~/project-tests
claude -p "전체 테스트 커버리지를 80%로 올려줘" --max-turns 20 &

wait
echo "모든 작업 완료"
```

## 실습

### 실습 1: 첫 Worktree 생성

```bash
cd ~/projects/my-project

# 현재 상태 확인
git worktree list

# 새 기능을 위한 Worktree 생성
git worktree add -b feature/profile ../my-project-profile main

# 새 Worktree에서 작업
cd ~/my-project-profile
claude "사용자 프로필 페이지 기능을 구현해줘"
```

### 실습 2: 핫픽스 시나리오

```bash
# 현재 feature 작업 중
cd ~/projects/my-project-feature
# (작업 중인 상태)

# 긴급 버그 발생 → 새 Worktree에서 처리
git worktree add -b hotfix/critical-bug ~/projects/my-project-hotfix main
cd ~/projects/my-project-hotfix
claude "현재 로그인이 안 되는 버그를 찾아서 수정해줘"

# 수정 완료 후
git add . && git commit -m "fix: 로그인 세션 만료 버그 수정"
git push origin hotfix/critical-bug

# 다시 feature로 돌아오기
cd ~/projects/my-project-feature
# 그대로 작업 재개
```

### 실습 3: Worktree 정리

```bash
# 작업 완료된 Worktree 삭제
git worktree remove ~/projects/my-project-profile
git worktree remove ~/projects/my-project-hotfix

# 목록 확인
git worktree list

# 고아 Worktree 정리
git worktree prune
```

---

## 주의사항

- 같은 브랜치는 두 개의 Worktree에 동시에 체크아웃할 수 없습니다.
- Worktree 디렉토리를 파일 탐색기로 직접 삭제하면 Git이 인식하지 못합니다. 반드시 `git worktree remove` 또는 `git worktree prune`을 사용하세요.
- Worktree 내에서 `git checkout`으로 브랜치를 바꾸면 다른 Worktree의 작업에 영향을 줄 수 있습니다.
