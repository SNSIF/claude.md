# 4-1. 내장 슬래시 커맨드

슬래시 커맨드는 `/`로 시작하는 특수 명령어로, Claude Code의 기능을 빠르게 제어하기 위한 인터페이스입니다. 대화 입력창에 직접 타이핑하며, Claude에게 자연어로 요청하는 것과 달리 즉각적으로 실행됩니다.

---

## 슬래시 커맨드 개요

일반 메시지와 슬래시 커맨드의 차이:

| 구분 | 예시 | 동작 |
|------|------|------|
| 일반 메시지 | `파일 목록 보여줘` | Claude가 해석 후 실행 |
| 슬래시 커맨드 | `/clear` | Claude 개입 없이 즉시 실행 |

슬래시 커맨드는 크게 두 가지 역할을 합니다.

1. **세션 제어** — 대화 상태, 컨텍스트, 비용 등을 관리
2. **설정 접근** — 권한, 훅, MCP 서버, 메모리 등을 조회·편집

---

## 전체 내장 커맨드 목록

### `/help`

사용 가능한 모든 슬래시 커맨드 목록과 간단한 설명을 출력합니다.

```
/help
```

커스텀 스킬이 등록되어 있으면 해당 스킬도 함께 표시됩니다.

---

### `/clear`

현재 대화의 컨텍스트(메시지 히스토리)를 완전히 초기화합니다. 새로운 작업을 시작할 때 이전 대화 내용이 영향을 미치지 않도록 할 때 사용합니다.

```
/clear
```

> 주의: 이전 대화 내용은 복구되지 않습니다. 중요한 내용은 CLAUDE.md나 메모리에 저장하세요.

---

### `/compact`

긴 대화 후 컨텍스트 창이 가득 찰 때 사용합니다. Claude가 현재까지의 대화를 요약하여 압축된 형태로 유지합니다. 메모리를 절약하면서도 대화의 핵심 맥락은 보존합니다.

```
/compact
```

컨텍스트가 80% 이상 차면 Claude Code가 자동으로 압축을 제안하기도 합니다. 자동 압축 임계값은 환경 변수 `CLAUDE_AUTOCOMPACT_PCT_OVERRIDE`로 조정할 수 있습니다.

---

### `/resume`

이전에 진행하던 세션을 재개합니다. 터미널을 닫았다가 다시 열거나, 작업 중간에 중단했을 때 마지막 세션으로 돌아갑니다.

```
/resume
```

여러 세션이 있다면 목록에서 선택할 수 있습니다.

---

### `/rename`

현재 세션에 이름을 부여하거나 기존 이름을 변경합니다. 여러 작업을 병행할 때 세션을 구분하기 쉬워집니다.

```
/rename
```

실행 후 새 이름을 입력하라는 프롬프트가 나타납니다.

```
새 세션 이름 입력: auth-refactor-2025
```

---

### `/config`

Claude Code의 설정 파일(`settings.json`)을 엽니다. 기본 에디터로 열리며, 저장 후 변경 사항이 즉시 반영됩니다.

```
/config
```

어떤 수준의 설정을 열지 선택할 수 있습니다.

- 사용자 전역 설정: `~/.claude/settings.json`
- 프로젝트 설정: `.claude/settings.json`

---

### `/memory`

Claude가 자동으로 저장한 메모리와 CLAUDE.md 내용을 확인하고 편집합니다.

```
/memory
```

메모리 파일 목록이 표시되며, 선택하면 에디터로 열립니다. 불필요한 메모리 항목을 직접 삭제하거나 수정할 수 있습니다.

---

### `/permissions`

도구별 허용/거부 권한을 시각적으로 관리하는 UI를 엽니다.

```
/permissions
```

현재 적용 중인 권한 규칙 목록과 각 도구의 상태를 확인하고 수정할 수 있습니다.

---

### `/hooks`

현재 프로젝트 또는 사용자 설정에 등록된 훅 목록을 표시합니다.

```
/hooks
```

훅은 특정 이벤트(도구 실행 전, 후 등)에 자동으로 실행되는 스크립트입니다. 등록된 훅이 없으면 빈 목록이 표시됩니다.

---

### `/agents`

현재 실행 중이거나 등록된 서브에이전트(subagent)를 관리합니다.

```
/agents
```

병렬 작업이나 복잡한 태스크를 위임한 에이전트의 상태를 확인하고 제어할 수 있습니다.

---

### `/mcp`

연결된 MCP(Model Context Protocol) 서버 목록과 상태를 표시합니다.

```
/mcp
```

각 서버의 연결 상태, 제공하는 도구 목록, 오류 메시지 등을 확인할 수 있습니다. MCP 서버 연결 문제를 진단할 때 유용합니다.

---

### `/skills`

등록된 커스텀 스킬 목록을 표시합니다.

```
/skills
```

전역 스킬(`~/.claude/skills/`)과 프로젝트 스킬(`.claude/skills/`)이 구분되어 표시됩니다.

---

### `/cost`

현재 세션에서 사용한 토큰 수와 예상 비용을 표시합니다.

```
/cost
```

출력 예시:

```
세션 비용 요약
─────────────────────
입력 토큰:   45,230
출력 토큰:   12,840
캐시 토큰:   38,100
예상 비용:   $0.0842
─────────────────────
```

장시간 작업 후 비용을 점검하거나, 비효율적인 프롬프트 패턴을 찾을 때 사용합니다.

---

### `/model`

현재 사용 중인 AI 모델을 전환합니다.

```
/model
```

사용 가능한 모델 목록이 나타나며, 방향키로 선택합니다.

```
> claude-opus-4-5
  claude-sonnet-4-5
  claude-haiku-3-5
```

작업 성격에 따라 모델을 선택하세요.

- 복잡한 코드 분석, 아키텍처 설계 → Opus
- 일반 개발 작업 → Sonnet (기본값, 비용 대비 성능 우수)
- 빠른 단순 작업 → Haiku

---

### `/effort`

Claude의 응답 노력 수준을 조정합니다. 수준이 높을수록 더 많은 토큰을 사용해 깊이 있게 생각합니다.

```
/effort low
/effort medium
/effort high
/effort max
```

| 수준 | 사용 시나리오 |
|------|--------------|
| `low` | 간단한 질문, 단순 코드 생성 |
| `medium` | 일반 개발 작업 (기본값) |
| `high` | 복잡한 버그 분석, 리팩토링 |
| `max` | 어려운 알고리즘, 아키텍처 설계 |

---

### `/login` / `/logout`

Anthropic 계정 인증을 관리합니다.

```
/login
/logout
```

`/login`은 브라우저를 열어 OAuth 인증을 진행합니다. API 키가 아닌 계정 기반 인증을 사용할 때 필요합니다.

---

### `/btw`

도구 접근(파일 읽기, 명령 실행 등) 없이 Claude에게 빠르게 질문합니다. "by the way"의 줄임말로, 현재 작업 컨텍스트를 유지하면서 간단히 물어볼 때 사용합니다.

```
/btw JavaScript에서 nullish coalescing이 뭐야?
```

파일을 읽거나 코드를 실행하지 않으므로 응답이 빠르고, 컨텍스트를 소모하지 않습니다.

---

### `/debug`

디버그 로깅을 활성화합니다. Claude Code 내부 동작(API 요청, 도구 실행, 에러 스택 등)을 상세히 출력합니다.

```
/debug
```

문제 진단 시 사용하며, 활성화된 상태에서 다시 `/debug`를 입력하면 비활성화됩니다. 환경 변수 `CLAUDE_CODE_DEBUG=true`로도 시작 시부터 활성화할 수 있습니다.

---

## 실습

### 실습 1: 비용 확인과 모델 전환

```bash
# Claude Code 실행 후 아래 순서로 입력

/cost           # 현재까지 사용한 비용 확인

/model          # 모델 목록 표시 → Haiku 선택

/btw 피보나치 수열을 재귀로 구현할 때 단점이 뭐야?
                # 빠른 질문 (도구 없이)

/cost           # 모델 전환 후 비용 비교
```

### 실습 2: 컨텍스트 관리

```bash
# 긴 작업 후 컨텍스트 상태 확인
/cost

# 압축 실행
/compact

# 비용 재확인 (압축 전후 토큰 수 비교)
/cost

# 새 작업 시작 전 완전 초기화
/clear
```

### 실습 3: 세션 관리

```bash
# 현재 세션 이름 지정
/rename
# 입력: backend-api-refactor

# 터미널 닫기 후 재시작
claude

# 이전 세션 재개
/resume
# 목록에서 backend-api-refactor 선택
```

### 실습 4: 권한과 MCP 확인

```bash
# 등록된 훅 확인
/hooks

# MCP 서버 상태 확인
/mcp

# 권한 규칙 UI 열기
/permissions
```
