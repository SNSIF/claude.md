# 4-3. 스킬 호출 방식

스킬을 만들었으면 두 가지 방법으로 실행할 수 있습니다. 직접 이름을 입력하는 명시적 호출과, Claude가 대화 내용을 보고 알아서 선택하는 자동 위임입니다.

---

## 명시적 호출

슬래시(`/`)에 스킬 이름을 붙여 직접 실행합니다.

```
/스킬명
/스킬명 인자
```

예시:

```
/commit
/deploy staging
/pr-review 142
/gen-migration add_users_table
```

스킬 frontmatter의 `name` 필드가 커맨드명이 됩니다. `name`을 지정하지 않았으면 파일명(확장자 제외)이 사용됩니다.

| 파일 경로 | name 미지정 | name: db-seed |
|-----------|-------------|---------------|
| `.claude/skills/db-seed.md` | `/db-seed` | `/db-seed` |
| `~/.claude/skills/review.md` | `/review` | 지정한 name |

---

## 자동 위임

대화 중 Claude가 스킬의 `description`을 보고 현재 요청에 맞는 스킬을 자동으로 선택해 실행합니다. 사용자가 스킬 이름을 몰라도 됩니다.

### 동작 원리

사용자가 "배포해줘"라고 입력하면:

1. Claude가 등록된 스킬 목록을 확인한다
2. 각 스킬의 `description`을 요청 내용과 비교한다
3. 가장 적합한 스킬을 선택해 실행한다

### 자동 위임이 잘 작동하는 description 예시

```yaml
# 좋은 예 — 구체적인 트리거 상황 명시
description: 스테이징된 Git 변경사항을 분석하고 Conventional Commits 형식으로 커밋합니다

# 나쁜 예 — 너무 모호함
description: 커밋 관련 작업
```

### 자동 위임 예시

```
# 사용자 입력
변경사항 커밋해줘

# Claude 내부 동작
→ "commit" 스킬 description과 일치
→ /commit 자동 실행
```

```
# 사용자 입력
PR #88 리뷰해줄 수 있어?

# Claude 내부 동작
→ "pr-review" 스킬 description과 일치
→ /pr-review 88 자동 실행
```

## 자동 위임 비활성화

특정 상황에서 Claude가 임의로 스킬을 실행하는 것을 막고 싶을 때 비활성화합니다.

### 방법 1: settings.json에서 전역 비활성화

```json
{
  "skills": {
    "autoDelegate": false
  }
}
```

이 설정 후에는 `/스킬명`으로만 실행됩니다.

### 방법 2: 스킬별 비활성화

스킬 frontmatter에 `auto-delegate: false`를 추가합니다.

```yaml
---
name: reset-db
description: 데이터베이스를 초기화하고 시드 데이터를 삽입합니다
allowed-tools: Bash
auto-delegate: false
---
```

위험한 작업(DB 초기화, 배포 등)은 자동 위임을 꺼두는 것이 안전합니다.

### 방법 3: 일시적으로 우회

자동 위임을 전체 활성화해두고, 특정 요청에서 스킬을 사용하지 않으려면 명시적으로 지시합니다.

```
스킬 사용하지 말고 직접 git commit 방법 알려줘
```

## 스킬 우선순위

같은 이름의 스킬이 여러 위치에 있을 때 적용 순서입니다.

```
프로젝트 스킬 (.claude/skills/)
    ↓ 없으면
사용자 전역 스킬 (~/.claude/skills/)
```

프로젝트 스킬이 항상 우선합니다. 전역 스킬을 프로젝트별로 재정의할 때 유용합니다.

### 예시: 전역 commit vs 프로젝트 commit

```
~/.claude/skills/commit.md          ← 일반 커밋 스킬
my-project/.claude/skills/commit.md ← Jira 이슈 번호를 자동 포함하는 커밋 스킬
```

`my-project` 안에서 `/commit`을 실행하면 프로젝트 버전이 실행됩니다. 다른 프로젝트에서는 전역 버전이 실행됩니다.

---

## 스킬 목록 확인

```
/skills
```

출력 예시:

```
등록된 스킬

[전역] ~/.claude/skills/
  /commit     스테이징된 변경사항을 커밋합니다
  /review     코드 품질을 분석하고 개선점을 제안합니다
  /explain    선택한 코드 블록을 설명합니다

[프로젝트] .claude/skills/
  /deploy     스테이징 또는 프로덕션에 배포합니다
  /seed-db    테스트용 시드 데이터를 삽입합니다
  /commit     Jira 이슈 연동 커밋 (전역 스킬 재정의)
```

## 스킬 디버깅

스킬이 예상대로 동작하지 않을 때 확인하는 방법입니다.

### 1. 디버그 모드 활성화

```
/debug
/commit
```

스킬 파일 파싱 결과, frontmatter 적용 여부, 도구 사용 로그가 상세히 출력됩니다.

### 2. 흔한 문제와 해결법

**스킬이 목록에 보이지 않는 경우**

```bash
# 파일 위치 확인
ls ~/.claude/skills/
ls .claude/skills/

# 파일 확장자 확인 (.md 이어야 함)
# commit.md (O) / commit.txt (X) / commit (X)
```

**frontmatter가 인식되지 않는 경우**

```markdown
<!-- 잘못된 예 — 파일 맨 앞에 공백/빈 줄이 있으면 안 됨 -->

---
name: commit
---

<!-- 올바른 예 — 파일의 첫 번째 문자가 --- 이어야 함 -->
---
name: commit
---
```

**allowed-tools 오류**

```yaml
# 잘못된 예 — 존재하지 않는 도구
allowed-tools: Shell, FileRead

# 올바른 예
allowed-tools: Bash, Read
```

**인자($ARGUMENTS)가 빈 문자열인 경우**

스킬 본문에서 인자 없이 호출된 경우를 처리합니다.

```markdown
$ARGUMENTS 가 비어 있으면 현재 브랜치의 최신 커밋을 대상으로 한다.
$ARGUMENTS 가 있으면 해당 값을 PR 번호로 사용한다.
```

## 실습: 만든 스킬 호출해보기

4-2에서 만든 `commit` 스킬을 다양한 방법으로 호출합니다.

### 준비

```bash
# 테스트용 프로젝트 생성
mkdir ~/skill-test && cd ~/skill-test
git init
echo "console.log('hello')" > index.js
git add index.js
```

### 명시적 호출 테스트

```bash
# Claude Code 실행
claude

# 스킬 목록 확인
/skills

# 스킬 명시적 실행
/commit
```

### 자동 위임 테스트

```bash
# 새 변경사항 만들기
echo "console.log('world')" >> index.js
git add index.js

# 자연어로 요청 (자동 위임)
이 변경사항 커밋해줘
```

Claude가 commit 스킬을 자동으로 선택해 실행하는지 확인합니다.

### 프로젝트 스킬로 전역 스킬 재정의

```bash
mkdir -p .claude/skills
```

아래 내용으로 `.claude/skills/commit.md`를 생성합니다.

```markdown
---
name: commit
description: 프로젝트 전용 커밋 스킬 — feat/fix 접두사 강제
allowed-tools: Bash
---

1. `git diff --staged` 확인
2. 반드시 `feat:` 또는 `fix:` 로 시작하는 메시지 작성
3. 커밋 실행
```

```bash
# 스킬 목록 재확인 — 프로젝트 스킬이 전역 스킬을 재정의했는지 확인
/skills

# 재정의된 스킬 실행
/commit
```
